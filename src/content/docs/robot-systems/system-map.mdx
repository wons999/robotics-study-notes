# 로봇 시스템 지도

## 주요 축

| 축 | 기록할 내용 |
| --- | --- |
| Mechanics | DoF, actuation, compliance, backlash, durability |
| Sensing | observation source, calibration, noise, latency, reliability |
| Control | joint control, task-space control, impedance, safety constraint |
| Learning | data collection, imitation policy, RL fine-tuning, residual control |
| Evaluation | task success, robustness, recovery, repeatability, cycle time |

## 작업 가설

로봇 behavior는 policy만의 문제가 아니다. hardware, sensing, control, data가 함께 만드는 결과다. 이 섹션의 노트는 그 의존성을 명시적으로 드러내야 한다.

## 계속 물어볼 질문

- task failure가 perception, control, mechanics, data 중 어디서 생겼는가?
- learning을 얹기 전에 model-based control로 해결해야 할 부분은 무엇인가?
- closed-loop policy input으로 쓸 만큼 안정적인 observation은 무엇인가?
- 이 시스템에 자연스러운 action representation은 joint command, task-space command, torque, latent motor command 중 무엇인가?
