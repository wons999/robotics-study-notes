# FK, IK, Jacobian

## 왜 중요한가

로봇 manipulation은 end-effector가 지금 어디 있는지, joint motion이 pose를 어떻게 바꾸는지, 그리고 constraint를 깨지 않으면서 목표 pose로 가려면 joint를 어떻게 업데이트해야 하는지에 달려 있다.

## 핵심 용어

| 용어 | 의미 | 실전 질문 |
| --- | --- | --- |
| FK | joint angle에서 end-effector pose를 계산 | end-effector가 지금 어디 있는가? |
| IK | 원하는 end-effector pose에서 joint angle을 찾음 | 로봇을 어떻게 그 위치로 보낼 것인가? |
| Jacobian | joint velocity와 end-effector velocity의 local map | 어떤 joint가 이 방향 움직임을 만드는가? |
| Singularity | 움직일 수 있는 방향이 사라지거나 증폭되는 configuration | solver가 왜 갑자기 불안정해졌는가? |
| Redundancy | task constraint보다 DoF가 많은 상태 | 로봇이 어떤 posture를 선호해야 하는가? |

## 최소 numerical IK 루프

```txt
q <- current joint state
repeat:
  x <- FK(q)
  e <- target_pose_error(x_target, x)
  J <- jacobian(q)
  dq <- J.T (J J.T + lambda^2 I)^-1 e
  q <- q + alpha dq
```

## Manipulation 시스템에서의 메모

Manipulation에서 IK는 단순히 pose에 도달하는 문제로 끝나지 않는다. joint limit, collision, contact, controller bandwidth, object motion이 모두 문제에 들어온다. 기하학적으로 깔끔한 해라도 시스템 constraint를 무시하면 실제로는 쓸 수 없을 수 있다.

## 열린 질문

- robot stack은 어떤 coordinate frame convention을 쓰는가?
- controller는 어떤 task frame을 기준으로 움직이는가?
- joint limit, collision constraint, preferred posture는 어디에 encode되어 있는가?
- feedback은 IK나 impedance objective를 어떻게 바꿔야 하는가?
