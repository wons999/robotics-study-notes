# T-Rex 세미나

T-Rex를 tactile-aware VLA 논문 하나로만 읽지 않고, **느린 vision-language reasoning과 빠른 tactile correction을 어떻게 한 policy 안에서 나누는가**라는 시스템 설계 문제로 읽는다.

## 반드시 전달할 Takeaway

| Takeaway | 설명 |
| --- | --- |
| tactile은 static input이 아니다 | 빠른 correction signal로 다뤄야 한다. |
| dataset 설계가 중요하다 | 100시간의 의미는 object-primitive coverage와 synchronized tactile stream에 있다. |
| architecture가 frequency mismatch를 풀어야 한다 | 느린 VLA backbone과 빠른 tactile feedback 사이를 분리해야 한다. |
| failure case가 다음 연구 방향이다 | alignment, force magnitude, multi-finger coordination, recovery가 남아 있다. |

## 토론 질문

1. tactile expert가 raw vision을 보지 않는 구조는 장점인가, 한계인가?
2. force VQ-VAE token은 실제 deployment에서 sensor drift에 얼마나 robust할까?
3. `pi_0.5 + tactile`이 낮은 성능을 보인 것은 tactile input 자체의 문제인가, integration 방식의 문제인가?
4. T-Rex 구조를 force-torque wrist sensor만 있는 robot에 옮기면 무엇이 빠질까?
5. Appendix H의 실패 사례를 줄이려면 RL, MPC, impedance control 중 무엇을 먼저 붙여야 할까?
