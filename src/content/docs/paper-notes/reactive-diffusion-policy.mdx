# Reactive Diffusion Policy

## 메타데이터

| 항목 | 내용 |
| --- | --- |
| 논문 | Reactive Diffusion Policy: Slow-Fast Visual-Tactile Policy Learning for Contact-Rich Manipulation |
| 저자 | Han Xue, Jieji Ren, Wendi Chen, Gu Zhang, Yuan Fang, Guoying Gu, Huazhe Xu, Cewu Lu |
| 연도 | RSS, 2025 |
| 링크 | https://arxiv.org/abs/2503.02881 |
| 태그 | tactile, diffusion policy, slow-fast control, imitation learning, contact-rich manipulation |

## 한 줄 요약

RDP는 느린 latent diffusion policy와 빠른 tactile feedback controller를 결합해, action chunk를 실행하는 도중에도 접촉 변화에 즉시 반응하게 만든 visual-tactile imitation learning 방법이다.

## 왜 T-Rex와 연결되는가

T-Rex를 이해할 때 RDP는 거의 바로 앞 단계의 선행 연구처럼 읽으면 된다. 둘 다 같은 병목을 본다.

| 병목 | RDP의 답 | T-Rex의 확장 |
| --- | --- | --- |
| action chunk는 길고 tactile 변화는 빠르다 | slow latent diffusion + fast tactile tokenizer | low-frequency action expert + high-frequency tactile expert |
| contact-rich task는 vision만으로 부족하다 | tactile/force feedback을 closed-loop correction에 사용 | force history + deformation map을 tactile expert에 사용 |
| teleoperation data에 tactile이 부족하다 | TactAR로 operator에게 tactile feedback 제공 | 100시간 tactile-synchronized bimanual dataset 구축 |

## 문제 설정

| 항목 | 내용 |
| --- | --- |
| task | 접촉 상태가 빠르게 바뀌는 contact-rich manipulation |
| observation | vision, proprioception, tactile/force feedback |
| action | 저주파 action chunk + 고주파 tactile correction |
| 학습 | demonstration 기반 imitation learning |
| 핵심 난점 | action chunking은 긴 동작을 잘 모델링하지만, chunk 내부의 tactile 변화에 늦게 반응한다 |

## 데이터와 teleoperation

RDP는 **TactAR**라는 teleoperation system을 함께 제안한다. 목적은 operator가 tactile/force feedback을 실시간으로 느끼면서 demonstration을 만들게 하는 것이다.

읽을 때 중요한 포인트는 데이터가 단순히 “vision trajectory”가 아니라는 점이다. 접촉이 중요한 과제에서는 demonstration을 수집하는 순간부터 tactile feedback이 operator의 행동에 영향을 준다. 즉, 데이터 자체가 contact-aware behavior를 포함한다.

## 모델 구조

RDP의 구조는 두 층으로 나뉜다.

| 구성 | 역할 |
| --- | --- |
| slow latent diffusion policy | 낮은 주파수에서 high-level action chunk를 latent space에서 생성 |
| fast asymmetric tokenizer | tactile feedback을 이용해 chunk 실행 중 빠르게 보정 |
| visual encoder | scene context와 object state를 반영 |
| tactile/force input | 접촉 변화와 힘 변화를 빠른 controller에 제공 |

핵심은 “diffusion policy가 action chunk를 만들고 끝”이 아니라, chunk 실행 중 tactile stream이 action을 계속 조정한다는 점이다.

## 방법론

RDP는 contact-rich behavior를 두 시간 스케일로 분해한다.

1. 느린 시간 스케일에서는 diffusion policy가 전체 trajectory shape를 만든다.
2. 빠른 시간 스케일에서는 tactile feedback이 현재 접촉 상태를 보고 action을 수정한다.
3. 이 둘을 분리해도 완전히 따로 노는 것이 아니라, latent/action representation을 통해 연결한다.

이 구조는 T-Rex의 asynchronous cascaded flow matching을 이해하는 좋은 준비 운동이다. T-Rex도 결국 “비싼 visual reasoning은 느리게, tactile correction은 빠르게”라는 같은 방향으로 간다.

## 실험에서 봐야 할 점

- visual-only imitation baseline 대비 contact-rich task에서 얼마나 좋아지는가?
- tactile/force sensor 종류가 달라져도 robust한가?
- 빠른 tactile pathway를 제거하면 어떤 실패가 늘어나는가?
- action chunk 길이가 길어질수록 tactile correction의 중요성이 커지는가?

## T-Rex를 읽기 위한 기억 포인트

- RDP는 tactile reactivity의 필요성을 잘 보여주는 baseline이다.
- T-Rex는 RDP보다 더 큰 VLA/foundation model setting으로 이 문제를 가져간다.
- RDP는 task-level imitation learning에 가깝고, T-Rex는 tactile-rich mid-training을 통한 reusable contact prior를 노린다.

## 한계

- generalist VLA라기보다는 contact-rich task를 위한 imitation learning architecture에 가깝다.
- language-conditioned broad generalization보다는 tactile-reactive control 자체가 중심이다.
- 대규모 human egocentric pretraining이나 bimanual dexterous tactile dataset까지는 다루지 않는다.
