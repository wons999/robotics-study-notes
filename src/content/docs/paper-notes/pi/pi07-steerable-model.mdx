## 한 줄 요약

π0.7은 단순 language instruction만 쓰는 VLA가 아니라, subtask instruction, generated visual subgoal, episode metadata, control mode를 함께 prompt로 넣어 mixed-quality robot/human/web/autonomous data를 학습 가능한 context로 정렬하고, 단일 generalist policy가 specialist-level dexterity, instruction following, cross-embodiment transfer, compositional task generalization을 보이도록 만든 PI 계열의 steerable VLA다.

## 문제의식

기존 VLA는 semantic concept을 이해하더라도 skill을 새 방식으로 재조합하는 능력이 약했다. 예를 들어 “object label이 새롭다” 수준의 generalization은 가능해져도, 새 appliance를 조작하거나, 다른 robot embodiment에서 같은 dexterous skill을 수행하거나, 같은 task를 더 빠르고 안정적인 strategy로 수행하는 것은 별도 fine-tuning 또는 specialist policy가 필요했다.

π0.7의 문제 설정은 다음 질문으로 읽을 수 있다.

| 질문 | π0.7의 답 |
| --- | --- |
| 다양한 품질의 data를 그냥 섞으면 되는가? | 아니며, episode quality/speed/mistake/control mode를 prompt metadata로 넣어야 한다. |
| VLA가 “무엇”뿐 아니라 “어떻게”를 따를 수 있는가? | subtask instruction, metadata, visual subgoal을 통해 strategy와 desired behavior를 steer한다. |
| task-specific specialist가 하던 dexterous task를 single model이 할 수 있는가? | π*0.6 specialist tasks에서 single π0.7이 비슷하거나 더 높은 throughput을 보인다. |
| unseen task를 action demonstration 없이 가르칠 수 있는가? | live language coaching episode를 high-level policy로 distill해 autonomous execution으로 연결한다. |
| cross-embodiment transfer가 단순 imitation인가? | UR5e transfer에서는 source strategy를 그대로 복사하지 않고 target morphology에 맞는 strategy가 나타난다. |

## 시스템 설정

논문은 single benchmark가 아니라 여러 robot platform에서 π0.7의 out-of-box, instruction-following, transfer, compositional generalization을 평가한다.

| 축 | 내용 |
| --- | --- |
| robot platforms | bimanual mobile manipulators, static bimanual robots, bimanual UR5e with Robotiq grippers, single-arm 6 DoF systems |
| gripper | 모든 manipulator는 parallel-jaw gripper |
| cameras | front camera, wrist camera on each arm, mobile robots는 rear-facing camera 추가 |
| control frequency | UR5e robots는 20 Hz, 그 외 robot은 50 Hz |
| action application | simple PD controller |
| end-effector commands | numerical IK로 target end-effector pose를 target joint position으로 변환 |
| inference action chunk | 50-step action chunks |
| denoising | 모든 실험에서 5 denoising steps |
| execution horizon | chunk 중 `H_hat` in `{15, 25}` steps를 실행 |

## Dataset

논문은 exact total hour/frame count를 한 표로 공개하지 않는다. 대신 data source의 성격과 이들을 prompt metadata로 정렬하는 방식이 핵심이다.

| Data source | 역할 |
| --- | --- |
| robot demonstrations | 다양한 task, robot platform, lab/home/in-the-wild environment의 기본 action data |
| suboptimal demonstrations | failure episode 또는 mistake가 많은 success episode도 metadata와 함께 사용 |
| autonomous evaluation episodes | prior policies, 특히 π*0.6 RL training 중 생성된 rollout을 distillation source로 사용 |
| human interventions | policy rollout 중 사람이 개입한 data |
| open-source robot datasets | cross-embodiment/generalist prior 강화 |
| egocentric human videos | human-object interaction prior와 world model/subgoal generation 보조 |
| multimodal web data | object localization, attribute prediction, visual QA, text-only prediction, video-language tasks |

중요한 점은 π0.7이 “좋은 data만 골라 쓰는” 방향이 아니라, 낮은 품질의 data도 품질/속도/mistake metadata를 붙여 학습에 포함한다는 것이다. 이 때문에 test time에는 high quality, no mistake, desired speed 같은 metadata를 prompt로 넣어 desired behavior를 유도할 수 있다.

## Method: Diverse Multimodal Prompting

π0.7의 training objective는 trajectory action chunk를 최근 observation history와 context `C_t`에 조건화하는 VLA objective로 정리된다.

```text
maximize E_D [ log πθ(a_{t:t+H} | o_{t-T:t}, C_t) ]
```

flow matching action expert는 이 likelihood를 직접 계산하지 않고, conditional flow objective로 action chunk를 생성한다. π0.7의 차이는 `C_t`를 단순 instruction으로 두지 않는 데 있다.

| Prompt component | 기호/형태 | 역할 |
| --- | --- | --- |
| task instruction | `l_t` | 전체 task intent, 예: clean the kitchen |
| subtask instruction | `l_hat_t` | 다음 semantic subtask, 예: open the fridge door |
| visual subgoal | `g_t` | near-future desired scene state, multi-view goal image |
| episode metadata | `m` | speed, quality, mistake label |
| control mode | `c` in `{joint, ee}` | joint-level 또는 end-effector action mode를 명시 |
| proprioception | `q_t` | robot state, model backbone token으로 embedding |

### Subtask Instruction

π0.5처럼 high-level semantic subtask를 low-level VLA prompt에 넣는다. 차이는 π0.7이 이 subtask instruction을 human live coaching, learned high-level policy, 또는 test-time generated plan에서 받아 유연하게 사용할 수 있다는 점이다.

### Visual Subgoal

subtask instruction은 “open the fridge door”처럼 intent는 주지만, 어느 손으로 어느 angle로 잡을지 같은 spatial detail은 부족할 수 있다. visual subgoal은 desired near-future image를 제공해 object-centric outcome과 arm/gripper outcome을 동시에 지정한다.

world model은 `g_ψ(g* | o_t, l_hat_t, m)` 형태로 subgoal image를 생성하며, BAGEL 14B mixture-of-transformers image model에서 초기화된다. training target은 segment end frame 또는 future frame이고, web/non-robot/human video data를 섞어 semantic/physical concept을 보존한다.

### Episode Metadata

| Metadata | 정의 | test-time 사용 |
| --- | --- | --- |
| overall speed | episode length를 500-step bin으로 discretize | task별 episode length의 15th percentile을 넣어 빠른 실행 유도 |
| overall quality | 1-5 score, 5가 highest quality | 항상 5로 설정해 high-quality behavior 유도 |
| mistake | segment 안 mistake 여부 | 항상 false로 설정해 mistake-free behavior 유도 |

이 metadata가 핵심이다. 낮은 품질의 autonomous data, failure, mistake 포함 episode도 학습에 넣되, model이 “이 data는 어떤 behavior quality를 나타내는지” 구분하게 만든다.

### Control Mode

control mode token은 `joint` 또는 `ee`를 prompt에 넣는다. Appendix E의 Figure 20은 prior models에서 EE control이 cross-embodiment tasks에 뚜렷한 이점을 주지 않았다고 보고하므로, main cross-embodiment 실험은 joint-space control 중심으로 정리된다.

## Model Architecture

| 구성 | 논문 기준 값/설명 |
| --- | --- |
| total parameter | 약 5B |
| VLM backbone | Gemma3 4B VLM |
| vision encoder | Gemma3/SigLIP 계열 400M vision encoder |
| video history | MEM-style encoder, temporal/spatial compression |
| action expert | 860M transformer, continuous action flow matching |
| action tokens | fixed 50 tokens, 50-step action chunk |
| timestep conditioning | adaptive RMSNorm |
| action attention | 50 action tokens는 서로 bidirectional attention, VLM backbone activation에도 attend |
| state/proprioception | MEM 방식처럼 linear projection으로 backbone dimension에 embedding |
| KI/FAST relation | VLM backbone은 FAST token CE supervision으로 안정화하고, action expert gradient는 VLM backbone으로 흐르지 않게 하는 Knowledge Insulation recipe를 사용 |

### Input Shape 관점

논문은 exact tensor shape를 config table로 주지는 않지만, architectural contract는 명확하다.

| Input | 범위 |
| --- | --- |
| camera observations | 최대 4 camera images: front, two wrist, optional rear |
| history frames | camera마다 최대 6 history frames |
| visual subgoals | 최대 3 subgoal images, rear view는 제외 |
| image resolution | observation/subgoal images는 `448x448`로 resize |
| history sampling | 1 second stride |
| state tokens | history state를 individual token으로 처리 |
| action chunk | 50 action tokens |

### Attention Pattern

π0.7은 image goals가 없으면 π0.5와 유사한 memory-aware image views attention pattern을 사용한다. image goals가 있으면 text prompt 뒤에 block-causal bidirectional block으로 추가한다. CFG inference에서는 positive/negative examples를 같은 sequence에 packing하되 서로 attend하지 않는 attention tree를 구성한다. world model도 current observation, VAE-encoded observation, noisy image goal group을 나눠 block-bidirectional attention을 사용한다.

## Training Recipe

| 학습 요소 | 내용 |
| --- | --- |
| backbone/action recipe | KI: VLM은 FAST CE loss로 안정화, action expert는 flow matching |
| robot data | high-quality demo뿐 아니라 suboptimal demo, failure, autonomous rollout까지 포함 |
| subgoal images in batch | visual subgoal은 batch example의 25%에만 추가 |
| subtask dropout | subgoal이 있는 example에서 subtask instruction을 30% dropout |
| metadata dropout | 전체 metadata는 15% dropout, speed/quality/mistake component는 각각 5% dropout |
| control mode dropout | 적용하지 않음 |
| history frame dropout | entire history frames 0.3 probability |
| rear view dropout | rear view image가 있으면 0.3 probability |
| subgoal real image sampling | 0.25 probability로 end-of-segment image, 0.75 probability로 0-4 seconds future image |
| generated subgoal training | world model generated images를 추가 training example로 넣어 train-test mismatch 완화 |
| RTC training | inference delay 0-12 timesteps를 simulate, 50Hz robot 기준 최대 240 ms |

## Runtime Inference

π0.7의 runtime은 single VLA call만이 아니라 high-level policy, world model, VLA가 비동기로 돌아가는 구조다.

| 단계 | 설명 |
| --- | --- |
| initial context | initial observation, task instruction, episode metadata, control mode |
| subtask generation | high-level policy 또는 human coaching이 `l_hat` 생성 |
| subgoal generation | world model이 current observation, subtask, metadata로 `g*` 생성 |
| context assembly | `{l, l_hat, g*, m, c}` |
| VLA action | `πθ(a given observation history and context C)`로 50-step action chunk 생성 |
| subgoal refresh | subtask가 바뀌거나 4 seconds가 지나면 non-blocking async로 새 subgoal 생성 |
| action refresh | `H_hat` steps가 지나면 RTC overlap을 고려해 새 action chunk 생성 |
| CFG | episode metadata 등 prompt component를 guidance target으로 사용, β in `{1.3, 1.7, 2.2}` |

minimal π0.7 inference는 H100 1장 기준 3 camera inputs, 5 denoising steps, training-time RTC에서 38 ms까지 최적화되었다고 appendix가 설명한다. MEM encoder와 subgoal images까지 켜면 overhead가 커지고 worst case 127 ms까지 간다. world model은 14B급이고 4xH100 tensor parallelism, 8-bit large matmul, SageAttention 변형을 사용해 25 denoising steps subgoal generation을 1.25 seconds로 만든다. 그래서 runtime은 subgoal generation을 blocking하지 않고 async로 돌린다.

## 한계와 주의

- π0.7 code/checkpoint는 public OpenPI inventory에서 확인되지 않았다. 재현 가능한 open-source release로 읽으면 안 된다.
- 논문은 exact total data hours/frames, optimizer, full training schedule을 모두 표로 공개하지 않는다.
- “unseen task”는 대규모 mixed dataset에서 완전히 증명하기 어렵다. 논문도 관련 skill이 다른 label/context로 존재할 수 있음을 한계로 인정한다.
- world model subgoal은 강력하지만 14B model과 4xH100 inference optimization이 필요하다. deployment cost가 작지 않다.
- generated subgoal이 틀리면 VLA를 잘못 steer할 수 있다. paper는 positive result 중심이고 failure taxonomy는 제한적이다.
- cross-embodiment result는 특정 robot/task set에 기반한다. tactile/contact-rich force feedback policy와는 modality setting이 다르다.

## 스터디에서 기억할 점

π0.7의 핵심은 “prompt를 instruction string에서 behavior contract로 확장했다”는 점이다. language는 task를 말하고, subtask는 다음 semantic step을 말하고, visual subgoal은 desired spatial state를 말하고, metadata는 speed/quality/mistake를 말하고, control mode는 action interface를 말한다. 이 context가 있기 때문에 mixed-quality data와 specialist/autonomous experience를 단일 generalist VLA로 되돌릴 수 있다.
