## 한 줄 요약

이 article은 외부에서 제안된 everyday manipulation challenge인 Robot Olympics를 π0.6 기반 모델에 task별로 fine-tuning해 시험한 capability note이며, 대부분 task에서 9시간 미만의 data로 평균 success `52%`, task progress `72%`를 얻었지만, π0.6 pretraining 없이 standard VLM을 fine-tune한 baseline은 어떤 task도 성공하지 못하고 progress `9%`에 그쳤다는 점을 강조한다.

## 왜 중요한가

이 글은 formal paper가 아니고, exact protocol이나 full result table도 없다. 그래도 PI family에서 중요하다. 이유는 세 가지다.

| 이유 | 의미 |
| --- | --- |
| π0.6의 중간 capability signal | π0.5 이후, π*0.6/RECAP과 π0.7 이전의 model generation이 실제 새 physical skill fine-tuning에 쓰인 사례다. |
| external task selection | task를 PI가 고른 것이 아니라 Benjie Holson의 Robot Olympics challenge에서 가져왔기 때문에 cherry-picking 위험을 일부 줄인다. |
| baseline contrast | standard VLM fine-tuning baseline이 no success / 9% progress였다는 비교로 robot foundation pretraining 필요성을 강조한다. |

Moravec's paradox 관점에서 이 글의 메시지는 “semantic reasoning이 아니라 physical grounding이 병목”이라는 것이다. LLM/VLM은 task 설명을 이해할 수 있어도, key reorientation, wet sponge, grease, bag edge separation, deformable cloth처럼 몸으로 익혀야 하는 skill basis가 없으면 실행하지 못한다.

## Challenge Setup

| 항목 | 공식 article 기준 |
| --- | --- |
| challenge source | Benjie Holson의 Robot Olympics challenge |
| task 선택 | PI가 직접 고른 내부 benchmark가 아니라 외부 proposed tasks |
| model | latest model based on `π0.6` |
| 학습 방식 | task별 fine-tuning |
| data 규모 | 대부분 task에서 `under 9 hours`; sock inside-out은 약 `8 hours` |
| 결과 요약 | 5개 category 중 3개에서 gold-medal task initial solution, 나머지 2개는 silver-medal task |
| aggregate metric | success rate `52%`, task progress `72%` |
| baseline | π0.6 없이 standard VLM fine-tuning. 어떤 task도 성공하지 못했고 average task progress `9%` |
| caveat | policies are often not consistent. 높은 success rate를 위해 모든 optimization을 한 것은 아님 |
| setup caveat | 일부 task는 original challenge가 mobile robot을 상정하지만 fixed non-mobile robot으로 수행 |

여기서 success와 task progress를 구분해야 한다. success는 완전 성공이고, progress는 task 일부를 얼마나 수행했는지 보는 partial measure다. article은 reliability/speed 최적화는 RECAP 같은 별도 work와 연결하고, 이 글 자체는 “π0.6 foundation model을 fine-tune하면 어디까지 되는가”를 보여주는 capability snapshot에 가깝다.

## Quantitative Claims

| Claim | 해석 |
| --- | --- |
| 3 out of 5 categories에서 gold task initial solution | π0.6 fine-tuning으로 외부 challenge의 최상위 task 일부를 처음 풀었다는 capability claim |
| 나머지 2 categories는 silver task | gold task 두 개는 hardware/gripper constraint 때문에 직접 성공으로 count하지 않음 |
| most tasks under 9 hours data | task별 대규모 teleoperation 없이도 foundation model fine-tuning으로 initial solution을 얻었다는 주장 |
| average success 52% | robust deployment 성능이라기보다 initial policies의 완전 성공률 |
| task progress 72% | partial completion까지 보면 더 높은 진행률 |
| standard VLM baseline 0 success / 9% progress | generic VLM fine-tuning만으로는 physical skill basis가 부족하다는 비교 |

article이 직접 “policies are often not consistent”라고 제한을 둔 점을 중요하게 봐야 한다. 이 글은 π0.6이 이미 production-grade로 안정적이라는 주장이 아니라, physical foundation pretraining이 없으면 시작조차 어려운 task를 작은 task data로 어느 정도 열 수 있다는 evidence다.

## Model / Training Interpretation

공식 article은 architecture diagram이나 optimizer/hyperparameter를 제공하지 않는다. 따라서 여기서는 π0.6 family context를 기준으로 해석을 제한한다.

| 축 | article에서 확인되는 것 | 확인되지 않는 것 |
| --- | --- | --- |
| base | π0.6 기반 latest model | exact checkpoint/config |
| downstream | task별 fine-tuning | optimizer, batch size, step count, data split |
| data | 대부분 task under 9h, sock ~8h | episode count, success/failure distribution |
| baseline | standard VLM fine-tuning without π0.6 | baseline architecture/config detail |
| evaluation | success rate, task progress | per-task full table, confidence intervals |

이 때문에 이 페이지는 paper survey가 아니라 article note로 둔다. π0.6의 architecture와 RECAP의 RL/fine-tuning loop는 각각 [π*0.6 / RECAP](../pistar06-recap-learns-from-experience/)과 다른 PI survey에서 확인하는 것이 맞다.

## Interpretation

이 article의 핵심 해석은 Moravec's paradox를 data sparsity 관점으로 바꾸는 것이다.

| 관점 | 설명 |
| --- | --- |
| LLM/VLM knowledge | web text/image는 “task가 무엇인지”를 설명할 수 있지만, sponge를 어느 힘으로 누르고 key를 어떻게 돌릴지는 직접 담지 않는다. |
| Physical skill basis | diverse robot behavior pretraining은 downstream task를 위한 low-level physical prior를 제공한다. |
| Fine-tuning | 새 task마다 모든 것을 처음부터 배우는 것이 아니라, 기존 physical prior를 task-specific data로 조정한다. |
| 후속 병목 | low-level skill basis가 좋아질수록 bottleneck은 reliability, speed, higher-level instruction, autonomous experience로 이동한다. |

이 관점은 바로 다음 흐름과 연결된다. [Human-to-Robot Transfer](../human-to-robot-transfer/)는 task-specific data source를 human video로 넓히고, [π*0.6 / RECAP](../pistar06-recap-learns-from-experience/)은 autonomous rollout/correction/RL로 reliability와 throughput을 끌어올린다.

## Study Takeaways

1. Robot Olympics는 π0.6의 “새 physical skill fine-tuning” capability를 보여주는 article note다.
2. 성공률 52%는 좋은 신호지만 robust deployment로 읽으면 안 된다. consistency 문제는 article이 직접 인정한다.
3. baseline no-success / 9% progress는 generic VLM만으로 physical skill을 grounding하기 어렵다는 근거다.
4. Gold task를 풀지 못한 category는 failure라기보다 hardware/task-rule caveat까지 함께 읽어야 한다.
5. 이 글은 RECAP, RLT, human-to-robot transfer가 왜 필요한지 보여주는 중간 연결고리다.
