## 한 줄 요약

Knowledge Insulation은 action expert의 flow-matching gradient가 pretrained VLM backbone을 손상시키는 문제를 막기 위해, backbone은 FAST-tokenized discrete action과 VLM/web task로 학습하고 action expert는 continuous flow objective로 학습하되 action expert에서 backbone으로 가는 gradient를 끊는 single-stage VLA training recipe다.

## 문제의식

<PaperFigure
  src="/pi/knowledge-insulation-paper/fig02-standard-vla-problems.png"
  alt="Figure 2 standard VLA recipe problems"
  caption="Fig. 2. continuous action expert, autoregressive FAST, KI recipe가 각각 language following, training speed, inference speed에서 어떤 trade-off를 갖는지 보여준다."
/>

VLM은 web-scale semantic knowledge와 language reasoning을 갖고 있지만, robot control에는 high-frequency continuous action이 필요하다. 단순 autoregressive VLA는 action을 token으로 내보낼 수 있지만, action chunk를 순차 decoding해야 하므로 느리고 continuous precision에도 한계가 있다. 논문은 π0-FAST가 1초 action chunk를 예측하는 데 RTX4090에서 약 750 ms가 걸린다고 적고, autoregressive VLA control frequency를 약 1.3 Hz 수준으로 본다.

π0 계열은 이 문제를 action expert와 flow matching으로 해결한다. action expert는 smaller module이므로 π0는 약 10 Hz control frequency를 달성할 수 있다. 하지만 새로 초기화한 action expert의 gradient가 pretrained VLM backbone을 직접 건드리면 language following과 training dynamics가 망가질 수 있다. 논문이 말하는 "knowledge insulation"은 바로 이 gradient interference를 막는 recipe다.

핵심 문제는 세 가지다.

| 문제 | 관찰 | 왜 중요한가 |
| --- | --- | --- |
| AR-VLA는 느림 | π0-FAST는 language following은 좋지만 task completion 시간이 길다. | real-time control과 dynamic manipulation에서 latency가 누적된다. |
| action expert gradient가 VLM을 훼손 | π0/joint-training은 일부 task에서 language instruction을 덜 따른다. | VLA의 장점인 web-scale semantic transfer가 사라진다. |
| backbone freeze만으로는 부족 | pretrained VLM은 robot control representation을 직접 갖고 있지 않다. | freeze는 knowledge는 보존하지만 action expert가 쓸 representation을 못 만든다. |

## Method

<PaperFigure
  src="/pi/knowledge-insulation-paper/fig01-knowledge-insulation-architecture.png"
  alt="Figure 1 Knowledge Insulation architecture"
  caption="Fig. 1. discrete action token objective와 continuous action expert objective를 동시에 쓰되, action expert gradient는 backbone으로 보내지 않는다."
/>

KI의 핵심은 "FAST와 flow 중 하나를 고르는 것"이 아니라, 둘을 학습 때 동시에 쓰고 inference 때는 continuous action expert를 쓰는 것이다.

| 구성 | 학습 때 역할 | inference 때 역할 |
| --- | --- | --- |
| FAST-tokenized discrete action | backbone이 robot action representation을 next-token CE로 빠르게 학습하게 함 | 사용하지 않음 |
| VLM/web/high-level task | semantic knowledge와 language/object prior를 유지 | prompt/context understanding에 간접 기여 |
| continuous action expert | flow matching으로 continuous action chunk 생성 능력을 학습 | 실제 action chunk를 빠르게 생성 |
| stop-gradient | action expert loss가 pretrained VLM backbone을 망가뜨리지 않게 함 | inference에는 gradient가 없으므로 구조적 학습 효과만 남음 |

논문은 standard VLA training을 두 loss로 정리한다.

```text
L_AR-VLA = E[- sum_j M_j log p_theta(y_{j+1} | x_{1:j})]

L_FLOW-VLA = E[ || omega - a_{1:H} - f_theta^a(a_{1:H}^{tau,omega}) ||^2 ]

a_{1:H}^{tau,omega} = tau * a_{1:H} + (1 - tau) * omega,
omega ~ N(0, I)
```

KI의 co-training objective는 language/FAST token prediction과 flow matching을 함께 묶는다.

```text
L_CO-VLA =
E[
  - sum_j M_l_j log p_theta(l_hat_{j+1} | x_{1:j})
  + alpha * M_act * || omega - a_{1:H} - f_theta^a(a_{1:H}^{tau,omega}) ||^2
]
```

여기서 `l_hat`은 일반 language token과 FAST-tokenized action token을 모두 포함한다. `M_l`은 token loss mask이고, `M_act`는 해당 sample에서 continuous action을 예측할지 나타낸다. 이 설계 덕분에 image-text-only VLM data, action-only robot data, language+action robot data를 한 training mixture 안에 섞을 수 있다.

중요한 attention rule은 FAST action token과 continuous action token이 서로 attend하지 못하게 하는 것이다. 둘이 서로를 보면 같은 action label의 두 표현이 shortcut처럼 새어 들어가 representation learning 실험이 흐려진다.

## Knowledge Insulation Gradient

논문의 stop-gradient는 "backbone을 전부 freeze한다"와 다르다. backbone은 FAST token/VLM objective로 계속 학습한다. 다만 continuous action expert의 flow loss gradient만 backbone으로 못 올라가게 한다.

논문은 attention probability를 backbone token과 action expert token으로 나누어 설명한다.

| block | 의미 |
| --- | --- |
| `P_bb` | backbone token이 backbone token에 attend |
| `P_ab` | action expert token이 backbone token에 attend |
| `P_aa` | action expert token이 action expert token에 attend |
| backbone -> action expert | backbone token은 action expert token을 attend하지 않음 |
| action expert -> backbone | forward 정보는 볼 수 있지만, `K_b`, `V_b` 쪽 gradient는 stop-gradient 처리 |

즉 forward pass에서는 action expert가 VLM backbone representation을 조건으로 삼을 수 있다. 하지만 backward pass에서는 action expert loss가 backbone의 key/value representation을 바꾸지 못한다. 이 때문에 논문은 `alpha=1`을 쓸 수 있다고 설명한다. flow loss가 사실상 독립된 weight set에 적용되기 때문이다.

## Model Architecture

| 항목 | 논문 기준 |
| --- | --- |
| VLM backbone | PaliGemma VLM pretrained weights |
| backbone scale | 2B language model backbone, `width=2048`, `depth=18`, `mlp_dim=16384`, `num_heads=18`, `num_kv_heads=1`, `head_dim=256` |
| action expert | smaller transformer, 300M parameters |
| action expert width | `width=1024`, `mlp_dim=4096`, 나머지 depth/head 계열은 backbone과 같은 설정 |
| action horizon | `H=50` |
| noisy action input | `a_tau = tau * action + (1 - tau) * noise` |
| timestep conditioning | sinusoidal encoding `phi(tau)` -> MLP -> adaptive RMSNorm으로 action expert 각 layer에 주입 |
| interaction | VLM/backbone token과 action expert token은 self-attention에서 상호작용하지만, 정보 흐름은 VLM -> action expert 방향으로 제한 |

state representation은 세 가지를 실험한다.

| state 방식 | 설명 | 장단점 |
| --- | --- | --- |
| text state | discretized state bin을 숫자 text로 입력 | VLM pretraining 분포와 가장 가깝지만 token 수가 많다. |
| special token state | bin을 special token으로 입력 | token 수가 적지만 새 embedding이 필요하다. |
| continuous state | real-valued `q`를 affine projection으로 embedding | compact하지만 projection이 새로 학습된다. |

## Training Recipe

| 단계/요소 | 내용 |
| --- | --- |
| action tokenization | FAST tokenizer로 continuous action chunk를 discrete token sequence로 압축 |
| continuous action | flow matching action expert를 동시에 학습 |
| co-training | VLM data, robot action data, language+action robot task를 함께 학습 |
| masking | FAST action token과 continuous action token은 서로 attend하지 않음 |
| gradient insulation | action expert에서 backbone key/value로 가는 gradient stop |
| timestep sampling | π0를 따라 low timestep을 강조하는 `Beta((s - tau) / s; alpha=1.5, beta=1), s=0.999` 계열 분포 사용 |
| inference | FAST token path는 버리고, action expert가 continuous action chunk를 생성 |
| compute overhead | continuous+discrete output을 같이 학습하므로 training compute는 약 20% 증가 |

compute overhead가 20% 늘어도, 논문은 convergence speed가 훨씬 빨라 wall-clock 기준으로 π0 같은 pure diffusion/flow VLA보다 빠르게 학습된다고 해석한다.

## Experiments and Results

### Items in Drawer

<PaperFigure
  src="/pi/knowledge-insulation-paper/fig04-items-in-drawer-baselines.png"
  alt="Figure 4 items in drawer baseline comparison"
  caption="Fig. 4. held-out kitchen drawer task에서 ours가 task completion과 language following 모두 최고다."
/>

items in drawer는 object를 고르고 drawer를 열고 넣고 닫아야 하는 held-out environment task다. π0와 joint-training은 language following이 약하고, π0-FAST는 language following은 높지만 task completion이 낮다. HybridVLA와 frozen backbone도 viable하지 않게 나온다. 이 figure가 KI 논문의 central evidence다. 단순히 continuous action expert가 있으면 빠른 것이 아니라, action expert gradient를 어떻게 다루는지가 language following을 좌우한다.

### Table Bussing Specialist

<PaperFigure
  src="/pi/knowledge-insulation-paper/fig05-table-bussing-architectures.png"

table bussing은 12개 object와 두 receptacle이 있는 task다. policy는 language command에 맞는 object를 골라 올바른 receptacle에 넣어야 한다. 논문은 π0-FAST가 language following과 performance는 좋지만 task 완료 시간이 ours보다 약 2배 길다고 설명한다. OpenVLA-OFT는 language following과 inference time은 괜찮지만 overall performance가 낮다.

## 한계와 주의

| 한계 | 의미 |
| --- | --- |
| training compute 증가 | discrete+continuous output을 같이 학습해 약 20% compute overhead가 있다. |
| language following은 완벽하지 않음 | 논문도 data correlation 때문에 language instruction을 가끔 무시할 수 있다고 적는다. |
| full recipe 공개 범위 제한 | OpenPI는 π0.5 flow matching head와 public fine-tuning path를 제공하지만, KI paper의 전체 co-training recipe 공개와는 다르다. |
| graph 수치 판독 | 여러 real-world result는 figure bar plot 중심이므로, 명시 수치가 아닌 값은 대략적 경향으로만 해석해야 한다. |
| task score 해석 | real-world score는 task progress rubric이다. 완전 성공률과 동일시하면 안 된다. |

## Citation Trail

### 직접 비교/상대화 논문

| 논문/모델 | 현재 논문에서의 연결 |
| --- | --- |
| [π0](./pi0-vla-flow-model/) | continuous action expert/flow VLA baseline. fast inference는 가능하지만 naive training은 language following과 convergence가 약하다. |
| [π0-FAST](./pi0-fast-action-tokenization/) | FAST token representation. train fast/language following 장점은 있지만 autoregressive inference가 느리다. |
| [π0.5](./pi05-open-world-generalization/) | π0.5의 two-stage idea를 formalize하고 single-stage KI recipe로 확장한다. |
| [OpenVLA](../openvla-open-source-vla/) / OpenVLA-OFT | autoregressive VLA와 parallel decoding 계열 비교 축. language following은 가능하지만 일부 real-world task performance가 낮다. |
| HybridVLA | autoregressive/diffusion hybrid baseline. KI와 비슷해 보이지만 action representation 사이 attention/masking 차이가 중요하다. |
| Transfusion | 한 transformer 안에서 token prediction과 continuous denoising을 함께 하는 multimodal generation 기반 비교 축. |
| [GROOT N1](../groot/n1-open-foundation-model/) | 논문은 robotics-specific modules가 포함된 fast inference VLA 예로 GROOT를 언급한다. KI 관점에서는 새 robot-specific module gradient가 VLM knowledge를 어떻게 건드리는지가 공통 질문이다. |

### 기반 방법론

| 기반 | KI에서의 역할 |
| --- | --- |
| PaliGemma | pretrained VLM backbone. KI가 보호하려는 semantic knowledge의 출처다. |
| Flow Matching / Rectified Flow | continuous action expert 학습 objective의 기반이다. |
| Action Chunking | `H=50` action chunk를 예측하는 control interface의 기반이다. |
| FAST / DCT action tokenization | backbone을 robot control에 맞게 빠르게 adaptation하는 discrete action learning signal이다. |
| Mixture-of-Transformers / modality-specific experts | backbone/action expert를 다른 weight set으로 처리하고 attention으로 상호작용시키는 설계 배경이다. |
| DROID / LIBERO / Open X-Embodiment | public benchmark와 broad robot data mixture의 기반이다. |
| RT-2 / PaLM-E | web-scale VLM knowledge를 robot action으로 transfer한다는 VLA 흐름의 초기 기반이다. |

## 스터디에서 기억할 점

1. KI는 "backbone freeze"가 아니라 "action expert gradient만 backbone에 못 가게 하고, backbone은 FAST/VLM objective로 계속 학습"하는 방법이다.
2. FAST token은 inference를 위한 것이 아니라 representation learning signal로 쓰인다. 실행은 continuous action expert가 담당한다.
3. VLM data co-training은 language following regularization을 넘어서 OOD object generalization에 중요하다.
4. π0-FAST는 train fast/language following, π0는 run fast/continuous precision, KI는 둘의 장점을 single-stage recipe로 결합하려는 시도다.
5. OpenPI의 public π0.5 support는 KI로 학습된 checkpoint와 flow runtime/fine-tuning path 중심이다. 논문 recipe 전체 공개로 오해하지 말아야 한다.
