## 한 줄 요약

RECAP은 imitation으로 얻은 VLA를 그대로 복사 학습하지 않고, robot이 실제 rollout에서 만든 성공/실패 경험과 expert correction을 distributional value function으로 평가한 뒤, binarized advantage indicator를 policy input에 넣어 π0.6을 더 빠르고 안정적인 π*0.6으로 개선하는 iterated offline RL recipe다.

## 문제의식

VLA는 supervised imitation만으로도 task를 가끔 성공할 수 있다. 하지만 robot control에서는 작은 mistake가 environment state를 바꾸고, 그 state가 training data distribution 밖으로 나가면 다음 mistake가 더 커지는 compounding error가 생긴다. RECAP의 출발점은 이 문제다.

| imitation-only 병목 | 왜 문제가 되나 | RECAP의 대응 |
| --- | --- | --- |
| demonstration state만 배움 | 실제 policy가 만든 mistake state에서 recovery supervision이 부족하다. | 현재 policy를 실행하고 expert가 intervene/correct한다. |
| 실패 rollout을 그대로 imitation하면 실패를 복사 | bad experience에도 학습 신호가 필요하지만, "그대로 따라하기"는 위험하다. | value function으로 action/trajectory의 advantage를 판단한다. |
| long-horizon credit assignment | espresso 실패가 마지막 serving이 아니라 초기 portafilter grasp에서 시작될 수 있다. | value change와 remaining steps-to-success 예측을 advantage로 쓴다. |
| large VLA에 PPO/REINFORCE 적용이 어려움 | flow matching action expert는 tractable log-likelihood가 단순하지 않다. | advantage-conditioned supervised policy extraction으로 RL signal을 주입한다. |

## Method Overview

<PaperFigure
  src="/pi/pistar06-paper/fig01-recap-overview.png"
  alt="Figure 1 RECAP overview"
  caption="Fig. 1. RECAP은 pre-trained VLA와 value function을 두고, task rollout/correction data로 value estimate를 갱신한 다음 advantage-conditioned VLA를 다시 학습한다."
/>

RECAP은 세 개의 subroutine을 반복한다.

1. **Data collection**: 현재 policy를 task에 실행하고, episode outcome label을 수집한다. 필요하면 expert teleoperator가 intervene해서 correction trajectory를 제공한다.
2. **Value function training**: 지금까지 모은 demonstration, autonomous rollout, correction data를 사용해 multi-task distributional value function을 학습한다.
3. **Advantage-conditioned policy training**: value function이 계산한 advantage를 binarized improvement indicator `I_t`로 바꾸고, 이 indicator를 VLA prefix/input에 넣어 policy를 supervised 방식으로 학습한다.

여기서 중요한 점은 bad rollout을 버리지 않는다는 것이다. RECAP은 좋은 action과 나쁜 action을 모두 학습 데이터로 유지하되, "이 action은 positive advantage인가?"라는 조건을 추가해 policy가 high-advantage behavior를 뽑도록 만든다.

## Value Function

<PaperFigure
  src="/pi/pistar06-paper/fig04-value-function-visualization.png"
  alt="Figure 4 value function visualization"
  caption="Fig. 4. value function은 remaining steps-to-success를 normalized value로 예측한다. 빨간 구간은 value drop, 초록 구간은 value increase로 표시된다."
/>

RECAP은 value function을 task success/failure label에서 직접 학습한다. 각 episode의 terminal outcome을 보고 reward를 다음처럼 정의한다.

```text
r_t =
  0        if t = T and success
 -C_fail   if t = T and failure
 -1        otherwise
```

따라서 successful episode에서는 value가 "성공까지 남은 step 수의 음수"가 되고, failed episode는 큰 negative value를 갖는다. 실제 학습에서는 task 길이가 다르므로 value를 task별 최대 episode length 기준으로 normalize해 `(-1, 0)` 범위로 둔다.

value function은 observation `o_t`와 language command `l`을 입력으로 받아 `B=201`개의 discretized value bins에 대한 distribution `p_phi(V | o_t, l)`을 예측한다. empirical return `R_t(tau)`를 binning한 뒤 cross entropy로 학습한다.

```text
min_phi E_{tau in D} [
  sum_{o_t in tau} H(R_t^B(tau), p_phi(V | o_t, l))
]
```

이 value function은 π*0.6 VLA와 같은 architecture design을 쓰지만, VLM backbone은 더 작은 `670M` parameter Gemma 3 initialized model을 사용한다. overfitting을 줄이기 위해 small mixture of multimodal web data에도 co-train한다.

## Advantage Conditioning

<PaperFigure
  src="/pi/pistar06-paper/fig03-vla-value-interaction.png"
  alt="Figure 3 interaction between VLA and value function"
  caption="Fig. 3. π*0.6 VLA는 binarized advantage indicator를 action prediction 전에 입력받는다. 이 indicator는 별도 value function이 계산한다."
/>

value function이 있으면 action의 advantage를 계산할 수 있다. post-training에서는 `N=50` lookahead를 쓴다.

```text
A_pi(o_t, a_t) =
  sum_{t'=t}^{t+N-1} r_{t'} + V_pi(o_{t+N}) - V_pi(o_t)
```

pretraining에서는 on-the-fly cost를 줄이기 위해 `N=T` 형태의 full-episode return estimate를 사용한다. advantage를 그대로 regression target으로 쓰지 않고, task-specific threshold `epsilon_l`로 binarize한다.

```text
I_t = 1(A_pi_ref(o_t, a_t, l) > epsilon_l)
```

policy training에서는 일반 action likelihood와 improvement-conditioned action likelihood를 함께 학습한다.

```text
min_theta E_{D_pi_ref} [
  -log pi_theta(a_t | o_t, l)
  -alpha log pi_theta(a_t | I_t, o_t, l)
]
```

π*0.6은 flow matching continuous action도 함께 갖기 때문에 실제 objective는 discrete action likelihood와 flow matching loss의 조합으로 해석한다. 논문은 continuous part의 exact log-likelihood가 직접 계산되지 않으므로, flow/diffusion likelihood lower-bound 관점으로 설명한다.

## π0.6 and π*0.6 Model Architecture

<PaperFigure
  src="/pi/pistar06-paper/model-card-fig01-pi06-architecture.png"
  alt="π0.6 model architecture from model card"
  caption="π0.6 model card Fig. 1. π0.6은 language subtask, web/VLM data, metadata, image inputs, discrete FAST action tokens, continuous action expert를 함께 다룬다."
/>

기존 초안에는 π0.6을 "5B-parameter VLM backbone"처럼 적었지만, model card 기준으로 더 정확히 쓰면 다음과 같다.

| 구성 | 공식 model card / PDF 기준 |
| --- | --- |
| VLM backbone | Gemma 3 `4B` initialized language backbone + SigLIP `400M` vision encoder |
| action expert | 약 `860M` parameters |
| image input | 최대 4개 image, 각 `448 x 448`; base camera, up to two wrist cameras, optional backward camera |
| attention | image token은 bidirectional, text token은 causal, action expert token은 bidirectional |
| training recipe | Knowledge Insulation. backbone은 FAST action token과 co-training examples를 예측하고, action expert는 continuous action을 flow matching으로 예측한다. action expert gradient는 main VLM backbone으로 흐르지 않는다. |
| output | textual next subtask `l_hat`, FAST-discretized action tokens, continuous action chunk |
| action interface | joint angles + gripper commands at 50 Hz |
| inference speed | model card 기준 3-camera input에서 H100 single GPU action chunk generation `63 ms` |

π*0.6은 이 π0.6에 advantage indicator를 추가한 RL-ready variant다. PDF는 advantage indicator가 `l_hat` 이후, discretized/continuous action 이전에 들어간다고 설명한다. 즉 high-level subtask prediction 자체를 바꾸기보다는, 이후 action prediction이 "positive advantage behavior"를 따르도록 조건화한다.

## Training Recipe

<PaperFigure
  src="/pi/pistar06-paper/algorithm01-recap.png"
  alt="Algorithm 1 RECAP"
  caption="Algorithm 1. RECAP은 value/policy pretraining 뒤 task demo로 초기화하고, rollout collection -> value update -> policy update를 반복한다."
/>

Algorithm 1을 단계별로 풀면 다음과 같다.

| 단계 | 내용 |
| --- | --- |
| pretrain value | multi-task demonstration dataset `D_demo`로 `V_pre`를 학습한다. |
| pretrain policy | `D_demo`, `V_pre`, advantage conditioning objective로 `pi_pre`를 학습한다. |
| task initialization | target task `l`의 demonstrations로 `D_l`을 초기화한다. |
| task value SFT | `V_pre`에서 시작해 `D_l`로 task value `V_l^0`를 학습한다. |
| task policy SFT | `pi_pre`에서 시작해 `D_l`, `V_l^0`로 task policy `pi_l^0`를 학습한다. demonstration 단계에서는 `I_t=True`로 둔다. |
| iteration | current policy `pi_l^{k-1}`로 rollout/correction data를 모아 `D_l`에 추가한다. |
| value update | 항상 `V_pre` checkpoint에서 시작해 누적 `D_l`로 `V_l^k`를 fine-tune한다. |
| policy update | 항상 `pi_pre` checkpoint에서 시작해 누적 `D_l`, `V_l^k`로 `pi_l^k`를 fine-tune한다. |

마지막 두 줄이 중요하다. 논문은 value/policy를 이전 iteration checkpoint에서 계속 이어 fine-tune하지 않고, pre-trained checkpoint에서 다시 fine-tune하는 방식을 사용한다. 이유는 multiple iterations에서 drift를 줄이기 위해서다.

Advantage threshold 설정도 공개되어 있다.

| 설정 | 값 |
| --- | --- |
| pretraining threshold | task별로 demonstration data의 약 `30%`가 positive advantage가 되도록 설정 |
| fine-tuning threshold | 일반적으로 evaluation rollout의 약 `40%`가 positive advantage가 되도록 설정 |
| T-shirt/shorts laundry | high success but slow policy setting이므로 약 `10%`만 positive advantage가 되도록 threshold를 높임 |
| advantage conditioning dropout | training 중 `30%` 확률로 advantage indicator conditioning을 drop |
| test-time CFG | 필요하면 `beta > 1`로 sharpening. 논문은 aggressive motion을 피하기 위해 보통 `beta in [1.5, 2.5]` 같은 moderate setting을 언급한다. |

비교 baseline은 다음과 같다.

| baseline | 의미 |
| --- | --- |
| pre-trained π0.5 | RL/RECAP 없음 |
| pre-trained π0.6 | supervised pretraining, advantage indicator 없음 |
| RL pre-trained π*0.6 | value function과 advantage indicator를 포함한 offline RL pretrain |
| π*0.6 offline RL + SFT | target task demonstrations로 fine-tune. demonstration은 `I_t=True`로 고정 |

결과의 핵심은 다음이다.

| 관찰 | 해석 |
| --- | --- |
| final π*0.6가 모든 task에서 throughput 개선 | RECAP은 단순 success rate뿐 아니라 실제 task/hour 기준 효율을 높인다. |

| failure rate가 약 2배 감소 | success rate figure 기준 difficult task에서 실패율이 크게 줄어든다. |
| T-shirt/shorts laundry는 SFT 이후 success가 이미 높음 | final model은 success보다 speed/throughput을 더 끌어올린다. |
| box assembly는 subtask별 success가 균일하게 높아짐 | retrieve/build/label/place stage 모두에서 final π*0.6가 가장 안정적이다. |

논문은 final π*0.6가 espresso/office use나 factory box assembly 같은 practical setting에 가까운 robustness에 도달한다고 주장한다. article은 qualitative claim으로 espresso drinks를 13시간 straight로 만들고, 새로운 집에서 novel laundry items를 2시간 이상 접고, real packaging factory box를 조립하는 video evidence를 제시한다.

## Iteration, Policy Extraction, Failure Removal

<PaperFigure
  src="/pi/pistar06-paper/fig09-12-iterations-ablation.png"
  alt="Figures 9 to 12 RECAP iterations, policy extraction, failure mode removal"
  caption="Fig. 9-12. iteration 수, policy extraction 방식, targeted failure removal을 나누어 RECAP 설계 요소를 검증한다."

세 가지 ablation을 분리해서 읽어야 한다.

| ablation | 설정 | 결과 |
| --- | --- | --- |
| multiple iterations | T-shirt/shorts folding과 box assembly에서 `i=1`, `i=2` 비교 | laundry throughput은 약 50% 개선, box는 두 번째 iteration 후 throughput 2배 개선. |
| policy extraction comparison | 같은 on-robot data로 RECAP, AWR, PPO 비교 | AWR/PPO도 reasonable result는 얻지만 RECAP throughput보다 낮다. PPO는 off-policy setting 안정화를 위해 작은 trust-region `eta=0.01`이 필요했다. |
| failure mode removal | collar-up strict laundry criterion, 추가 demo/intervention 없이 RL만으로 개선 | 두 iteration, iteration당 600 trajectories 후 success `97%`와 높은 speed를 달성. |

이 ablation은 RECAP이 단순히 "더 많은 data를 넣어서 좋아진다"가 아니라, value-conditioned policy extraction이 어떤 learning signal을 제공하는지 보여준다.

## 한계와 주의

| 한계 | 의미 |
| --- | --- |
| 완전 autonomous system은 아님 | 논문도 expert correction과 human-provided success annotation이 들어간다고 명시한다. |
| reward design과 annotation 필요 | sparse success label을 task마다 정의해야 하고, success criteria가 결과 해석을 좌우한다. |
| robot operation cost | espresso/box/laundry처럼 긴 task에서는 reset, monitoring, correction, failure handling cost가 크다. |
| public reproducibility 제한 | value function, rollout infrastructure, π*0.6 checkpoint/code가 공개되어 있지 않다. |
| safety/runtime constraints | on-robot RL은 bad policy rollout을 포함하므로 hardware safety, intervention, timeout criterion이 필수다. |
| task-specific specialist 중심 | final results는 task-specific π*0.6 specialists다. 단일 generalist로의 distillation은 π0.7 흐름에서 다룬다. |

## Study Takeaways

1. RECAP의 핵심은 robot이 만든 bad experience를 버리거나 복사하지 않고, value/advantage annotation이 붙은 supervised data로 바꾸는 것이다.
2. value function은 reward predictor라기보다 "remaining steps-to-success" critic으로 쓰인다.
3. π*0.6은 π0.6에 advantage conditioning을 추가한 RL-ready VLA variant다.
4. correction은 모든 문제를 해결하는 optimal supervision이 아니라, catastrophic mistake recovery와 exploration bottleneck 해결에 가깝다.
5. on-robot experience를 넣었을 때 diverse laundry와 espresso 같은 어려운 task에서 throughput이 2배 이상 오른다.
6. RECAP은 PI family에서 "runtime correction" 이후 "deployment experience를 training signal로 환류"하는 단계다.
