## 한 줄 요약

HI Robot은 π0를 reactive low-level “System 1”로 두고, 같은 VLM backbone 계열의 high-level “System 2”가 scene, prompt, 실시간 feedback을 해석해 intermediate language command를 속삭이듯 생성하는 hierarchical VLA system이다.

## 문제의식

실제 사용자는 robot에게 완성된 low-level command를 주지 않는다. “테이블을 정리하되 저건 버리지 마”, “샌드위치를 만들어줘”, “장 본 물건을 정리해줘”처럼 instruction은 multi-stage이고, 중간에 correction이 들어오며, object reference가 현재 scene에 ground되어야 한다. Flat VLA가 이 모든 것을 바로 action chunk로 내면 prompt constraint를 잊거나, 이미 잘 학습된 autopilot behavior로 돌아갈 위험이 있다.

Figure 1의 예시는 이 문제를 세 가지 failure mode로 분해한다. 첫째, 사용자는 처음부터 여러 constraint가 섞인 instruction을 준다. 둘째, robot이 움직이는 중간에 “그건 trash가 아니야” 같은 correction이 들어온다. 셋째, 사용자가 기존 목표를 바꾸거나 추가 조건을 붙인다. HI Robot의 목표는 이 모든 것을 low-level action으로 직접 압축하지 않고, 먼저 scene-grounded language command로 분해하는 것이다.

## 시스템 설정

| 항목 | 논문/appendix 기준 내용 |
| --- | --- |
| low-level robot control | π0 VLA가 image, robot state, low-level language command를 받아 continuous action chunk를 생성 |
| high-level inference | PaliGemma-3B 기반 VLM이 image observations와 user prompt/interjection을 받아 low-level command와 optional robot speech를 생성 |
| camera input | task/robot별 base camera와 wrist-mounted cameras. UR5e는 over-the-shoulder + wrist, ARX 계열은 base + two wrist cameras |
| user input | text command 또는 speech input. speech는 local Whisper large-v2로 transcription |
| robot output | low-level language command는 π0로 전달, verbal utterance는 TTS로 사용자에게 재생 후 low-level command에서 제거 |
| high-level trigger | 1초가 지났거나 user interaction이 새로 들어오면 high-level command를 recompute |
| hardware | appendix 기준 1-2x RTX 4090으로 real-time inference. high-level training은 약 2 hours on 8x H100 |
| robot embodiments | UR5e single arm, bimanual ARX, Mobile ALOHA 기반 mobile ARX |

Appendix의 robot detail은 embodiment별 action space를 분리한다. UR5e는 6-DoF arm + parallel jaw gripper와 7-dimensional configuration/action space를 사용한다. Bimanual ARX는 two 6-DoF ARX arms, three cameras, 14-dimensional configuration/action space다. Mobile ARX는 Mobile ALOHA platform 위에 two 6-DoF ARX arms를 얹고 nonholonomic base action 2 dimensions가 추가되어 14-dimensional configuration space와 16-dimensional action space를 쓴다.

## Method: System 1 / System 2

<PaperFigure
  src="/pi/hirobot-paper/fig02-hierarchical-vla-overview.png"
  alt="Figure 2: HI Robot hierarchical VLA overview"
  caption="논문 Figure 2. High-level policy는 user prompt/interjection과 multi-view image를 받아 robot verbal response와 low-level language command를 만들고, low-level π0 policy가 이를 motor action으로 실행한다."
/>

| 구성 | 공식 글 기준 역할 |
| --- | --- |
| low-level π0 | images, robot state, language subcommand를 받아 motor action을 출력하는 reactive System 1 |
| high-level VLM policy | complex prompt와 scene을 해석하고 next intermediate step을 language로 생성하는 System 2 |
| shared backbone idea | high-level policy도 π0와 같은 VLM backbone을 사용한다고 설명된다. |
| whisper interface | “pick up one piece of whole wheat bread”처럼 low-level π0가 실행 가능한 bite-sized command를 생성한다. |
| feedback handling | “that’s not trash” 같은 real-time contextual feedback을 scene object와 현재 manipulation target에 ground한다. |

공식 글의 핵심은 “hierarchy가 planning을 멋있게 보이게 한다”가 아니다. Web-scale VLM pretraining은 이미지와 text context를 보고 textual answer를 생성하는 데 강하다. High-level policy가 먼저 language step을 생성하게 하면, 이 semantic reasoning 능력을 motor action으로 직접 압축하지 않고 사용할 수 있다.

논문 수식 관점에서는 high-level policy가 image history와 user language를 조건으로 robot speech와 skill label을 생성하고, low-level policy는 image/state/subcommand를 조건으로 action chunk를 생성한다. 두 policy 모두 VLM family를 쓰지만 역할은 다르다. high-level은 “무엇을 해야 하는가”와 “사용자에게 어떻게 응답할 것인가”를 담당하고, low-level은 π0 계열 flow-matching action policy로 실제 motor execution을 담당한다.

### Inference Contract

| 단계 | 입력 | 출력 | 주의 |
| --- | --- | --- | --- |
| user instruction | open-ended prompt, later interjection | current task context | prompt는 중간에 바뀔 수 있고, “that” 같은 지시어는 image와 현재 action context에 ground되어야 한다. |
| high-level VLM | images `I1..In`, prompt/interjection `l_t` | intermediate command `l_hat_t`, optional utterance `u_t` | utterance는 사용자에게 말하고, low-level command로 넘기기 전 제거한다. |
| low-level π0 | images, robot state `q_t`, `l_hat_t` | action chunk `A_t` | low-level은 atomic command를 수행하는 System 1로 동작한다. |
| recompute loop | 1 second elapsed 또는 new user feedback | updated `l_hat_t` | quick physical control은 low-level chunking이 담당하고, semantic correction은 high-level이 낮은 빈도로 담당한다. |

이 분리는 두 가지 이점을 준다. 첫째, high-level command가 natural language라서 robot이 지금 무엇을 하려는지 사람이 inspect할 수 있다. 둘째, web-scale VLM pretraining이 잘하는 “image + text context에서 textual answer 생성” 능력을 action head로 바로 눌러 넣지 않고, low-level π0가 이해할 수 있는 bite-sized command로 변환한다.

## Training: Synthetic Interaction Data

<PaperFigure
  src="/pi/hirobot-paper/fig03-data-generation-high-level-policy.png"
  alt="Figure 3: HI Robot data generation for the high-level policy"
  caption="논문 Figure 3. Teleoperated robot demonstrations를 short skill로 segment하고 human skill annotation을 붙인 뒤, data-generator VLM이 synthetic user prompt와 robot speech를 만들어 high-level policy를 학습한다."
/>

HI Robot은 단순 atomic command demonstration만으로는 rich multi-step interaction이 부족하다고 본다. 그래서 robot observations와 human-labeled skills를 hypothetical prompts 및 human interjections와 pairing하는 synthetic labeling recipe를 사용한다.

| synthetic label | 필요한 이유 |
| --- | --- |
| complex prompt | 여러 constraint를 한 번에 받은 상황을 학습 |
| human interjection | 중간 correction, 금지, 우선순위 변경을 학습 |
| skill label | high-level step이 low-level π0 command로 이어지게 함 |
| scene-grounded response | “that”, “not trash” 같은 지시어를 현재 image context에 연결 |

이 data recipe는 HI Robot이 user와 대화하는 chatbot이 되기 위한 것이 아니라, open-ended instruction을 physical action sequence로 안정적으로 분해하기 위한 supervision이다.

### Dataset Recipe

| dataset symbol | 생성 방식 | 포함 정보 | 학습에 쓰이는 곳 |
| --- | --- | --- | --- |
| `Ddemo` | teleoperation으로 full robot demonstration 수집 | coarse overall goal, image/state/action trajectory | low-level policy training의 raw demonstration source |
| `Dlabeled` | full episode를 1-3 second short skills로 segmentation하고 human skill label 부착 | `l_hat_t`, images, short skill segments, heuristic movement primitives | high-level supervised pairs와 low-level skill/action 학습 |
| `Dsyn` | data-generator VLM `pgen`이 visual context, prior skill labels, current skill label, task prompt template을 조건으로 synthetic interaction 생성 | synthetic user prompt/interjection `l_t`, robot utterance `u_t`, target skill label | high-level VLM의 next-token training |

Synthetic data appendix는 scenario와 response를 구조화해서 생성한다고 설명한다. scenario type에는 negative task, situated correction, specific constraint가 포함되고, response type에는 simple confirmation, clarification, error handling이 포함된다. 또한 `pgen`은 현재 skill label만 보지 않고 episode 안의 prior skill labels `l_hat_0..l_hat_{t-1}`도 조건으로 받아, 이미 lettuce/tomato를 올린 sandwich에 추가 ingredient를 요청하는 식의 temporal consistency를 만들도록 설계된다.

### Training Objective and Hyperparameters

| 구성 | 학습 objective / recipe |
| --- | --- |
| high-level policy | `Dsyn ∪ Dlabeled`에서 image-language tuple을 next-token cross entropy로 학습 |
| low-level policy | `Dlabeled ∪ Ddemo`에서 π0 flow-matching objective로 action chunk 학습 |
| backbone | low-level/high-level 모두 PaliGemma-3B에서 시작. high-level은 full model fine-tuning |
| optimizer | AdamW, beta1 0.9, beta2 0.95, no weight decay |
| stabilization | gradient norm clip 1, EMA decay 0.999 |
| learning rate | first 1,000 steps warmup 후 constant `1e-5` |
| batch size | 512 |
| high-level training cost | 약 2 hours on 8x H100 |

## Model Architecture and Runtime

| 모듈 | 구조 | 역할 |
| --- | --- | --- |
| High-Level VLM policy | PaliGemma-3B VLM fine-tuned for command/utterance generation | images와 open-ended prompt를 prefix로 받고, low-level command와 robot speech를 suffix로 생성 |
| Low-Level VLA policy | π0: PaliGemma-3B + flow-matching action expert | low-level command를 continuous action chunk로 변환 |
| Data-generator VLM | large pretrained VLM `pgen` | visual context와 skill labels를 보고 synthetic user prompt/interjection과 robot utterance 생성 |
| Speech stack | Whisper large-v2 STT + Cartesia API TTS | spoken user interaction과 robot verbal response 연결 |

Appendix latency는 real-time feasibility를 판단하는 좋은 기준이다. Low-level on-board inference는 image encoding 14 ms, observation processing 32 ms, action prediction x10 27 ms로 총 73 ms이며, off-board + WiFi는 총 86 ms다. High-level single decoding step은 RTX 4090에서 47 ms prefill + 13.2 ms decode, H100에서 17.3 ms prefill + 5.7 ms decode로 보고된다. 논문은 이 조합이 약 10 Hz control rate에 feasible하며, action chunking을 쓰면 robot을 50 Hz로 제어할 수 있다고 설명한다.

## Experiments and Results

<PaperFigure
  src="/pi/hirobot-paper/fig04-evaluation-task-domains.png"

공식 pi.website 글은 table bussing, sandwich making, grocery shopping을 real-world evaluation task로 든다. 비교 대상은 GPT-4o high-level baseline과 flat VLA policy다.

### Task Domains

| domain | robot/task difficulty | prompt/feedback difficulty |
| --- | --- | --- |
| Table Bussing | dishes/utensils/trash를 bin/trash로 분류하고 edge grasp, singulation, plate tilting 같은 manipulation이 필요 | “only trash but not dishes”, “bus all yellowish things”, “that’s not trash”, “leave it alone” |
| Sandwich Making | bread와 최대 six ingredients를 다루며 deformable/delicate item을 정확히 집고 쌓아야 함 | ingredient set, dietary preference, allergy, “that’s all, no more” 같은 live correction |
| Grocery Shopping | shelf에서 여러 item을 골라 basket에 넣고 table로 옮기는 mobile bimanual manipulation | “movie night”, “something sweet”, “something to drink”, “I also want KitKat”처럼 implicit/variable object request |

### Baselines and Metrics

| baseline / metric | 정의 |
| --- | --- |
| Expert human high-level | oracle high-level baseline. 사람이 low-level π0가 성공하기 쉬운 language command를 직접 입력한다. |
| GPT-4o high-level | HI Robot과 같은 hierarchical decomposition을 쓰되 high-level을 GPT-4o API로 대체한다. prompt에는 robot setup과 가능한 skill labels를 포함한다. |
| Flat VLA | high-level 없이 π0 low-level policy를 그대로 사용한다. |
| Flat VLA with synthetic data | hierarchy 없이 low-level π0가 complex prompt synthetic data까지 직접 학습한다. hierarchy 효과를 분리해 보기 위한 ablation이다. |
| Instruction Accuracy | blind human evaluator가 high-level predicted instruction이 user intent와 current observation에 맞는지 판단한다. flat baseline은 행동 intent를 해석해 점수화한다. |
| Task Progress | correct location/configuration에 성공적으로 놓인 object proportion으로 long-horizon completion을 granular하게 본다. |

각 evaluation은 task/method마다 20 trials로 구성된다. 이 점이 중요하다. Figure 5의 success는 단일 rollout demo가 아니라 blind human evaluator가 instruction accuracy와 task progress를 따로 점수화한 결과다.

| Ablation | 무엇을 검증하나 | 해석 |
| --- | --- | --- |
| without synthetic data | high-level policy가 human-labeled skill만으로 open-ended interaction을 처리할 수 있는가 | synthetic user prompt/robot speech가 없으면 clarification, constraint, compositional prompt 대응이 약해진다. |
| flat VLA with synthetic data | hierarchy 없이 하나의 VLA가 user prompt를 바로 action으로 처리할 수 있는가 | 같은 data가 있어도 high-level language reasoning과 low-level action execution을 분리한 구조가 더 안정적이다. |

Figure 7은 average 기준 synthetic data가 빠질 때 Instruction Accuracy gap 46%, Task Progress gap 39%가 생긴다고 표시한다. Figure 8은 같은 synthetic data를 쓰더라도 hierarchical HI Robot이 flat VLA보다 average Instruction Accuracy 19%, Task Progress 34% 높다고 표시한다. 즉 synthetic data는 “언어 다양성”을 만들고, hierarchy는 “중간에 다시 생각하기”를 가능하게 하는 서로 다른 축의 기여다.

## PI Family 안에서의 위치

HI Robot은 `π0.5`의 high-level/low-level inference와 강하게 연결된다. `π0.5`가 open-world home에서 semantic subtask를 “스스로 말하고” low-level flow action을 실행한다면, HI Robot은 그 구조를 human interaction, prompt constraint, correction handling까지 확장한다.

| --- | --- |

| [π0.5](../pi05-open-world-generalization/) | high-level semantic inference + low-level action expert |

Failure case는 high-level과 low-level로 나뉜다. High-level은 memory가 없어 long-context instruction reasoning에 약할 수 있다. Low-level은 proximal object bias 때문에 user constraint를 잠시 무시하거나, dropped object 같은 OOD recovery에서 error accumulation이 생길 수 있다. 논문은 stronger instruction-following model, long-context model, adversarial data generation, failure recovery data/annotation을 완화 방향으로 제시한다.

## 한계와 주의

- 공개 article은 high-level policy의 full architecture/training hyperparameter를 모두 노출하지 않는다.
- HI Robot의 성공은 low-level π0가 subcommand를 충분히 수행할 수 있다는 전제에 의존한다.
- synthetic interaction label은 다양성과 품질이 중요하며, 잘못된 hypothetical prompt는 model behavior를 왜곡할 수 있다.
- ambiguity를 다루는 system이므로 success metric과 instruction-following metric을 분리해 봐야 한다.
- 현재 시스템은 memory가 제한적이므로 long-context instruction이나 많은 interjection이 누적되는 장기 task에서는 약해질 수 있다.
- low-level π0의 object proximity bias, dropped object recovery, OOD recovery 실패가 high-level reasoning만으로 해결되지는 않는다.
