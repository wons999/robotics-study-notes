## 한 줄 요약

OpenPI는 PI 계열 VLA를 논문 아이디어가 아니라 실제 checkpoint, data transform, fine-tuning config, policy wrapper, inference server로 만져볼 수 있게 공개한 software layer다.

## 공개 범위

공식 article은 OpenPI release의 목적을 “π0를 각자 robot/task에 fine-tune해 실험할 수 있게 하는 것”으로 설명한다. 공개 범위는 다음 네 가지다.

| 공개 항목 | 내용 |
| --- | --- |
| base model | `π0 base`, `π0-FAST base`, 이후 repo 기준 `π0.5 base` 계열 checkpoint |
| demo checkpoint | DROID, ALOHA, LIBERO 등 widely available platform용 fine-tuned checkpoint |
| inference example | real-world/simulated robot platform에서 policy를 실행하는 예시 |
| fine-tuning code | custom task/platform data로 base model을 fine-tune하는 code/config |

공식 article은 PI 내부 실험에서 1-20 hours data가 다양한 task fine-tuning에 충분했던 경우가 있다고 말하지만, 바로 caveat를 붙인다. π0는 원래 PI robot용으로 개발되었고, ALOHA/DROID 같은 public platform과 다르기 때문에 모든 robot에서 성공을 보장하지 않는다. 이 문장은 OpenPI를 “plug-and-play product”가 아니라 research release로 읽어야 한다는 근거다.

## Checkpoint Inventory

| checkpoint | 공식 article 기준 의미 | 주의 |
| --- | --- | --- |
| `π0 base` | paper의 표준 pre-trained π0. OXE와 PI의 7 robot platforms로 학습됨. | fine-tuning용이 기본 목적이며, pre-training에 있는 task는 zero-shot으로 될 수 있다. |
| `π0-FAST base` | FAST tokenizer를 쓰는 autoregressive discretized control model. | language following은 somewhat better일 수 있지만 PI 경험상 inference cost가 about 4-5x higher. |
| `π0 DROID`, `π0-FAST DROID` | DROID dataset 기반 Franka/DROID-like platform용 fine-tuned model. | article은 entirely new DROID environments에서 instruction following 가능하다고 설명한다. |
| `π0 ALOHA` | towel folding, food scooping 등 ALOHA task용 demo checkpoint. | robot setup에 민감할 수 있지만 training에 없던 ALOHA station에서도 실행 사례가 있다. |
| `π0 LIBERO` | LIBERO benchmark fine-tuned checkpoint. | simulation benchmark out-of-box evaluation용이다. |

DROID demo는 University of Montreal, University of Washington, KAIST 등 training에 없던 여러 location에서 실행된 예를 article이 든다. ALOHA demo도 training에 없던 새 ALOHA station에서 실행됐다고 설명한다. 다만 “new location에서 예시가 있다”와 “모든 robot/site에서 robust하다”는 다르다.

## 코드 레벨 확인

| 경로 | 역할 |
| --- | --- |
| `src/openpi/models/pi0.py` | π0/π0.5 flow-matching model, prefix KV cache, action sampling |
| `src/openpi/models/pi0_fast.py` | π0-FAST autoregressive token model |
| `src/openpi/models/pi0_config.py` | `Pi0Config`, `Pi0FASTConfig`, `ModelType.PI0/PI05/PI0_FAST` |
| `src/openpi/training/config.py` | 공개 fine-tuning config inventory |
| `src/openpi/transforms.py` | image resize, prompt tokenization, FAST tokenization, state/action padding |
| `src/openpi/policies/policy_config.py` | checkpoint load, norm stats, transform chain, runtime policy construction |
| `scripts/serve_policy.py` / docs | policy server/client style deployment path |

OpenPI를 읽을 때 가장 중요한 점은 paper system과 public repo boundary를 분리하는 것이다. π0/π0-FAST/π0.5의 core recipe와 일부 checkpoint는 공개되어 있지만, PI 내부의 최신 π*0.6, RLT, π0.7, partner deployment stack이 모두 같은 형태로 공개된 것은 아니다.

## PI Family 안에서의 위치

OpenPI는 PI family의 실험 재현 축이다. `π0`는 foundation action model, `π0-FAST`는 token-action route, `π0.5`는 open-world generalization으로 이어지고, OpenPI는 이 모델들을 실제 robot data에 맞춰 fine-tune할 수 있는 entry point가 된다.

| 연결 문서 | OpenPI에서 확인할 것 |
| --- | --- |
| [π0](../pi0-vla-flow-model/) | flow matching action expert가 code에서 어떻게 구현되는지 |
| [π0-FAST](../pi0-fast-action-tokenization/) | FAST tokens가 raw model output이 아니라 transform을 거쳐 action으로 복원되는 구조 |
| [π0.5](../pi05-open-world-generalization/) | `pi05=True` config와 paper의 full hybrid co-training system 사이의 차이 |
| [The Physical Intelligence Layer](../physical-intelligence-layer/) | research code release와 partner/product layer가 어디서 갈라지는지 |

## 한계와 주의

- 공개 checkpoint는 baseline/fine-tuning seed로 봐야 하고, 각 robot의 calibration, camera pose, action convention, dataset schema가 맞지 않으면 실패할 수 있다.
- `π0-FAST base`는 training efficiency 장점과 runtime inference cost caveat를 같이 적어야 한다.
- demo checkpoint는 subsequent fine-tuning용이라기보다 out-of-box demonstration 성격이 강하다고 article이 설명한다.
- 최신 PI research article이 모두 OpenPI checkpoint로 공개된 것은 아니다.
