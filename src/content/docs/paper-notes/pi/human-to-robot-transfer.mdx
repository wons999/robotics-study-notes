## 한 줄 요약

이 논문은 egocentric human video를 별도 hand-to-robot 변환 모듈 없이 또 하나의 embodiment data처럼 `π0.5`에 50:50 co-finetune하면, 충분히 다양한 robot pretraining 이후 human-only scenario의 scene/object/task 정보를 robot policy가 활용할 수 있고 평균 성능이 거의 2배로 좋아진다는 것을 보인다.

## 문제의식

Robot data는 비싸고 느리며 embodiment와 site가 바뀔 때마다 새로 모으기 어렵다. 반대로 human egocentric video는 wearable camera로 쉽게 모을 수 있고 real-world task variation을 많이 담을 수 있지만, robot과 morphology, camera viewpoint, action space가 다르다.

이 논문의 질문은 다음처럼 단순하다.

| 질문 | 의미 |
| --- | --- |
| human video를 robot policy 학습에 직접 넣을 수 있는가? | hand masking, robot-hand generation, humanoid hardware alignment 같은 명시적 transfer mechanism 없이 가능한지 본다. |
| 어떤 조건에서 가능한가? | model 자체보다 pretraining robot data의 scene/task/embodiment diversity가 transferability를 만든다는 가설을 검증한다. |
| human data는 무엇을 주는가? | low-level action만이 아니라 scene rule, object category, task semantic concept을 제공한다. |

핵심은 "human video만으로 robot을 학습한다"가 아니다. 충분히 diverse하게 pretrain된 `π0.5`가 있을 때, human video를 robot data mixture에 넣으면 robot data에 없던 concept을 policy가 흡수할 수 있는지 확인한다.

## System / Data Collection

<PaperFigure
  src="/pi/human-to-robot-paper/fig06-data-collection-apparatus.png"
  alt="Figure 6 human data collection apparatus"
  caption="Fig. 6. human data는 head-mounted camera와 양쪽 wrist camera로 수집한다. wrist camera는 robot wrist view와 sensor gap을 줄이기 위한 설계다."
/>

Human data collection은 robot teleoperation episode와 비슷한 형태로 맞춘다. 사람이 wearable camera를 쓰고 task를 반복 수행하며, 손이 camera FOV 안에 들어오도록 지시받는다.

| 항목 | 논문 기준 |
| --- | --- |
| device | head-worn high-resolution camera, optional left/right wrist cameras |
| synchronization | head camera와 두 wrist camera가 time-synchronized stream을 제공 |
| benchmark human data | bussing 3h, spice 3h, dresser 3h, sort eggs 5h |
| processing | visual SLAM으로 head camera의 6D motion `e_t in R^6`를 world frame 기준으로 복원 |
| hand tracking | 두 손 각각 17개 3D keypoint `h_t^e in R^{3 x 17}`를 head camera frame에서 복원 |
| language annotation | robot teleoperation data와 유사하게 각 arm action을 text-based subtask로 dense annotation |

논문 discussion은 이 recipe가 아직 "vast passive human video"를 바로 쓰는 단계가 아니라는 점도 분명히 한다. 현재 benchmark는 episodic data이며 human data 규모는 10s of hours 수준이다.

## Benchmark Dataset

<PaperFigure
  src="/pi/human-to-robot-paper/fig03-training-mixture-benchmark.png"

Benchmark는 human data에만 있는 concept을 robot이 실행할 수 있는지 보는 네 가지 scenario로 구성된다.

| 축 | task | robot data가 가진 것 | human data가 추가하는 것 | score |
| --- | --- | --- | --- | --- |
| Scene transfer | Spice | 여러 robot kitchen/scene의 spice rack tidying | unseen target kitchen | binary success |
| Scene transfer | Dresser | 다양한 bedroom/dresser tidying | target apartment dresser의 jewelry/hair tie placement rule | binary success |
| Object transfer | Bussing | trash/dinnerware 중심 table bussing | kitchen tools 같은 unseen object set | 9 items 중 correctly bussed items, `[0, 1]` normalize |
| Task transfer | Sort Eggs | egg picking/placing into carton | white/brown eggs를 color별 carton에 sorting하는 semantic rule | correctly sorted eggs + final carton closing, `[0, 1]` normalize |

Appendix A에 따르면 각 experiment는 `20-40`회 evaluation을 수행하고 error bar는 `1 standard error`다.

## Method

<PaperFigure
  src="/pi/human-to-robot-paper/fig04-model-architecture.png"
  alt="Figure 4 model architecture"
  caption="Fig. 4. π0.5 VLA를 그대로 사용하되 human/robot data 모두에서 high-level subtask prediction과 low-level action prediction을 학습한다."
/>

방법론은 "human을 또 하나의 robot embodiment처럼 다룬다"로 요약할 수 있다. 명시적 alignment loss, hand-to-robot rendering, robot morphology conversion을 추가하지 않는다.

| 단계 | 내용 |
| --- | --- |
| base model | `π0.5` VLA |
| human observation | head/wrist camera images |
| human low-level action | 손의 6DoF end-effector trajectory와 head/base motion에서 만든 relative action |
| robot low-level action | robot end-effector relative action chunk |
| high-level target | dense subtask language token |
| fine-tuning mix | human generalization task data와 nearest-neighbor robot task data를 50:50 |
| shorthand | human egocentric data를 넣은 model을 `π0.5 + ego`로 표기 |

### Action Space Alignment

Robot action은 end-effector pose 기준으로 정의한다. 논문은 joint position 대신 end-effector pose를 택한다. human joint position을 robot joint로 대응시키기 어렵기 때문이다.

| action | shape / 구성 |
| --- | --- |
| robot action chunk | `a in R^{H x 16}` |
| robot action 구성 | left arm 6DoF EE trajectory + gripper, right arm 6DoF EE trajectory + gripper, 2D base action |
| human hand action | 양손 palm/middle/ring finger keypoints가 span하는 6DoF "end-effector" pose |
| human base/head action | head camera pose를 chunk 첫 timestep base camera frame으로 project |
| human action dimension | `2 x 6 + 6 = 18` |
| gripper | human hand openness는 직접 추정하지 않고, gripper action은 robot data에서만 학습 |

여기서 alignment는 완벽한 physical equivalence가 아니라 "relative end-effector trajectory"라는 공유 형식이다. 논문의 claim은 이 정도의 rough alignment와 scaled pretraining만으로 transfer가 나타난다는 점이다.

### Training Objectives

π0.5 계열과 마찬가지로 low-level action에는 두 경로가 함께 쓰인다.

```text
low-level action:
  discrete FAST action tokens -> next-token prediction
  continuous action expert    -> flow matching loss

high-level subtask:
  subtask language tokens     -> next-token prediction
```

High-level subtask prediction은 observation과 high-level language command를 보고 short atomic action description을 예측하고, 이 subtask string이 다시 action generation condition으로 들어간다. 즉 human video는 "무엇을 해야 하는가"와 "어떻게 움직이는가"를 모두 supervision한다.

## Representation Analysis

<PaperFigure
  src="/pi/human-to-robot-paper/fig05-tsne-representations.png"
  alt="Figure 5 TSNE representation of human and robot data"
  caption="Fig. 5. pretraining이 없거나 작을 때는 human/robot embedding이 분리된다. full diverse pretraining에서는 두 distribution이 같은 task manifold 위에서 겹친다."
/>

TSNE 분석은 성능 개선의 원인을 설명하기 위한 diagnostic이다. Appendix C 기준 절차는 다음과 같다.

1. co-finetuned VLA에 human/robot observations를 입력한다.
2. VLM backbone final layer의 output embeddings 중 처음 200개를 mean-pool한다. 논문은 이 구간이 대략 task 정보를 담는다고 설명한다.
3. pooled embedding을 TSNE로 2D 시각화한다.

해석은 명확하다. 작은/no pretraining에서는 human data와 robot data가 서로 다른 cluster로 분리된다. pretraining diversity가 커지면 같은 task에 대한 human/robot observations가 더 잘 겹치고, 이 representation overlap이 transfer performance와 같이 증가한다.

## Main Results

<PaperFigure
  src="/pi/human-to-robot-paper/fig07-human-to-robot-finetuning.png"
  alt="Figure 7 human to robot finetuning main result"
  caption="Fig. 7. π0.5+ego는 scene, object, task generalization 모두에서 π0.5보다 높은 score를 보인다."
/>

주요 결과는 human data가 robot data에 없는 concept을 제공했을 때 policy score가 크게 좋아진다는 것이다.

| task | π0.5 | π0.5 + ego | 해석 |
| --- | --- | --- | --- |
| Spice | 32% | 71% | unseen kitchen의 scene-specific rule을 human video에서 흡수 |
| Dresser | 25% | 50% | target dresser의 placement rule을 human data가 제공 |
| Bussing | 53% | 63% | unseen kitchen tools/object categories에 대한 object transfer |
| Sort Eggs | robot-only sorting accuracy 57% | human co-training sorting accuracy 78% | robot data에는 egg picking만 있고 sorting rule은 human data에서만 제공 |

Sort Eggs에서는 평균적으로 `π0.5`보다 4개 더 많은 eggs를 올바르게 배치했다. 이 결과가 중요한 이유는 robot-only model이 이미 egg pick/place skill은 갖고 있지만, color sorting이라는 task semantic은 모른다는 점이다.

## Pretraining Scale

<PaperFigure
  src="/pi/human-to-robot-paper/fig02-per-task-improvement.png"
  alt="Figure 2 per task improvement from human data"
  caption="Fig. 2. human data 추가 이득은 pretraining robot data의 diversity가 커질수록 커진다. 0%/25%에서는 거의 나타나지 않고, 75% 이상에서 뚜렷해진다."
/>

논문은 여러 pretrained initialization을 비교한다.

| initialization | 의미 |
| --- | --- |
| `0%` | base VLM initialization only |
| `25%`, `50%`, `75%`, `100%` | target robot embodiments인 ARX/mobile ARX에서 scene-task combination diversity fraction을 늘린 VLA pretraining |
| `100% + X-emb` | π0.5 full VLA pretraining mixture. diverse non-target robot embodiments 포함 |

각 initialization에서 두 policy를 학습한다. 하나는 nearest robot teleoperation data만 fine-tune하고, 다른 하나는 동일 robot data에 human embodiment data를 추가한다. Fig. 2는 두 policy score 차이를 human-to-robot transfer magnitude로 본다.

핵심 결론은 다음이다.

| 관찰 | 의미 |
| --- | --- |
| 0%/25%에서는 human data co-training gain이 작거나 없다 | human video는 model이 작거나 robot pretraining이 빈약하면 그냥 domain gap으로 남는다. |
| 75%/100%에서 gain이 커진다 | enough robot pretraining이 embodiment-agnostic representation을 만든다. |
| `100% + X-emb`에서 transfer가 더 좋아진다 | target robot만이 아니라 non-target robot embodiments까지 포함한 diversity가 human data 흡수 능력을 키운다. |

## Ablations and Comparisons

<PaperFigure
  src="/pi/human-to-robot-paper/fig09-human-vs-target-robot-data.png"
  alt="Figure 9 human data compared to target robot data"
  caption="Fig. 9. Eggs/Dresser에서는 human data가 target robot data와 비슷한 수준의 이득을 주지만, Bussing에서는 target robot data가 더 강한 upper bound로 남는다."

Human data가 target robot data를 완전히 대체한다는 주장은 아니다. Fig. 9는 다음처럼 읽어야 한다.

| 비교 | 결론 |
| --- | --- |
| Sort Eggs / Dresser | comparable amount의 human data와 target robot data가 비슷한 성능을 만든다. |
| Bussing | target robot data가 human data보다 더 효과적이다. 논문 본문은 Bussing에서 human data와 target robot data 차이를 `25% vs 65%`로 설명한다. |
| 전체 해석 | human data는 cheap/scalable source지만, contact-rich object interaction에서는 target robot data가 여전히 upper bound일 수 있다. |

<PaperFigure
  src="/pi/human-to-robot-paper/fig10-human-vs-cross-embodiment.png"
  alt="Figure 10 human data compared to cross embodiment robot data"
  caption="Fig. 10. Bussing에서 UR5 data와 human data 모두 ARX transfer에 도움을 주지만 target ARX data 수준에는 못 미친다."
/>

Cross-embodiment comparison에서는 Bussing task를 위해 UR5 robot data `400 demonstrations`, `7.45 hours`를 추가로 수집하고 ARX robot으로 transfer한다. human-to-ARX transfer와 UR5-to-ARX transfer 모두 baseline보다 낫지만 target ARX data에는 못 미친다. 논문의 해석은 human data transfer가 cross-embodiment robot transfer와 비슷한 성격을 가진다는 것이다.

<PaperFigure
  src="/pi/human-to-robot-paper/fig11-high-vs-low-level-transfer.png"
  alt="Figure 11 high level vs low level transfer"
  caption="Fig. 11. mobile tasks에서는 high-level subtask prediction과 low-level action prediction 둘 다 human data로 co-train해야 가장 좋다."
/>

High-level vs low-level ablation도 중요하다.

| 설정 | 의미 | 관찰 |
| --- | --- | --- |
| robot-only HL+LL | human data 없음 | baseline |
| robot-only HL + co-trained LL | low-level action만 human data 사용 | command를 받는 motion은 일부 개선되지만 high-level command가 틀릴 수 있다. |
| co-trained HL + robot-only LL | high-level subtask만 human data 사용 | high-level은 맞아도 low-level policy가 command를 제대로 실행하지 못할 수 있다. |
| co-trained HL+LL | 둘 다 human data 사용 | 가장 안정적 |

Bussing/Eggs는 evaluation에서 high-level policy를 쓰지 않으므로 transfer가 low-level action prediction에서 나온다. Spice/Dresser 같은 mobile tasks는 high-level subtask와 low-level action이 모두 중요하다.

<PaperFigure
  src="/pi/human-to-robot-paper/fig12-wrist-camera-ablation.png"
  alt="Figure 12 wrist camera ablation"
  caption="Fig. 12. wrist camera는 Dresser와 Bussing에서 관찰성을 높여 이득을 주고, Spice/Eggs에서는 비슷한 성능을 보인다."
/>

Wrist camera ablation은 data collection design의 practical lesson을 준다. Dresser/Bussing처럼 손-물체 interaction과 occlusion이 중요한 task는 wrist camera가 도움이 된다. 반면 Spice/Eggs는 head camera만으로도 sufficient observability가 확보되어 추가 이득이 작다. 논문은 이 결과를 근거로 future human data collection에서는 wrist camera를 포함하는 방향을 선호한다.

## 한계와 주의

| 한계 | 의미 |
| --- | --- |
| dataset scale 제한 | benchmark human data는 14h task data 중심이고, discussion도 10s of hours 수준이라고 밝힌다. |

| target robot data가 여전히 강함 | Bussing에서는 human data보다 target robot data가 더 효과적이다. |
| action alignment는 rough | human gripper action은 직접 추정하지 않고 robot data에 의존한다. |
| passive web-scale human video는 아직 아님 | episodic wearable-camera data를 사용하며, everyday passive data pretraining은 future work에 가깝다. |
| TSNE는 diagnostic | representation overlap은 transfer와 상관된 evidence지만 causal proof는 아니다. |

## Study Takeaways

1. human video transfer는 "human data만 넣으면 된다"가 아니라 "diverse robot pretraining 이후에 나타나는 capability"로 보는 게 정확하다.
2. 이 논문은 explicit alignment algorithm보다 scaled cross-embodiment representation을 신뢰하는 PI식 recipe다.
3. human video는 scene rule, object category, task semantic, low-level trajectory를 모두 줄 수 있다.
4. High-level subtask와 low-level action을 따로 ablate하면 둘 다 transfer에 기여한다.
5. wrist camera는 모든 task에 필요한 것은 아니지만 hand-object detail이 중요한 task에서는 observability를 올린다.
6. robot-only zero-shot generalization과 human-data transferability는 같은 곡선이 아니다. pretraining diversity는 "새 task를 혼자 푸는 능력"보다 "새 data source를 흡수하는 능력"을 더 크게 키울 수 있다.
