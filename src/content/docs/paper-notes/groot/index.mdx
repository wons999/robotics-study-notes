# GR00T Model Family

<SectionFeedback page="GR00T Model Family" section="Overview" />

## 역할

이 문서는 GR00T 계열을 빠르게 잡기 위한 family overview다. 개별 논문이나 release의 기술 detail은 각 상세 Survey 문서에서 다룬다.

핵심 원칙은 다음과 같다.

- family overview는 얕고 넓게 정리한다.
- 개별 paper survey는 항상 깊게 정리한다.
- 논문 기준 수치와 model card/release 기준 수치를 섞지 않는다.
- N1, N1.5, N1.6, N1.7은 이름이 비슷하지만 backbone, action interface, 공개 코드 기준이 다르므로 같은 실험 표처럼 취급하지 않는다.

## 큰 흐름

GR00T 계열의 질문은 “여러 robot embodiment와 여러 종류의 데이터를 하나의 generalist robot policy로 어떻게 묶을 것인가”다. N1 논문은 이 질문을 data pyramid와 dual-system VLA 구조로 푼다. 이후 N1.5/N1.6/N1.7은 공개 checkpoint와 implementation path를 업데이트하면서 backbone, action representation, deployment stack을 바꿔 간다.

| 축 | N1 논문 | N1.7 release 기준 |
| --- | --- | --- |
| VLM backbone | Eagle-2, GR00T-N1-2B total 2.2B params | Cosmos-Reason2-2B / Qwen3-VL architecture, GR00T-N1.7-3B |
| action chunk | 논문 구현에서 horizon `H=16` | 공개 N1.7 config default `action_horizon=40` |
| inference | L40 bf16에서 16-action chunk 63.9 ms, `K=4` denoising | model card timing은 4 denoising steps, TensorRT full pipeline 포함 |
| data framing | real robot, simulation, neural-generated, human video pyramid | human video pretraining, relative EEF action, updated GR00T LeRobot format |
| code path | 논문 설명 기준 | `gr00t/model/gr00t_n1d7`, `launch_finetune.py`, `Gr00tPolicy` 등 공개 구현 기준 |

## 공개 checkpoint inventory

Hugging Face API 기준으로 확인한 공개 GR00T 계열은 다음과 같이 읽는 것이 안전하다.

| 분류 | 공개 checkpoint 예시 | 해석 |
| --- | --- | --- |
| 초기 N1 | `nvidia/GR00T-N1-2B`, task-tuned N1 checkpoints | N1 paper와 직접 연결되는 초기 공개 checkpoint 계열 |
| N1.5 base | `nvidia/GR00T-N1.5-3B` | N1.6/N1.7 이전의 공개 foundation checkpoint |
| N1.5 task variants | `GR00T-N1.5-3B_Assemble_Trocar`, `GR00T-N1.5-3B-WaveHand`, `SO_ARM_Starter_Gr00t` | task/domain별 fine-tuned checkpoint |
| N1.6 base | `nvidia/GR00T-N1.6-3B` | N1.7 직전 open VLA checkpoint |
| N1.6 variants | `GR00T-N1.6-fractal`, `GR00T-N1.6-bridge`, `GR00T-N1.6-DROID`, `GR00T-N1.6-BEHAVIOR1k`, `GR00T-N1.6-G1-PnPAppleToPlate` | benchmark/domain별 fine-tuned checkpoint |
| N1.7 base | `nvidia/GR00T-N1.7-3B` | 현재 repo README가 중심으로 두는 EA base model |
| N1.7 benchmark variants | `GR00T-N1.7-SimplerEnv-Bridge`, `GR00T-N1.7-SimplerEnv-Fractal`, `GR00T-N1.7-DROID`, `GR00T-N1.7-LIBERO` | model card가 명시한 4개 post-trained model |
| N1.7 specialized variants | `GR00T-H-N1.7`, `SO_ARM_Starter_Gr00tN17` | N1.7 기반 파생 checkpoint |

## 작성 원칙

이 family 안의 새 문서를 만들 때는 다음처럼 나눈다.

- paper가 있으면 paper survey로 작성하고, T-Rex 수준의 depth를 목표로 한다.
- paper가 없고 model card/release note만 있으면 model/release survey로 작성한다.
- code가 공개되어 있으면 README가 아니라 실제 model, loss, dataset loader, training script, policy wrapper를 확인한다.
- 같은 이름의 GR00T라도 N1 paper 기준과 N1.7 code 기준이 다르면 “차이” 섹션에 분리해서 쓴다.
