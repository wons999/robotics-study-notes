## 한 줄 요약

## Family 안에서의 위치

| 비교 축 | N1 paper | N1.5 | N1.6/N1.7 방향 |
| --- | --- | --- | --- |
| primary evidence | arXiv paper + appendix | model card/checkpoint 중심 | model card + current Isaac-GR00T code |
| 역할 | data pyramid와 dual-system VLA 제안 | 공개 foundation checkpoint 제공 | backbone, processor, deployment stack 업데이트 |
| 모델 설명 | Eagle-2 VLM + DiT action module | vision/text transformer + MLP + flow matching transformer | N1.6은 architecture 설명 강화, N1.7은 `gr00t_n1d7` 코드 기준 |
| 학습 설명 | real/sim/human/neural data pyramid | real/synthetic data로 post-training 가능 | N1.7은 human video pretraining, relative EEF action, TensorRT path 강조 |
| 주의점 | paper result는 N1 기준 | 독립 paper result가 아님 | release별 성능/데이터/라이선스 기준이 다름 |

N1.5를 N1 paper의 구현 그대로라고 읽으면 안 된다. N1.5는 N1 paper가 만든 연구 질문을 “공개 checkpoint + fine-tuning workflow”로 쓰기 시작한 단계다.

## N1.5 Release Branch가 추가로 말하는 것

`n1.5-release` README는 N1.5를 N1 대비 architecture와 data가 모두 업데이트된 release로 설명한다. 이 내용은 Hugging Face model card보다 release motivation이 더 직접적이지만, paper appendix처럼 모든 수치를 제공하지는 않는다.

| 변화 | 공식 설명 | Survey 해석 |
| --- | --- | --- |
| Frozen VLM | pretraining과 finetuning에서 VLM을 frozen으로 유지 | language understanding을 보존하고 post-training 때 robot-specific gradient가 VLM을 흔드는 위험을 줄이는 방향 |
| Enhanced VLM grounding | Eagle 2.5 기반 grounding 개선, GR-1 grounding task에서 40.4 IoU vs Qwen2.5VL 35.5 | N1.5가 N1 paper의 Eagle-2 설명에서 한 단계 업데이트된 release임을 보여줌 |
| Simplified adapter | vision encoder와 LLM 사이 MLP connector를 단순화하고 layer normalization 추가 | VLM feature와 action transformer 사이 bottleneck을 release-level 성능 요소로 취급 |
| FLARE objective | flow matching loss alongside Future Latent Representation Alignment | human ego video에서 action-free representation을 끌어오는 추가 학습 압력으로 해석 |
| DreamGen integration | synthetic neural trajectories를 포함 | N1 paper의 neural trajectory/data pyramid 방향을 release workflow로 확장 |
| Language following | GR-1 manipulation tasks에서 93.3% vs N1 46.6%로 보고 | 정식 paper table은 아니지만 release branch가 제시한 N1 대비 핵심 claim |
| New embodiment heads | `OXE_DROID`, `AGIBOT_GENIE1`, `NEW_EMBODIMENT` 등 | N1.5가 humanoid-only paper artifact가 아니라 fine-tuning용 multi-embodiment checkpoint로 쓰이도록 설계됨 |

주의할 점은 source마다 표현이 조금 다르다는 것이다. Hugging Face model card의 architecture description은 SigLip2/T5와 flow matching transformer를 적고, `n1.5-release` README는 Eagle 2.5, FLARE, DreamGen 같은 release-level 업데이트를 강조한다. 따라서 문서에서는 “HF checkpoint config로 닫히는 구현 계약”과 “release README가 말하는 개선 claim”을 분리해서 읽는다.

## 입력과 출력 계약

| 구성 | 내용 |
| --- | --- |
| vision input | robot camera RGB frames, embodiment별 view 수는 달라질 수 있음 |
| language input | natural language instruction |
| state input | robot proprioception, embodiment별 dimension이 다름 |
| embodiment condition | 어떤 robot/action schema인지 구분하기 위한 embodiment index 또는 processor 설정 |
| output | robot embodiment에 대응되는 continuous action vector |
| action horizon | model card는 “chunk of actions”라고 설명하며, 정확한 horizon은 checkpoint/config를 함께 확인해야 함 |

핵심은 “모든 robot을 하나의 고정 action vector로 강제로 맞춘다”가 아니다. 각 embodiment의 state/action을 shared transformer space에 올리고, 마지막에는 해당 embodiment의 action dimension으로 decode하는 계열로 이해해야 한다.

| 모듈 | 역할 | Survey에서 봐야 할 점 |
| --- | --- | --- |
| vision transformer | image observation을 visual token으로 변환 | view 수가 embodiment마다 달라도 token sequence로 concatenate하는 구조 |
| text transformer | instruction을 language token으로 변환 | action transformer의 condition으로 들어감 |
| MLP connector | vision-language feature와 DiT/action transformer 사이 연결 | N1.5에서 sim benchmark 성능 개선을 위해 수정된 것으로 설명됨 |
| proprio encoder | proprioception을 action transformer가 쓸 token/embedding으로 변환 | embodiment별 state dimension 차이를 흡수해야 함 |
| flow matching transformer | proprio/action self-attention과 vision-language cross-attention을 interleave | N1 paper의 System 1 계열 역할 |
| action decoder | velocity prediction 또는 denoised action을 embodiment-specific motor command로 변환 | output dimension과 normalization이 중요 |

N1.5는 “VLM이 직접 action token을 autoregressive하게 생성한다”기보다, vision-language features가 continuous action flow model을 condition하는 구조다.

## Model Card 기준 세부 구조

N1.5 model card는 GR00T N1.X architecture diagram을 기준으로 더 구체적인 component를 제시한다.

| Component | model card 근거 | 해석 |
| --- | --- | --- |
| vision encoder | RGB camera frames are processed through SigLip2 | image를 고정 크기 token sequence로 만들고, multi-view는 token sequence concat으로 처리한다. |
| text encoder | text is encoded by T5 | instruction token이 action transformer의 semantic condition이 된다. |
| proprio encoder | embodiment ID로 indexed된 MLP 사용 | variable-dimension proprioception을 max length로 padding한 뒤 embodiment-specific MLP가 처리한다. |
| action encoder/decoder | action encoding과 velocity prediction decoding은 embodiment별 MLP | action dimension과 motor semantics가 embodiment마다 다르다는 점을 구조적으로 반영한다. |
| flow matching transformer | DiT, diffusion step conditioning은 AdaLN | timestep condition이 transformer block normalization에 들어가는 diffusion/flow 계열 구조다. |
| MLP connector | N1.5에서 post-VLM connector가 sim benchmark 성능 개선을 위해 수정됨 | VLM feature와 DiT action head 사이의 bottleneck이 중요한 성능 요소였다는 뜻이다. |

이 표에서 중요한 부분은 SigLip2/T5/DiT라는 이름 자체보다 “vision-language foundation model과 action diffusion transformer 사이에 connector와 embodiment-specific action interface가 있다”는 점이다. Survey나 세미나에서 GR00T를 단일 VLM처럼 설명하면 핵심을 놓친다.

## Model Config Snapshot

Hugging Face `config.json`에서 확인한 N1.5 checkpoint의 구조적 계약이다. model card의 개념 설명보다 구체적이므로, fine-tuning/serving을 생각할 때는 이 값을 먼저 본다.

| 항목 | 값 | 의미 |
| --- | --- | --- |
| `model_type` | `gr00t_n1_5` | current N1.7 code path와 구분해야 하는 checkpoint family |
| `architectures` | `GR00T_N1_5` | Transformers config가 기대하는 model class |
| `hidden_size` | 2048 | backbone output/action head cross-attention dimension의 기준 |
| `action_horizon` | 16 | 한 번에 예측하는 continuous action chunk 길이 |
| `action_dim` / `max_action_dim` | 32 / 32 | N1.5 checkpoint가 처리하는 action vector padding 상한 |
| `max_state_dim` | 64 | proprioception vector padding 상한 |
| `num_inference_timesteps` | 4 | inference에서 velocity prediction을 적용하는 denoising step 수 |
| `num_timestep_buckets` | 1000 | diffusion timestep embedding bucket 수 |
| `noise_beta_alpha`, `noise_beta_beta`, `noise_s` | 1.5, 1.0, 0.999 | N1 paper의 Beta timestep sampling 계열과 같은 noise schedule family |
| `num_target_vision_tokens` | 32 | action head 쪽으로 압축/선택되는 vision token 수 |

Action head 세부는 다음과 같다.

| 항목 | 값 | 해석 |
| --- | --- | --- |
| action head hidden size | 1024 | action/proprio token pathway 내부 width |
| DiT layers | 16 | flow matching transformer depth |
| DiT attention heads | 32 | action/proprio self-attention과 VL cross-attention의 head 수 |
| DiT cross-attention dim | 2048 | VLM feature와 맞물리는 conditioning dimension |
| DiT output dim | 1024 | action decoder로 이어지는 representation size |
| dropout | 0.2 | model card에는 나오지 않는 regularization 설정 |
| norm type | `ada_norm` | diffusion timestep conditioning이 AdaLN 계열로 들어간다는 model card 설명과 일치 |

Backbone config도 사용 관점에서 중요하다.

| 항목 | 값 | 해석 |
| --- | --- | --- |
| `select_layer` | 12 | VLM 중간 layer feature를 action head condition으로 쓰는 N1 paper 계열 선택 |
| `tune_visual` | true | visual encoder는 fine-tuning 대상 |
| `tune_llm` | false | language model은 frozen으로 두는 release 설정 |
| `use_flash_attention` | true | GPU serving/training 최적화 전제 |
| `torch_dtype` | `bfloat16` | 로딩/추론 dtype 기준 |

## Input/Output Contract

| 구분 | N1.5 model card 기준 |
| --- | --- |
| vision input | variable number of `224x224` uint8 RGB image frames |
| state input | 1D floating point robot proprioception |
| language input | string instruction |
| output type | continuous-value action vectors |
| output shape 설명 | 2D action output으로 표기되며, 실제 motor control 의미는 robot DoF/embodiment에 따라 달라짐 |
| runtime | PyTorch |
| OS | Linux |
| supported hardware | NVIDIA Ampere, Blackwell, Jetson, Hopper, Lovelace |

N1.5는 image resolution을 `224x224`로 명시한다. N1.6 model card의 “any resolution” 및 N1.7 README의 native aspect ratio 지원과 구분해야 한다.

## Flow Matching 학습 구조

| 단계 | 의미 |
| --- | --- |
| clean action | demonstration에서 온 continuous action chunk |
| noise | Gaussian noise vector |
| corrupted action | clean action과 noise 사이를 random interpolation해서 만든 training input |
| condition | vision tokens, language tokens, proprioception, timestep |
| target | corrupted action을 clean action 쪽으로 이동시키는 velocity field |
| inference | Gaussian noise에서 시작해 velocity prediction을 반복 적용하여 continuous action을 재구성 |

이 구조는 [Flow Matching](../../flow-matching-generative-modeling/)과 [PI Model Family](../../pi/) 계열 VLA와 직접 연결된다. 다만 N1.5 model card만으로는 timestep distribution, loss normalization, action mask 처리까지 확정할 수 없으므로, 그 세부는 N1.7 code-level Survey처럼 source가 있는 release에서 확인한다.

N1.5 config를 함께 보면 “model card만으로는 확정할 수 없다”는 부분 중 일부가 좁혀진다. `num_inference_timesteps=4`, `noise_beta_alpha=1.5`, `noise_beta_beta=1.0`, `noise_s=0.999`는 N1 paper의 flow-matching/noise schedule과 직접 이어진다. 다만 loss mask, dataset sampling, world-modeling loss weight는 config만으로도 확정할 수 없으므로 여전히 구현 revision 확인 대상이다.

## World-Modeling Objective

N1.5 model card는 flow matching objective와 world-modeling objective를 jointly 사용했다고 설명한다. Survey에서는 이를 다음처럼 해석한다.

| 관점 | 의미 |
| --- | --- |
| action objective | 현재 observation과 instruction에서 action chunk를 생성하는 policy learning |
| world-modeling objective | visual/state transition에 대한 representation learning pressure를 추가해 action policy가 dynamics-sensitive feature를 쓰도록 유도 |
| 불확실한 부분 | loss weight, target modality, prediction horizon은 model card만으로는 제한적으로만 확인 가능 |
| 작성 원칙 | “world modeling을 썼다”는 사실과 “어떤 tensor를 어떤 loss로 예측했다”는 구현 detail을 분리 |

이 지점이 N1.5 Survey에서 가장 조심해야 할 부분이다. 코드나 config를 확인하지 않은 채 auxiliary loss 형태를 만들어 쓰면 안 된다.

## References가 말해주는 방법론 흐름

| Reference | N1.5에서 맡는 역할 |
| --- | --- |
| Eagle 2 | vision-language backbone/post-training strategy 흐름 |
| Rectified Flow / Flow Straight and Fast | noise에서 data로 가는 vector field 학습의 generative modeling 기반 |
| pi_0 | vision-language-action flow model로 continuous robot action을 생성하는 직접 비교 축 |
| GR00T N1 paper | humanoid foundation model, data pyramid, action module의 family origin |

N1.5의 모델카드는 독립 논문이 아니지만, references만 봐도 “VLM + rectified/flow action generation + robot foundation model”이라는 세 축이 명확하다.

## 학습과 Post-Training

| 단계 | N1.5에서의 의미 |
| --- | --- |
| foundation pretraining | 다양한 robot embodiment와 manipulation data를 통해 cross-embodiment prior를 만든다. |
| post-training | target humanoid robot, task, environment에 맞춰 real/synthetic data로 fine-tune한다. |
| synthetic data | Isaac/GR00T ecosystem의 synthetic data generation 흐름과 연결된다. |
| real data | target robot의 camera/state/action schema가 processor와 맞아야 한다. |
| evaluation | N1 paper result를 N1.5 자체 result로 옮기지 않는다. model card 또는 별도 benchmark card의 수치만 N1.5 근거로 쓴다. |

N1.5를 실제로 쓸 때는 model 자체보다 dataset conversion이 병목이 될 가능성이 높다. camera key, state key, action key, normalization, embodiment tag가 맞아야 policy가 의미 있는 action을 낸다.

## N1.5와 N1 Paper의 연결

| N1 paper 개념 | N1.5에서 이어지는 부분 |
| --- | --- |
| System 2 vision-language reasoning | pretrained vision/language encoder가 instruction과 visual context를 제공 |
| System 1 action module | flow matching transformer가 continuous action chunk를 생성 |
| cross-embodiment learning | embodiment-specific projection/decoder 필요성으로 이어짐 |
| data pyramid | model card에서는 real/synthetic post-training 가능성으로 압축됨 |
| low-data adaptation | target robot/task data로 post-training하는 사용 방식으로 이어짐 |

N1.5 문서는 N1 paper의 appendix나 result table을 반복하는 곳이 아니다. 대신 “N1의 방법론이 공개 checkpoint에서 어떤 형태의 사용 계약으로 바뀌었는가”를 정리하는 것이 목적이다.

## N1.6/N1.7과의 차이

| 항목 | N1.5 | N1.6 | N1.7 |
| --- | --- | --- | --- |
| 공개 설명 | foundation checkpoint/model card 중심 | architecture/training 설명이 더 구체적 | source code, README, model card가 가장 구체적 |
| license/use | non-commercial use로 명시 | model card 조건 확인 필요 | commercial/non-commercial ready, Apache 2.0 repository 흐름 |
| backbone | model card는 vision/text transformers와 Eagle reference를 연결 | Eagle 2.5 계열 설명 | Cosmos-Reason2-2B/Qwen3-VL |
| implementation anchor | checkpoint revision 확인 필요 | model card 중심 | `gr00t_n1d7` source code |
| deployment | fine-tuning 사용 중심 | variants 증가 | ONNX/TensorRT, relative EEF action, SONIC |

## 재현 체크리스트

| 체크 | 이유 |
| --- | --- |
| checkpoint revision | current repository가 N1.7 중심으로 바뀌어 old processor와 mismatch가 날 수 있다. |
| license/use restriction | N1.5는 non-commercial use로 설명되므로 공개/상업 사용 조건을 N1.7과 섞으면 안 된다. |
| embodiment tag | state/action projector 선택과 action decoder가 달라질 수 있다. |
| modality config | view, state, action slice가 processor와 정확히 맞아야 한다. |
| action normalization | min/max, relative/absolute, joint/EEF target을 확인해야 한다. |
| performance claim source | N1 paper, N1.5 card, N1.7 card 수치를 한 표에 섞지 않는다. |

## 공식 사용 Workflow

N1.5 release branch는 실제 사용 흐름을 다음처럼 정리한다.

| 단계 | 해야 하는 일 | 의미 |
| --- | --- | --- |
| dataset 준비 | robot demonstration을 video, state, action triplet으로 수집 | N1.5는 action 없는 웹 비디오만으로 바로 target robot을 제어하는 모델이 아니다. |
| LeRobot-compatible schema 변환 | `modality.json`을 포함한 GR00T schema로 정리 | state/action/video key와 semantic slice가 embodiment tag와 맞아야 한다. |
| embodiment head 선택 | `GR1`, `OXE_DROID`, `AGIBOT_GENIE1`, `NEW_EMBODIMENT` 등 | action decoder와 proprio encoder 선택에 직접 연결된다. |
| fine-tuning | `gr00t_finetune.py` 또는 notebook workflow | target robot/task에 맞춰 post-training하는 것이 release의 주요 사용 방식 |
| inference/deployment | `Gr00tPolicy` 또는 inference service/server-client | policy output action chunk를 robot controller가 실행 가능한 command로 연결해야 한다. |

README는 4090에서 fine-tuning할 때 `--no-tune_diffusion_model` option을 제안하고, 권장 fine-tuning은 가능한 batch size를 키워 20k steps를 학습하는 방향이라고 설명한다. LoRA fine-tuning도 가능하지만, release README는 full model fine-tuning을 더 권장한다.

## 한계와 주의

- N1.5는 독립 논문이 아니므로, T-Rex처럼 main text/appendix 기반 result table을 모두 복원할 수 있는 자료 형태가 아니다.
- model card는 architecture 개념을 제공하지만, loss weight, scheduler, batch size, processor implementation detail은 제한적으로만 공개한다.
- current Isaac-GR00T code는 N1.7 중심이므로, N1.5를 재현하려면 repository revision과 checkpoint processor를 맞춰야 한다.
- N1.5의 “cross-embodiment” 주장은 target embodiment fine-tuning data와 schema 품질에 크게 의존한다.
- non-commercial license 조건이 명시되어 있으므로 N1.7의 commercial/non-commercial ready 설명과 섞으면 안 된다.

## 기억할 점

GR00T N1.5의 핵심은 “새 논문 contribution”이 아니라 “N1 연구 아이디어가 공개 checkpoint 사용 방식으로 옮겨간 첫 중간 release”라는 점이다. Survey에서는 N1 paper 결과와 N1.5 model card 설명을 분리하고, checkpoint 사용 계약과 재현 리스크를 중심으로 읽어야 한다.
