## 입력과 출력 계약

| 구성 | 설명 |
| --- | --- |
| visual input | variable number of RGB views |
| language input | natural language instruction |
| state input | robot proprioception, embodiment-specific dimension |
| action input during training | clean action과 Gaussian noise 사이에서 interpolation된 noised action trajectory |
| output | continuous-value action vector, robot DoF와 embodiment에 따라 의미가 달라짐 |
| runtime engine | PyTorch |
| hardware family | Ampere, Blackwell, Jetson, Hopper, Lovelace 계열 지원으로 설명됨 |

N1.6은 “variable views + language + proprioception + noisy action”을 하나의 transformer policy에 넣고, action velocity를 예측해 continuous action을 복원하는 VLA release로 읽는다.

| 모듈 | 역할 | 해석 |
| --- | --- | --- |
| vision transformer | 여러 camera frame에서 image token을 만든다. | view 수가 달라도 image token sequence로 합친다. |
| text transformer | instruction token을 만든다. | vision token 뒤 또는 함께 action transformer condition으로 쓰인다. |
| proprio encoder | robot state를 shared representation으로 올린다. | embodiment별 dimension 차이를 흡수해야 한다. |
| action encoder | noised action chunk와 timestep 정보를 token화한다. | flow matching objective의 입력이다. |
| flow matching transformer | proprio/action self-attention과 vision/language cross-attention을 interleave한다. | N1 paper의 DiT System 1과 같은 역할 계열이다. |
| action decoder | predicted velocity/action representation을 embodiment별 motor command로 바꾼다. | output vector의 physical meaning은 robot별로 다르다. |

N1.6은 N1.5와 마찬가지로 “action을 language token처럼 생성”하는 방식이 아니다. language와 image는 condition이고, action은 continuous vector field로 생성된다.

## N1.5 대비 Architecture 변화

NVIDIA Research의 N1.6 페이지는 N1.6이 N1.5를 단순 checkpoint update한 것이 아니라 architecture, data, modeling을 함께 바꾼 release라고 설명한다.

| 변화 | N1.5 대비 의미 |
| --- | --- |
| VLM backbone | internal NVIDIA Cosmos-2B VLM variant 사용. flexible resolution과 native aspect ratio encoding을 지원한다고 설명됨 |
| DiT scale | DiT가 16 layers에서 32 layers로 커짐 |
| adapter 제거 | N1.5의 post-VLM 4-layer transformer adapter를 제거 |
| VLM tuning | pretraining 중 VLM top 4 layers를 unfreeze |
| action representation | 대부분 embodiment에서 absolute joint/EEF target 대신 state-relative action chunk를 예측 |

이 내용 때문에 N1.6은 N1.5보다 “action head만 조금 좋아진 모델”이 아니다. 더 큰 DiT, 다른 VLM adaptation 방식, relative action representation이 함께 들어간 transition release다.

공식 출처 간 naming도 구분해 두는 것이 좋다. NVIDIA Research page는 N1.6 backbone을 “internal NVIDIA Cosmos-2B VLM variant”라고 설명하지만, Hugging Face `config.json`은 `backbone_model_type=eagle`, `model_name=nvidia/Eagle-Block2A-2B-v2`로 노출한다. 따라서 이 Survey에서는 N1.6을 “Eagle/Cosmos 계열 VLM으로 노출된 N1.6 checkpoint”로 읽고, N1.7의 Cosmos-Reason2-2B/Qwen3-VL code path와는 분리한다.

## Model Config Snapshot

Hugging Face `config.json`에서 확인한 N1.6 checkpoint의 구조적 계약이다. Research page의 설명과 model card 문장만으로는 부족한 수치를 여기서 닫는다.

| 항목 | 값 | 의미 |
| --- | --- | --- |
| `model_type` / `architectures` | `Gr00tN1d6` / `Gr00tN1d6` | N1.6 전용 checkpoint family. N1.7 source path와 구분해야 한다. |
| `model_name` | `nvidia/Eagle-Block2A-2B-v2` | HF config상 backbone anchor. Research page의 Cosmos-2B 표현과 함께 분리해서 읽는다. |
| `backbone_model_type` | `eagle` | model card reference의 Eagle 2.5 흐름과 연결된다. |
| `action_horizon` | 50 | 한 번에 예측하는 continuous action chunk 길이. N1 paper/N1.5의 16과 다르다. |
| `max_state_dim` / `max_action_dim` | 128 / 128 | N1.5의 64/32보다 큰 padding 상한으로, 더 넓은 embodiment/action schema를 전제한다. |
| `max_num_embodiments` | 32 | embodiment-specific state/action module 선택 범위의 상한으로 읽을 수 있다. |
| `num_inference_timesteps` | 4 | inference denoising step 수. N1/N1.5와 같은 4-step flow/action sampling 계열이다. |
| `num_timestep_buckets` | 1000 | timestep embedding bucket 수. |
| `noise_beta_alpha`, `noise_beta_beta`, `noise_s` | 1.5, 1.0, 0.999 | N1 paper의 Beta timestep sampling 계열과 같은 noise schedule family. |
| `model_dtype` / `torch_dtype` | `bfloat16` / `bfloat16` | N1.6은 bf16 loading/training path를 전제한다. |

Action head와 DiT 설정은 N1.5 대비 변화가 가장 명확하다.

| 항목 | N1.6 config 값 | 해석 |
| --- | --- | --- |
| DiT layers | 32 | Research page의 “2x larger DiT” claim과 일치한다. |
| DiT attention heads | 32 | action/proprio self-attention과 VL cross-attention head 수. |
| DiT cross-attention dim | 2048 | VLM feature와 action transformer가 만나는 dimension. |
| DiT output dim | 1024 | action decoder로 이어지는 hidden size. |
| hidden size | 1024 | action head 내부 width. |
| norm type | `ada_norm` | diffusion timestep conditioning이 AdaLN으로 들어간다는 model card 설명과 일치한다. |
| `use_relative_action` | true | Research page의 state-relative action chunk 설명과 직접 대응된다. |
| `use_alternate_vl_dit` | true | N1.6에서 VL-DiT 연결 방식이 바뀐 release임을 시사한다. |

Backbone과 preprocessing 관련 값도 사용 관점에서 중요하다.

| 항목 | 값 | 해석 |
| --- | --- | --- |
| `select_layer` | 16 | action head condition으로 쓰는 VLM layer 선택. |
| `tune_top_llm_layers` | 4 | Research page의 “VLM top 4 layers unfreeze”와 일치한다. |
| `tune_llm` / `tune_visual` | false / false | 전체 LLM/vision encoder를 다 여는 것이 아니라 top layer/connector/DiT 중심으로 읽어야 한다. |
| `tune_projector` / `tune_diffusion_model` | true / true | VLM-action connector와 DiT는 학습 대상. |
| `shortest_image_edge` / `crop_fraction` | 256 / 0.95 | model card는 any resolution을 말하지만, config에는 preprocessing 힌트가 남아 있다. |
| `formalize_language` | true | language instruction preprocessing이 단순 raw string passthrough만은 아닐 수 있다. |
| `color_jitter_params` | brightness/contrast/hue/saturation 0.1 | post-training regularization/data augmentation 설명과 연결된다. |

## Input/Output Contract

| 구분 | N1.6 model card 기준 |
| --- | --- |
| vision input | variable number of RGB image frames from robot cameras |
| vision resolution | any resolution |
| state input | 1D floating point robot proprioception |
| language input | string instruction |
| output | continuous-value action vectors |
| output 의미 | robot embodiment의 DoF와 motor control target에 따라 달라짐 |
| runtime | PyTorch |
| OS | Linux |
| supported hardware | NVIDIA Ampere, Blackwell, Jetson, Hopper, Lovelace |

N1.6부터 resolution 설명이 N1.5의 `224x224`보다 유연해진다. N1.7 README의 native aspect ratio 지원과 연결되지만, N1.7과 동일한 Qwen3/Cosmos-Reason2 code path라고 단정하면 안 된다.

Config 기준으로는 `shortest_image_edge=256`, `crop_fraction=0.95`, `image_target_size=null`, `image_crop_size=null`이므로, model card의 “any resolution”은 고정 224 입력이 아니라 dynamic preprocessing path를 뜻한다고 보는 것이 안전하다.

## Flow Matching 학습 해석

| 단계 | 내용 |
| --- | --- |
| clean action 준비 | demonstration action chunk를 target으로 둔다. |
| noise sampling | Gaussian noise vector를 action과 같은 shape로 만든다. |
| random interpolation | clean action과 noise를 timestep에 따라 섞어 corrupted action을 만든다. |
| transformer conditioning | corrupted action, proprioception, timestep, vision-language embeddings를 입력한다. |
| velocity prediction | clean action 방향으로 이동하는 vector field를 예측한다. |
| inference | Gaussian noise에서 시작해 velocity prediction을 반복 적용해 action을 재구성한다. |

N1.6에서 중요한 점은 flow matching이 단순 loss 이름이 아니라 runtime inference 방식까지 결정한다는 점이다. 학습 때는 noise-action interpolation의 vector field를 맞추고, inference 때는 이 vector field를 적분해 action chunk를 만든다.

Config에서 `action_horizon=50`, `num_inference_timesteps=4`, `use_relative_action=true`가 함께 확인되므로, N1.6의 inference는 “50-step relative action chunk를 4번의 velocity prediction update로 복원하는 continuous action policy”로 요약할 수 있다. 단, action dimension과 physical meaning은 embodiment별 schema에 의해 결정된다.

## World-Modeling Objective와 Representation

N1.6 model card 역시 flow matching과 world-modeling objective의 joint training 흐름을 계승한 것으로 읽힌다.

| 항목 | 의미 |
| --- | --- |
| 목적 | action generation feature가 visual dynamics와 task-relevant state transition을 더 잘 보게 만든다. |
| 기대 효과 | pure imitation loss보다 environment/object 변화에 민감한 representation을 얻는다. |
| 제약 | 공개 model card만으로는 exact target, loss weight, prediction horizon을 확정하기 어렵다. |
| Survey 원칙 | objective가 언급됐다는 사실과 구현 detail을 분리해 쓴다. |

이 부분은 특히 “확실히 확인한 것”과 “계열상 추론한 것”을 구분해야 한다. N1.7 source code처럼 loss implementation을 직접 확인한 release와 같은 강도로 쓰면 안 된다.

## Training Recipe와 Post-Training

| 항목 | N1.6 research page 기준 |
| --- | --- |
| pretraining steps | 300K steps |
| pretraining global batch | 16,384 |
| post-training | small task-specific datasets |
| post-training steps | 보통 10K-30K steps |

N1 paper의 200K pretraining steps/Table 6과 N1.6의 300K steps는 같은 수치가 아니다. 모델 family 안에서 training recipe가 바뀌었다는 점을 명시해야 한다.

## Real-World Rollout Lessons

N1.6 research page의 discussion은 실제 robot rollout에서 중요한 engineering lesson을 준다.

| Lesson | 해석 |
| --- | --- |
| relative actions default | 상대 action은 smoother/more accurate motion을 만들지만, small dataset에서는 error accumulation으로 correction ability가 약해질 수 있다. |
| pretrained statistics | task distribution이 pretraining data와 비슷하면 도움이 되고, 다르면 post-training statistics가 필요할 수 있다. |
| faster convergence | N1.6은 N1.5보다 빠르게 수렴하고 action이 smoother하지만, overfitting 방지를 위해 더 강한 regularization이 필요하다. |
| regularization | stronger state regularization, additional data augmentation, pretraining data co-training을 사용한다. |
| DAgger | real-world underperformance가 있을 때 DAgger가 성능 개선에 효과적이라고 권장된다. |
| RTC | train-time/test-time RTC가 asynchronous rollout에서 motion smoothness와 robustness를 높인다. |
| remaining challenge | multi-task language following과 OOD task generalization은 여전히 어렵고, fine-grained subtask annotation이 도움은 되지만 완전한 해결은 아니다. |

이 섹션은 model card보다 더 중요하다. 실제 로봇에서 foundation checkpoint를 task policy로 바꿀 때 어떤 문제가 남는지 보여주기 때문이다.

## N1.6에서 확인해야 하는 구현 질문

| 질문 | 확인 이유 |
| --- | --- |
| action space가 joint인지 EEF인지 | N1.7은 relative EEF action을 핵심 변화로 내세운다. N1.6과 섞으면 action interface 설명이 틀린다. |
| relative/absolute command 여부 | cross-embodiment transfer 해석이 달라진다. |
| embodiment projector 수와 tag | action/state decoder 선택이 달라진다. |
| world-modeling loss target | model card만으로는 구현 detail이 부족하다. |
| variant별 processor | Bridge/DROID/Fractal/BEHAVIOR1k는 observation/action schema가 다를 수 있다. |
| trainable module | VLM, connector, DiT 중 무엇을 fine-tune했는지 variant별로 다를 수 있다. |

## N1.7로 넘어가는 변화

| 변화 | N1.6 대비 의미 |
| --- | --- |
| new VLM backbone | Eagle 계열에서 Cosmos-Reason2-2B/Qwen3-VL로 이동 |
| relative EEF action | human/robot embodiment 사이 action representation을 더 맞추는 방향 |
| human video pretraining | 20K hours EgoScale human video를 pretraining에 포함 |
| flexible resolution/native aspect ratio | image preprocessing에서 padding 없이 native aspect ratio 지원 |
| simplified processor | `processing_gr00t_n1d7.py` 중심으로 data processing path가 명확해짐 |
| TensorRT/ONNX | full pipeline export와 inference frequency 개선 |

이 변화 때문에 N1.7 source code를 N1.6 구현이라고 역으로 적용하면 안 된다. N1.6은 N1.7의 직전 release로 비교해야지, 같은 코드 경로로 취급하면 안 된다.

## References가 말해주는 방법론 흐름

| Reference | N1.6에서 맡는 역할 |
| --- | --- |
| Eagle 2.5 | long-context/post-training VLM backbone 흐름 |
| Rectified Flow | action flow objective의 generative modeling 기반 |
| pi_0 | VLA flow policy의 직접 비교 축 |
| GR00T N1 paper | humanoid generalist foundation model의 original formulation |

N1.6은 N1.5보다 backbone/data/action representation 변화가 크지만, 방법론 축은 여전히 VLM-conditioned flow action policy다.

## 실무 해석

N1.6은 다음 질문에 답하기 위한 release로 보면 좋다.

| 질문 | N1.6에서의 답 |
| --- | --- |
| GR00T가 VLA 형태로 어떻게 action을 만들까? | vision/language/proprioception을 condition으로 flow matching transformer가 action chunk를 복원한다. |
| cross-embodiment는 어디서 처리할까? | state/action projection과 embodiment-specific decoder/processor 쪽에서 처리해야 한다. |
| benchmark별 모델은 무엇을 의미할까? | base model이 아니라 dataset/domain별 post-trained checkpoint일 수 있다. |
| N1.7을 왜 새 release로 봐야 할까? | backbone, action representation, processor, deployment path가 바뀌기 때문이다. |

## 한계와 주의

- N1.6은 독립 paper가 아니므로 N1 paper처럼 Table/Figure 전체를 복원할 근거가 제한적이다.
- model card의 architecture 설명은 상세하지만, full training recipe와 loss implementation은 공개 코드 수준으로 닫혀 있지 않다.
- N1.6 variants는 이름만 보고 같은 evaluation 조건으로 묶으면 안 된다.
- N1.7에서 개선된 요소를 N1.6에 소급해서 설명하지 않는다.

## 기억할 점

GR00T N1.6은 N1.5와 N1.7 사이의 중요한 transition release다. N1.5보다 architecture와 variant 설명은 구체적이고, N1.7보다 code-level anchor는 약하다. 따라서 Survey에서는 “model card로 확인한 구조”와 “N1.7 code로 확인 가능한 구현”을 분리하는 것이 핵심이다.
