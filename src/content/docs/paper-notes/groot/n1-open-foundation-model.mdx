## 한 줄 요약

GR00T N1은 humanoid robot foundation model을 만들기 위해 VLM 기반 reasoning module과 DiT 기반 fast action module을 결합하고, real robot, simulation, neural-generated video, human video를 action/latent-action target으로 통합해 학습하는 open VLA 모델이다.

## Family 안에서의 위치

GR00T N1은 [GR00T Model Family](../)의 기준 paper다. 이후 N1.5, N1.6, N1.7 release는 backbone과 action interface, 공개 코드가 달라졌지만, “dual-system VLA + heterogeneous data pyramid + flow matching action generation”이라는 핵심 질문은 N1에서 시작된다.

| 구분 | N1 paper 기준 |
| --- | --- |
| 공개 checkpoint | GR00T-N1-2B |
| parameter count | total 2.2B, VLM 1.34B |
| VLM backbone | Eagle-2 |
| action module | DiT-based flow-matching policy |
| action chunk | `H=16` |
| inference | 16-action chunk sampling 63.9 ms on L40 bf16, `K=4` denoising |
| runtime framing | VLM System 2는 10 Hz, action System 1은 120 Hz를 목표로 설명 |

N1.7 공개 구현은 Cosmos-Reason2-2B/Qwen3-VL, action horizon 40, TensorRT path 등으로 바뀌었다. 따라서 이 문서의 수식, 실험, dataset 수치는 N1 paper 기준으로 읽어야 한다.

## 문제의식

논문이 보는 핵심 bottleneck은 “Internet-scale humanoid robot dataset이 없다”는 점이다. humanoid hardware는 general-purpose robot form factor로 매력적이지만, 특정 humanoid 플랫폼에서 모을 수 있는 real robot trajectory는 foundation model을 학습하기엔 너무 적다.

GR00T N1은 이 문제를 두 방향에서 푼다.

| 문제 | GR00T N1의 답 |
| --- | --- |
| robot embodiment가 너무 다양함 | embodiment-specific state/action encoder-decoder로 서로 다른 state/action dimension을 shared DiT space에 올린다. |
| real humanoid data가 부족함 | data pyramid를 만들어 human video, neural video, simulation, real robot trajectory를 함께 쓴다. |
| action 없는 video가 많음 | latent action과 IDM pseudo-action으로 action-less video를 학습 target으로 변환한다. |
| reasoning과 motor control의 속도 요구가 다름 | VLM System 2와 DiT System 1을 분리하되 end-to-end로 결합한다. |
| 새 task data가 적음 | pretrained GR00T를 post-training해 low-data regime에서 fine-tune한다. |

## Training Data Generation

### Latent Actions

Action-less video를 학습에 쓰기 위해 GR00T N1은 latent action을 만든다. 방법은 LAPA 계열 아이디어에 가깝다.

<PaperFigure
  src="/groot/n1-paper/fig04-latent-actions.png"
  alt="Figure 4: latent actions across embodiments"
  caption="논문 Figure 4. 서로 다른 robot/human embodiment에서 비슷한 latent action이 유사한 arm/hand motion semantics를 묶는다는 점을 보여준다."
/>

1. VQ-VAE를 학습한다.
2. encoder는 현재 frame `x_t`와 future frame `x_{t+H}`를 보고 latent action `z_t`를 출력한다.
3. decoder는 `x_t`와 `z_t`로 `x_{t+H}`를 reconstruct한다.
4. 학습 후에는 encoder를 inverse dynamics처럼 사용해 frame pair에서 latent action label을 추출한다.
5. 이 latent action을 별도 “LAPA embodiment”처럼 취급해 flow matching target으로 쓴다.

핵심은 human video를 robot action space로 직접 retarget하지 않고, 여러 embodiment에서 공유 가능한 learned latent action space로 우회한다는 점이다. Fig. 4는 서로 다른 robot/human embodiment에서 유사한 latent action이 “right arm left/right movement” 같은 공통 motion semantics를 갖는다고 해석한다.

## Model Architecture

GR00T N1은 Fig. 2/3 기준으로 VLM System 2와 DiT System 1을 결합한다.

<PaperFigure
  src="/groot/n1-paper/fig03-model-architecture.png"
  alt="Figure 3: GR00T N1 model architecture"
  caption="논문 Figure 3. VLM token은 DiT block의 cross-attention condition으로 들어가고, state/action은 embodiment-specific encoder/decoder를 통해 shared action generation path에 연결된다."
/>

| 모듈 | 역할 |
| --- | --- |
| Eagle-2 VLM | image + text를 vision-language token으로 encode |
| state encoder | embodiment-specific MLP가 proprioceptive state를 shared embedding으로 project |
| action encoder | noised action chunk와 diffusion timestep을 action token으로 encode |
| DiT blocks | state/action tokens에 self-attention, VLM tokens에 cross-attention |
| action decoder | embodiment-specific MLP가 final action tokens를 motor action으로 decode |

### System 2: Eagle-2 VLM

N1 paper의 VLM은 Eagle-2다. Eagle-2는 SmolLM2 LLM과 SigLIP-2 image encoder 기반으로 설명된다.

| 항목 | N1 paper 기준 |
| --- | --- |
| image resolution | 224 x 224 |
| image tokens | pixel shuffle 후 64 image tokens per frame |
| text/image input | VLM chat format |
| extracted feature | `(batch size, sequence length, hidden dimension)` |
| layer choice | final layer보다 middle-layer LLM embedding이 faster inference와 higher success rate를 보였고, GR00T-N1-2B는 12th layer 사용 |
| frozen/trainable | Appendix Table 6 기준 text tokenizer frozen, vision encoder unfrozen |

### System 1: DiT Action Module

DiT는 noised action token과 state token을 처리하고 VLM token에 cross-attention한다. self-attention과 cross-attention block을 alternating 형태로 사용한다는 점에서 Flamingo/VIMA 계열 conditioning 구조와 연결된다.

N1 논문 구현의 action chunk는 `H=16`이다. action chunk는 `A_t = [a_t, a_{t+1}, ..., a_{t+H-1}]`로 정의된다.

## Flow Matching Objective

Ground-truth action chunk `A_t`, timestep `tau`, Gaussian noise `epsilon`이 있을 때 논문은 noised action을 다음처럼 정의한다.

```text
A_t^tau = tau * A_t + (1 - tau) * epsilon
```

논문 수식은 denoising vector field target을 `epsilon - A_t`로 표기한다. 그러나 update 식은 `A_t^{tau+1/K} = A_t^tau + (1/K) V_theta(...)`로 적혀 있고, 현재 N1.7 공개 코드는 `velocity = actions - noise`를 학습한 뒤 Euler update에 더한다. 즉 구현 관점에서는 noise에서 clean action으로 이동하는 `A_t - epsilon` 방향으로 이해하는 것이 자연스럽다. Survey에서는 이 부분을 “논문 표기와 공개 코드 구현의 sign convention 차이 가능성”으로 분리해서 기억한다.

| 항목 | 내용 |
| --- | --- |
| noising path | noise와 action 사이 linear interpolation |
| timestep distribution | `p(tau) = Beta((s - tau) / s; 1.5, 1)`, `s=0.999` |
| inference init | `A_t^0 ~ N(0, I)` |
| inference update | forward Euler integration |
| denoising steps | `K=4` works well across embodiments |
| N1.7 code check | `Gr00tN1d7ActionHead.forward`는 `noisy_trajectory=(1-t)*noise+t*actions`, `velocity=actions-noise`, masked MSE 사용 |

## Training Recipe

### Pre-Training

Pre-training은 data pyramid 전체에서 flow matching loss를 사용한다.

| source | target |
| --- | --- |
| robot datasets | ground-truth robot action + learned latent action |
| human videos | learned latent action |
| neural trajectories | latent action + IDM predicted action |
| simulation trajectories | simulator action / generated task action |

### Post-Training

Post-training은 single embodiment 또는 downstream task dataset에 맞춰 fine-tune한다. 논문은 VLM language component는 frozen으로 두고 나머지 model을 fine-tune한다고 설명한다.

### Training Infrastructure

논문은 GR00T-N1-2B pretraining에 약 50,000 H100 GPU hours를 사용했다고 적는다. 최대 1024 GPUs를 단일 모델 학습에 사용했고, OSMO와 Ray 기반 custom library로 multi-node training/data ingestion을 관리했다고 설명한다. Compute-constrained fine-tuning은 single A6000에서도 테스트했으며, adapter layers와 DiT만 tuning하면 batch size up to 200, vision encoder까지 tuning하면 batch size up to 16이라고 보고한다.

## Post-Training with Neural Trajectories

Post-training에서 real task data가 적을 때 neural trajectory를 같이 쓴다.

| 항목 | 내용 |
| --- | --- |
| simulation task | human-collected trajectory만으로 video generation model fine-tune |
| real-world task | full data가 아니라 10% post-training data만 사용해 realistic low-data setting 유지 |
| generated video action label | latent action 또는 IDM-labeled action |
| sampling ratio | real-world trajectories와 neural trajectories를 1:1로 co-training |
| multi-view generation | multiple view task는 subimages를 grid로 만들어 video model fine-tune |
| future work | img2img diffusion으로 novel initial frame 생성, long-horizon multi-round generation, liquids/articulated objects quantitative eval |

IDM은 current/future image pair를 condition으로 action chunk를 생성하도록 학습한다. Appendix F에 따르면 IDM은 System 1 DiT module과 SigLIP-2 vision embedding을 사용하고, flow-matching objective로 embodiment별 30K 또는 60K steps 학습된다.

## Dataset Format and Action Standardization

Appendix E는 GR00T의 dataset format과 action/state standardization을 설명한다. 핵심은 LeRobot format을 그대로 쓰되, cross-embodiment VLA에 필요한 metadata를 더 강하게 요구한다는 점이다.

| 구성 | 내용 |
| --- | --- |
| tabular data | robot states, actions, metadata는 parquet |
| image/video | MP4 video 또는 PNG sequence, parquet에서 reference |
| metadata | statistics, episode indices, dataset metadata는 JSON |
| episode | synchronized observation-action pairs |
| modality config | `meta/modality.json`이 state/action vector dimension의 semantic meaning을 정의 |
| annotation | task description, validity flag, success indicator 등 multiple annotation support |

Action/state standardization은 다음 practice를 포함한다.

| 항목 | 목적 |
| --- | --- |
| EEF rotation state -> 6D rotation | Euler singularity/discontinuity 회피 |
| EEF rotation action -> axis-angle | compact하고 smooth한 rotation control |
| min-max normalization | joint state/action, EEF position/action range 통일 |
| consistent ordering | left-to-right order, EEF rotation -> position -> gripper closeness 순서 |

## Main Results

### Pre-Training Evaluation

Pretrained GR00T-N1-2B는 real GR-1 humanoid에서 두 개의 task로 평가된다.

| Task | 평가 | Result |
| --- | --- | ---: |
| left-to-right bimanual handover 후 shelf placement | 5 objects x 3 trials | 76.6% (11.5/15) |
| novel object를 unseen target container에 넣기 | 5 objects x 3 trials | 73.3% (11/15) |

논문은 0.5 점수를 “object를 correctly grasp했지만 container placement 실패”로 설명한다. 이 partial credit은 real robot success rate 해석에서 중요하다.

### Simulation Results

Table 2는 100 demonstrations per task 기준 평균 success rate다.

| Model | RoboCasa | DexMG | GR-1 | Average |
| --- | ---: | ---: | ---: | ---: |
| BC Transformer | 26.3% | 53.9% | 16.1% | 26.4% |
| Diffusion Policy | 25.6% | 56.1% | 32.7% | 33.4% |
| GR00T-N1-2B | 32.1% | 66.5% | 50.0% | 45.0% |

가장 큰 차이는 GR-1 tabletop에서 난다. GR00T-N1-2B는 Diffusion Policy 대비 +17.3%p, BC Transformer 대비 +33.9%p 높다. 이 결과는 GR00T가 humanoid/dexterous embodiment에 pretrained prior를 잘 옮긴다는 주장과 연결된다.

Appendix Table 4와 Fig. 10은 demo 수별 결과를 제공한다.

| Benchmark | Diffusion Policy 30/100/300 demos | GR00T-N1-2B 30/100/300 demos |
| --- | --- | --- |

| DexMG average | 23.7 / 46.9 / 68.4 | 29.6 / 58.5 / 74.2 |

GR-1에서는 30 demos에서도 GR00T가 43.2%로 Diffusion Policy 300 demos의 40.4%를 넘는다. 이 부분이 sample efficiency claim의 핵심이다.

## 한계와 주의

- N1의 많은 핵심 data는 in-house GR-1 teleop, neural trajectory generation, commercial-grade multimodal LLM filtering에 의존한다. 공개 checkpoint와 dataset이 있어도 전체 training pipeline 재현성은 제한적이다.
- Latent action과 IDM pseudo-action은 action-less video를 활용하는 강력한 장치지만, pseudo-label quality가 policy learning에 직접 영향을 준다.
- Simulation/real task 모두 manipulation 중심이다. locomotion이나 whole-body navigation까지 일반화된 foundation policy로 읽으면 과장이다.
- N1 paper와 N1.7 public release는 backbone과 action interface가 다르다. N1.7 code detail을 N1 paper 실험 결과의 구현이라고 단정하면 안 된다.
- Tactile/force feedback은 핵심 modality가 아니다. contact-rich dexterous manipulation에서는 [T-Rex](../../t-rex-tactile-reactive-dexterous-manipulation/) 같은 tactile-reactive pathway와 별도 비교가 필요하다.

## 기억할 점

GR00T N1의 핵심은 “큰 VLM + action DiT”만이 아니다. 더 중요한 contribution은 action 없는 human/neural video까지 latent action과 IDM으로 action-conditioned training corpus에 편입하고, 그 corpus를 embodiment-aware flow matching policy로 학습했다는 점이다. 이 때문에 GR00T N1은 model architecture paper이면서 동시에 data strategy paper다.
