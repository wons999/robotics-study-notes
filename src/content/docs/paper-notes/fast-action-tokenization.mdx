## One-line Summary

FAST compresses high-frequency continuous robot action sequences into frequency-space tokens, making autoregressive VLA action modeling more practical for dexterous tasks.

## Problem Setting

- Robot embodiment: diverse robot action spaces and control frequencies.
- Observation: VLA observations paired with continuous action trajectories.
- Action: high-frequency continuous action sequences tokenized for autoregressive models.
- Data: large robot action trajectory corpus.
- Task: train autoregressive VLA policies for dexterous high-frequency behavior.

## Core Idea

Simple per-dimension, per-timestep action binning performs poorly for high-frequency dexterous control. FAST uses a discrete cosine transform based compression scheme so action sequences can be represented with fewer, more meaningful tokens.

## Method Details

- Architecture: action tokenizer used with VLA policies.
- Training recipe: train or apply frequency-space tokenization over robot trajectories.
- Data: large robot action trajectories.
- Inference / control loop: VLA predicts compressed action tokens, then decoder reconstructs continuous actions.

## What I Should Remember

- FAST is another answer to the action representation problem.
- pi_0 and T-Rex use flow matching for continuous action generation; FAST keeps an autoregressive token route viable.
- High-frequency dexterous control exposes the weakness of naive action discretization.

## Limitations

- Tokenization quality depends on action sequence structure.
- Compression can discard high-frequency details if poorly tuned.
- It does not address tactile sensing directly.
