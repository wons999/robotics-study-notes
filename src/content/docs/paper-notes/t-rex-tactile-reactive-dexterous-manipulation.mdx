## 한 줄 요약

<SectionFeedback page="T-Rex" section="한 줄 요약" />

T-Rex는 vision-language-action 계열 정책에 **빠른 tactile refinement expert**를 붙이고, 100시간 규모의 tactile-rich bimanual robot dataset으로 mid-training하여, 접촉이 중요한 dexterous manipulation에서 더 안정적으로 반응하게 만든 모델이다.

## 문제의식

<SectionFeedback page="T-Rex" section="문제의식" />

기존 VLA나 imitation policy는 이미지와 언어를 보고 그럴듯한 action을 만들 수 있지만, contact-rich manipulation에서는 자주 깨진다. 이유는 크게 세 가지다.

| 문제 | 왜 중요한가 |
| --- | --- |
| tactile signal 부족 | 물체가 미끄러지는지, 눌리는지, 끼워졌는지 같은 정보는 RGB만으로 늦거나 부정확하게 보인다. |
| 주파수 mismatch | VLA backbone은 무겁고 느린데, 촉각 feedback은 빠르게 반응해야 한다. |
| 데이터 분포 부족 | 고자유도 손과 양팔 조작에서 접촉이 풍부한 robot data가 충분하지 않다. |

T-Rex의 핵심 주장은 단순하다. **Touch를 또 하나의 input token으로만 넣으면 부족하고, tactile feedback을 빠른 제어 보정 신호로 다룰 구조가 필요하다.**

## 데이터셋

### 데이터셋 규모

| 항목 | 값 |
| --- | --- |
| 총 길이 | 100시간 |
| episode 수 | 7,755개 |
| object 수 | 207개 |
| motor primitive 수 | 22개 |
| object-motor primitive 조합 | 502개 |
| median episode length | 약 29.8초 |
| 수집 기간 | 10주 |

### 기록되는 modality

각 episode는 timestamp가 맞춰진 여러 stream으로 저장된다.

| modality | 구체 내용 |
| --- | --- |
| RGB | head ZED X Mini 1개, wrist ZED X One S 2개, 640x360, 30Hz |
| proprioception | 양팔 `2 x 7` joint position/velocity, 양손 `2 x 22` Sharpa Wave hand joint state |
| wrist pose | 양손 wrist의 SE(3) end-effector pose |
| tactile | 10개 fingertip 각각의 single-channel deformation depth map과 6-axis net wrench |
| action | bimanual arm relative end-effector command, hand joint command |
| language | VLM 기반으로 생성하고 사람이 검수한 task instruction |

데이터는 high-level teleoperation thread의 30Hz cadence에 맞춰 기록된다. 즉 RGB, proprioception, wrist pose, tactile, action, language가 같은 timestamp 기준으로 정렬되어 있고, contact 순간의 force/deformation 변화와 operator action을 함께 추적할 수 있다.

### taxonomy가 중요한 이유

T-Rex dataset은 “task 이름 몇 개”로만 구성되지 않는다. 207개 object와 22개 motor primitive를 조합한 뒤, 물리적으로 말이 되는 조합만 남긴다. 예를 들어 단단한 block에 `pour`를 적용하는 식의 불가능한 조합은 제거한다.

| 항목 | 값 | 해석 |
| --- | --- | --- |
| 후보 조합 | `207 x 22 = 4,554` | 모든 object와 motor primitive를 단순 곱한 후보 |
| 최종 조합 | 502개 | per-primitive feasibility checklist로 불가능 조합 제거 |
| 조합당 demonstration | 평균 약 16개 | 같은 primitive-object 조합 안에서도 action variation을 확보 |
| episode 길이 | median 29.8초, IQR 21.0-41.1초 | 짧은 grasp clip이 아니라 접촉 변화가 포함된 조작 episode |

이 방식의 장점은 policy가 개별 task만 외우는 것이 아니라 다음 같은 재사용 가능한 contact behavior를 배우게 한다는 점이다.

- 누르기
- 잡기
- 밀기
- 문지르기
- 끼우기
- 비틀기
- 변형시키기
- 미끄러짐을 보정하기

### 데이터 구성 예시

하나의 episode를 실제 학습 샘플로 보면 다음처럼 읽을 수 있다.

| 필드 | 예시 | policy가 얻는 정보 |
| --- | --- | --- |
| `head_rgb[t]` | 전체 작업대와 목표 물체 | task context, distractor 구분 |
| `left_wrist_rgb[t]`, `right_wrist_rgb[t]` | 손가락 주변 근접 시야 | occlusion이 많은 접촉 영역 보완 |
| `arm_qpos[t]`, `arm_qvel[t]` | 양팔 7-DoF 상태 | 현재 end-effector 움직임과 안정성 |
| `hand_qpos[t]` | 양손 22-DoF finger joint | grasp aperture, finger configuration |
| `wrist_pose[t]` | 양손 SE(3) pose | object-relative manipulation geometry |
| `tactile_force[t-k:t]` | 10 fingertip x 6D wrench history | grip force, slip, collision, pressing strength |
| `tactile_deform[t]` | fingertip deformation depth map | 접촉 위치, edge, shear, local geometry |
| `action[t:t+H]` | arm delta + finger joint command chunk | flow matching target action |
| `instruction` | “top cup을 분리해라” 같은 imperative sentence | language-conditioned task grounding |

### scene diversity와 annotation

논문 부록 G는 scene-level diversity도 강조한다. 수집 시 6개의 tabletop backdrop을 사용하고, 210개 이상의 non-target item pool에서 0-5개 distractor object를 함께 배치한다. 같은 object-motor pair라도 초기 위치와 회전을 매 episode randomize하므로, policy는 특정 위치/배경을 외우기보다 language와 contact cue로 목표를 찾도록 압박받는다.

언어 annotation은 episode마다 head camera에서 4-6개 frame을 subsampling하고, teleoperation 중 기록한 최소 label인 target object name과 motor primitive name을 함께 VLM에 넣어 imperative sentence를 생성한다. 이후 사람이 hallucination이나 부정확한 설명을 검수한다. 즉 language는 단순 task id가 아니라 episode motion을 설명하는 자연어 instruction으로 들어간다.

### data cleaning과 release

수집 후에는 unstable tactile measurement, corrupted sensor stream, teleoperation failure, extreme joint velocity가 포함된 demonstration을 제거한다. 논문은 raw sensor stream, derived tactile representation, language annotation, data loader, preprocessing script를 MIT license로 공개할 계획이라고 설명한다.

## 모델 구조

<SectionFeedback page="T-Rex" section="모델 구조" />

![T-Rex 모델 구조](https://arxiv.org/html/2606.17055v1/x3.png)

그림 출처: [T-Rex Figure 3](https://arxiv.org/html/2606.17055v1)

T-Rex는 Mixture-of-Transformer-Experts 구조다. 중요한 점은 expert들이 같은 속도로 돌지 않는다는 것이다. visual-language reasoning은 느리게, tactile correction은 빠르게 동작한다.

| 구성요소 | 역할 |
| --- | --- |
| Vision-language backbone | 이미지와 언어 instruction에서 task context를 만든다. |
| Latent expert | visual-language context를 처리하고 future visual representation을 예측한다. |
| Action expert | coarse action chunk를 low-frequency로 생성한다. |
| Tactile expert | tactile force/deformation을 보고 action을 high-frequency로 보정한다. |
| Tactile encoder | force history와 deformation map을 tactile token으로 바꾼다. |

### tactile encoder

T-Rex의 tactile encoder는 두 갈래다.

| stream | 입력 | 역할 |
| --- | --- | --- |
| temporal force stream | fingertip force history | 최근 contact force 변화를 VQ-VAE token으로 압축한다. |
| spatial deformation stream | tactile deformation map | 접촉 위치와 국소 변형 형태를 convolution encoder로 뽑는다. |

이 설계는 중요한 trade-off를 건드린다. Force vector는 빠르고 compact하지만 접촉 geometry를 잃기 쉽고, deformation map은 공간 정보가 풍부하지만 처리 비용이 있다. T-Rex는 둘을 함께 써서 시간적 force 변화와 공간적 접촉 형상을 동시에 잡으려 한다.

#### temporal force VQ-VAE

| 항목 | 값 / 구조 |
| --- | --- |
| 입력 | fingertip별 6D force vector의 최근 `T = 16` frame window |
| encoder | 1D temporal convolution, 두 개의 strided block으로 temporal dimension downsample |
| pooling | temporal mean pooling |
| continuous embedding | 256-dimension |
| codebook size | `K = 64` |
| codebook update | EMA update, underutilized code reseeding으로 collapse 방지 |
| decoder | quantized token에서 원 force sequence 복원 |
| loss | magnitude-weighted MSE, high-force contact frame에 더 큰 penalty |
| sharing | 5개 finger가 convolution weight를 공유하고, finger identity embedding을 주입 |

이 encoder는 손가락마다 하나의 discrete force token을 만든다. 핵심은 sensor drift와 noisy force history를 그대로 policy에 넣지 않고, contact pattern vocabulary로 압축한다는 점이다.

#### tactile deformation encoder

| 항목 | 값 / 구조 |
| --- | --- |
| 입력 | fingertip별 single-channel deformation depth map |
| backbone | ResNet-18 변형 |
| input stem | single-channel tactile map을 받도록 수정 |
| retained stages | first three residual stages |
| projection | 각 stage 뒤에 `3 x 3` convolution을 붙여 128 channel width로 재투영 |
| final step | spatial tensor flatten 후 policy token space로 linear projection |
| 학습 방식 | self-supervised convolutional autoencoder로 pretrain 후 policy learning 중 freeze |

deformation map은 force vector가 잃기 쉬운 edge, slip, shear, local contact geometry를 담는다. T-Rex는 VQ-VAE force token과 deformation embedding을 concatenation해 tactile expert가 사용할 unified tactile observation을 만든다.

## 핵심 방법론: Asynchronous Cascaded Flow Matching

<SectionFeedback page="T-Rex" section="핵심 방법론" />

T-Rex는 action generation을 flow matching으로 본다. 일반적인 flow matching은 noise에서 action으로 가는 denoising trajectory를 학습한다. T-Rex는 이 trajectory를 둘로 나눈다.

| 단계 | 담당 expert | 의미 |
| --- | --- | --- |
| 앞부분 denoising | action expert | visual-language context 기반의 coarse action 생성 |
| 뒷부분 denoising | tactile expert | 실제 tactile feedback을 반영한 빠른 action refinement |

이 구조를 “asynchronous cascaded”라고 볼 수 있다. 비싼 visual-language 계산은 action chunk 단위로 낮은 주파수에서 수행하고, tactile expert는 cached context를 재사용하면서 더 높은 주파수로 action을 보정한다.

### objective와 conditioning

두 expert는 같은 flow target velocity를 회귀하지만 condition이 다르다.

| expert | condition | 역할 |
| --- | --- | --- |
| action expert | head/wrist camera feature, language prompt, future-prediction token이 포함된 visual-language latent context | slow stream에서 noisy action을 intermediate action state까지 denoise |
| tactile expert | high-frequency tactile token, slow stream에서 detached된 intermediate state와 KV cache | raw visual observation을 다시 보지 않고 fast stream에서 남은 denoising 수행 |

전체 loss는 action expert loss, tactile expert loss, future-frame visual prediction loss를 함께 최적화한다. 논문은 `lambda_tac = 1.0`, `lambda_future = 0.5`를 사용한다. deployment와 training의 mismatch를 줄이기 위해 tactile token frame index를 visual context와 일부러 어긋나게 하는 delay augmentation도 넣는다. delay는 `{0, 4, 8, 12}` frame 중 하나로 sampling된다.

### inference loop

| stream | 주기/역할 | 계산 내용 |
| --- | --- | --- |
| slow stream | action chunk window마다 실행 | visual-language context 계산, Gaussian noise에서 `K_slow` step denoising, boundary state와 KV cache 저장 |
| fast stream | action chunk 내부 offset `{0, 4, 8, 12}`에서 실행 | real-time tactile stream을 읽고, cached KV와 boundary state를 clone한 뒤 `K_fast` Euler step으로 action chunk 갱신 |
| low-level control | 300Hz controller | policy가 비동기적으로 갱신한 target을 추종 |

thread safety를 위해 deployment runtime은 single-threaded request socket과 execution lock을 사용한다. fast tick은 in-flight slow tick이 cache와 boundary state를 shared memory에 commit하기 전에는 시작하지 않는다.

### 왜 이게 중요한가

Contact-rich manipulation에서는 몇 프레임 사이에도 상황이 바뀐다. 손가락이 살짝 미끄러지거나, 물체가 변형되거나, 삽입 축이 틀어질 수 있다. 이때 매번 큰 VLA를 다시 돌리면 느리고, tactile을 action chunk 시작점에만 넣으면 늦다. T-Rex의 구조는 이 문제를 architecture level에서 해결하려는 시도다.

## 학습 recipe

<SectionFeedback page="T-Rex" section="학습 recipe" />

T-Rex는 세 단계로 학습한다.

| 단계 | 데이터 | 목적 |
| --- | --- | --- |
| pre-training | 대규모 human egocentric video/action data | 넓은 visuomotor prior 학습 |
| mid-training | 100시간 T-Rex tactile robot dataset | robot embodiment와 tactile contact dynamics에 맞춤 |
| post-training | target task별 약 100개 demonstration | 특정 평가 task에 빠르게 적응 |

여기서 가장 중요한 단계는 mid-training이다. T-Rex는 tactile dataset을 단순 추가 데이터가 아니라, human-derived prior를 실제 robot contact dynamics에 연결하는 grounding 단계로 본다.

## 평가 task와 결과

<SectionFeedback page="T-Rex" section="평가 task와 결과" />

T-Rex는 12개 real-world tactile-reactive task에서 평가된다. 과제는 insertion, deformable object handling, force-sensitive transfer, bimanual coordination처럼 RGB만으로는 부족한 조작을 포함한다.

비교 대상은 다음 계열이다.

- vision-only 혹은 visuo-tactile imitation policy
- tactile-aware VLA
- EgoScale 계열
- pi_0 계열 flow-matching VLA
- RDP 같은 slow-fast tactile policy

평가는 task마다 16회 rollout으로 수행하고, object position과 rotation을 trial마다 randomize한다. metric은 task success rate이며, multi-stage task에는 progress-based 혹은 additive rubric을 적용해 부분 성공도 반영한다.

### tactile modality와 architecture ablation

Table 2는 6개 대표 tactile-reactive task에서 tactile modality와 architecture design을 제거/변경한 결과다.

| configuration | Flip Page | Apply Toothpaste | Split Cup | Open Lock | Extract Card | Screw Lightbulb | Average |
| --- | --- | --- | --- | --- | --- | --- | --- |
| Full Model | 96 | 66 | 78 | 47 | 70 | 35 | 65 |
| w/o Tactile | 76 | 39 | 58 | 23 | 34 | 20 | 42 (-23%) |
| MLP Force + Deform | 89 | 58 | 72 | 44 | 58 | 29 | 58 (-7%) |
| Deform only | 82 | 57 | 71 | 36 | 55 | 25 | 54 (-11%) |
| MLP Force + VQ-VAE Force | 92 | 63 | 65 | 38 | 67 | 28 | 59 (-6%) |
| w/o Async | 92 | 61 | 73 | 45 | 59 | 30 | 60 (-5%) |

| ablation | 무엇을 검증하나 | 해석 |
| --- | --- |
| w/o Tactile | tactile feedback 자체의 필요성 | 평균 -23%p로 가장 큰 하락. contact-rich task에서는 vision/proprioception만으로 부족하다. |
| MLP Force + Deform | force history를 VQ-VAE 대신 단순 MLP로 처리 | 평균 -7%p. temporal force를 discrete token으로 압축하는 설계가 도움이 된다. |
| Deform only | force stream 없이 deformation map만 사용 | 평균 -11%p. spatial contact geometry만으로는 force dynamics가 부족하다. |
| MLP Force + VQ-VAE Force | deformation map 없이 force 계열만 사용 | 평균 -6%p. force token이 강하지만 local geometry까지 있으면 더 좋다. |
| w/o Async | asynchronous tactile refinement 제거 | 평균 -5%p. fast tactile loop가 실제 성능에 기여한다. |

핵심 결론은 **데이터, tactile representation, 비동기 control-frequency 설계가 함께 필요하다**는 것이다. 어느 하나만 넣는다고 T-Rex 수준이 나오지 않는다.

## Data efficiency

<SectionFeedback page="T-Rex" section="Data efficiency" />

![T-Rex data efficiency](https://arxiv.org/html/2606.17055v1/x6.png)

그림 출처: [T-Rex Figure 6](https://arxiv.org/html/2606.17055v1)

T-Rex는 target task마다 많은 demonstration을 새로 모으기보다, tactile-rich mid-training을 먼저 해두고 post-training demo 수를 줄이는 방향을 노린다. 이 관점에서 Figure 6은 “foundation-style robot policy가 tactile dataset을 통해 얼마나 빠르게 downstream task에 적응하는가”를 보는 그림이다.

![Post-training data efficiency](https://arxiv.org/html/2606.17055v1/x5.png)

그림 출처: [T-Rex Figure 5](https://arxiv.org/html/2606.17055v1)

Figure 5는 `Apply Toothpaste`, `Split Cup`, `Extract Card`에서 downstream demonstration 수를 10, 20, 50, 100, 200개로 바꾸며 본 data efficiency curve다. tactile-grounded T-Rex mid-training을 거친 모델은 적은 demonstration 구간에서 특히 유리하다. 이 결과는 T-Rex dataset이 단순히 최종 task demo를 많이 모은 것이 아니라, downstream contact-rich skill에 재사용 가능한 prior를 만든다는 주장을 뒷받침한다.

### mid-training dataset ablation

Figure 6은 100시간 T-Rex dataset mid-training과 100시간 task-specific data mid-training, 그리고 mid-training 없는 조건을 비교한다. 논문은 post-training task 6개와 zero-shot task 4개를 골라 비교하고, T-Rex dataset 방식이 더 강한 generalization과 zero-shot transfer를 만든다고 해석한다. zero-shot task에는 T-Rex dataset 안의 motor primitive인 pick, slide, press, wipe 계열이 포함된다.

## 실패 사례와 한계

<SectionFeedback page="T-Rex" section="실패 사례와 한계" />

논문이 스스로 인정하는 한계는 꽤 현실적이다.

| 한계 | 해석 |
| --- | --- |
| long-horizon precision task | offline BC/post-training만으로는 tight contact tolerance를 끝까지 맞추기 어렵다. |
| tactile hardware bottleneck | sensor drift, calibration, device variation, palm coverage 문제가 남아 있다. |
| platform specificity | 특정 bimanual dexterous hardware와 teleoperation stack에 의존한다. |
| online recovery 부족 | 실패 후 회복이나 탐색은 RL/online adaptation이 더 필요할 수 있다. |

## 이 논문을 읽을 때 기억할 것

<SectionFeedback page="T-Rex" section="이 논문을 읽을 때 기억할 것" />

- tactile은 “추가 observation”이 아니라 **빠른 correction signal**로 다뤄야 한다.
- VLA의 느린 reasoning과 tactile의 빠른 feedback 사이에는 구조적 mismatch가 있다.
- T-Rex의 핵심은 이 mismatch를 variable-rate MoT와 cascaded flow matching으로 나눈 것이다.
- 100시간 tactile dataset은 task demo 모음이 아니라 reusable motor primitive와 contact behavior를 배우기 위한 mid-training corpus다.
- force history와 deformation map을 함께 쓰는 tactile representation이 중요하다.
