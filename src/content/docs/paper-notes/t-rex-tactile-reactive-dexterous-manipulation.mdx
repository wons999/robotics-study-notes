## 한 줄 요약

<SectionFeedback page="T-Rex" section="한 줄 요약" />

T-Rex는 vision-language-action 계열 정책에 **빠른 tactile refinement expert**를 붙이고, 100시간 규모의 tactile-rich bimanual robot dataset으로 mid-training하여, 접촉이 중요한 dexterous manipulation에서 더 안정적으로 반응하게 만든 모델이다.

## 문제의식

<SectionFeedback page="T-Rex" section="문제의식" />

기존 VLA나 imitation policy는 이미지와 언어를 보고 그럴듯한 action을 만들 수 있지만, contact-rich manipulation에서는 자주 깨진다. 이유는 크게 세 가지다.

| 문제 | 왜 중요한가 |
| --- | --- |
| tactile signal 부족 | 물체가 미끄러지는지, 눌리는지, 끼워졌는지 같은 정보는 RGB만으로 늦거나 부정확하게 보인다. |
| 주파수 mismatch | VLA backbone은 무겁고 느린데, 촉각 feedback은 빠르게 반응해야 한다. |
| 데이터 분포 부족 | 고자유도 손과 양팔 조작에서 접촉이 풍부한 robot data가 충분하지 않다. |

T-Rex의 핵심 주장은 단순하다. **Touch를 또 하나의 input token으로만 넣으면 부족하고, tactile feedback을 빠른 제어 보정 신호로 다룰 구조가 필요하다.**

| 항목 | 값 |

| 수집 기간 | 10주 |

### 기록되는 modality

각 episode는 timestamp가 맞춰진 여러 stream으로 저장된다.

| modality | 구체 내용 |
| --- | --- |
| RGB | head ZED X Mini 1개, wrist ZED X One S 2개, 640x360, 30Hz |
| proprioception | 양팔 `2 x 7` joint position/velocity, 양손 `2 x 22` Sharpa Wave hand joint state |
| wrist pose | 양손 wrist의 SE(3) end-effector pose |
| tactile | 10개 fingertip 각각의 single-channel deformation depth map과 6-axis net wrench |
| action | bimanual arm relative end-effector command, hand joint command |
| language | VLM 기반으로 생성하고 사람이 검수한 task instruction |

데이터는 high-level teleoperation thread의 30Hz cadence에 맞춰 기록된다. 즉 RGB, proprioception, wrist pose, tactile, action, language가 같은 timestamp 기준으로 정렬되어 있고, contact 순간의 force/deformation 변화와 operator action을 함께 추적할 수 있다.

### taxonomy가 중요한 이유

T-Rex dataset은 “task 이름 몇 개”로만 구성되지 않는다. 207개 object와 22개 motor primitive를 조합한 뒤, 물리적으로 말이 되는 조합만 남긴다. 예를 들어 단단한 block에 `pour`를 적용하는 식의 불가능한 조합은 제거한다.

| 항목 | 값 | 해석 |
| --- | --- | --- |
| 후보 조합 | `207 x 22 = 4,554` | 모든 object와 motor primitive를 단순 곱한 후보 |
| 최종 조합 | 502개 | per-primitive feasibility checklist로 불가능 조합 제거 |
| 조합당 demonstration | 평균 약 16개 | 같은 primitive-object 조합 안에서도 action variation을 확보 |
| episode 길이 | median 29.8초, IQR 21.0-41.1초 | 짧은 grasp clip이 아니라 접촉 변화가 포함된 조작 episode |

이 방식의 장점은 policy가 개별 task만 외우는 것이 아니라 다음 같은 재사용 가능한 contact behavior를 배우게 한다는 점이다.

- 누르기
- 잡기
- 밀기
- 문지르기
- 끼우기
- 비틀기
- 변형시키기
- 미끄러짐을 보정하기

### 데이터 구성 예시

하나의 episode를 실제 학습 샘플로 보면 다음처럼 읽을 수 있다.

| 필드 | 예시 | policy가 얻는 정보 |
| --- | --- | --- |
| `head_rgb[t]` | 전체 작업대와 목표 물체 | task context, distractor 구분 |
| `left_wrist_rgb[t]`, `right_wrist_rgb[t]` | 손가락 주변 근접 시야 | occlusion이 많은 접촉 영역 보완 |
| `arm_qpos[t]`, `arm_qvel[t]` | 양팔 7-DoF 상태 | 현재 end-effector 움직임과 안정성 |
| `hand_qpos[t]` | 양손 22-DoF finger joint | grasp aperture, finger configuration |
| `wrist_pose[t]` | 양손 SE(3) pose | object-relative manipulation geometry |
| `tactile_force[t-k:t]` | 10 fingertip x 6D wrench history | grip force, slip, collision, pressing strength |
| `tactile_deform[t]` | fingertip deformation depth map | 접촉 위치, edge, shear, local geometry |
| `action[t:t+H]` | arm delta + finger joint command chunk | flow matching target action |
| `instruction` | “top cup을 분리해라” 같은 imperative sentence | language-conditioned task grounding |

### scene diversity와 annotation

논문 부록 G는 scene-level diversity도 강조한다. 수집 시 6개의 tabletop backdrop을 사용하고, 210개 이상의 non-target item pool에서 0-5개 distractor object를 함께 배치한다. 같은 object-motor pair라도 초기 위치와 회전을 매 episode randomize하므로, policy는 특정 위치/배경을 외우기보다 language와 contact cue로 목표를 찾도록 압박받는다.

언어 annotation은 episode마다 head camera에서 4-6개 frame을 subsampling하고, teleoperation 중 기록한 최소 label인 target object name과 motor primitive name을 함께 VLM에 넣어 imperative sentence를 생성한다. 이후 사람이 hallucination이나 부정확한 설명을 검수한다. 즉 language는 단순 task id가 아니라 episode motion을 설명하는 자연어 instruction으로 들어간다.

### data cleaning과 release

수집 후에는 unstable tactile measurement, corrupted sensor stream, teleoperation failure, extreme joint velocity가 포함된 demonstration을 제거한다. 논문은 raw sensor stream, derived tactile representation, language annotation, data loader, preprocessing script를 MIT license로 공개할 계획이라고 설명한다.

## 모델 구조

<SectionFeedback page="T-Rex" section="모델 구조" />

![T-Rex 모델 구조](https://arxiv.org/html/2606.17055v1/x3.png)

그림 출처: [T-Rex Figure 3](https://arxiv.org/html/2606.17055v1)

T-Rex는 Mixture-of-Transformer-Experts 구조다. 중요한 점은 expert들이 같은 속도로 돌지 않는다는 것이다. visual-language reasoning은 느리게, tactile correction은 빠르게 동작한다.

| 구성요소 | 역할 |
| --- | --- |
| Vision-language backbone | 이미지와 언어 instruction에서 task context를 만든다. |
| Latent expert | visual-language context를 처리하고 future visual representation을 예측한다. |
| Action expert | coarse action chunk를 low-frequency로 생성한다. |
| Tactile expert | tactile force/deformation을 보고 action을 high-frequency로 보정한다. |
| Tactile encoder | force history와 deformation map을 tactile token으로 바꾼다. |

### tactile encoder

T-Rex의 tactile encoder는 두 갈래다.

| stream | 입력 | 역할 |
| --- | --- | --- |
| temporal force stream | fingertip force history | 최근 contact force 변화를 VQ-VAE token으로 압축한다. |
| spatial deformation stream | tactile deformation map | 접촉 위치와 국소 변형 형태를 convolution encoder로 뽑는다. |

이 설계는 중요한 trade-off를 건드린다. Force vector는 빠르고 compact하지만 접촉 geometry를 잃기 쉽고, deformation map은 공간 정보가 풍부하지만 처리 비용이 있다. T-Rex는 둘을 함께 써서 시간적 force 변화와 공간적 접촉 형상을 동시에 잡으려 한다.

#### temporal force VQ-VAE

| 항목 | 값 / 구조 |
| --- | --- |
| 입력 | fingertip별 6D force vector의 최근 `T = 16` frame window |
| encoder | 1D temporal convolution, 두 개의 strided block으로 temporal dimension downsample |
| pooling | temporal mean pooling |
| continuous embedding | 256-dimension |
| codebook size | `K = 64` |
| codebook update | EMA update, underutilized code reseeding으로 collapse 방지 |
| decoder | quantized token에서 원 force sequence 복원 |
| loss | magnitude-weighted MSE, high-force contact frame에 더 큰 penalty |
| sharing | 5개 finger가 convolution weight를 공유하고, finger identity embedding을 주입 |

이 encoder는 손가락마다 하나의 discrete force token을 만든다. 핵심은 sensor drift와 noisy force history를 그대로 policy에 넣지 않고, contact pattern vocabulary로 압축한다는 점이다.

#### tactile deformation encoder

| 항목 | 값 / 구조 |
| --- | --- |
| 입력 | fingertip별 single-channel deformation depth map |
| backbone | ResNet-18 변형 |
| input stem | single-channel tactile map을 받도록 수정 |
| retained stages | first three residual stages |
| projection | 각 stage 뒤에 `3 x 3` convolution을 붙여 128 channel width로 재투영 |
| final step | spatial tensor flatten 후 policy token space로 linear projection |
| 학습 방식 | self-supervised convolutional autoencoder로 pretrain 후 policy learning 중 freeze |

deformation map은 force vector가 잃기 쉬운 edge, slip, shear, local contact geometry를 담는다. T-Rex는 VQ-VAE force token과 deformation embedding을 concatenation해 tactile expert가 사용할 unified tactile observation을 만든다.

## 핵심 방법론: Asynchronous Cascaded Flow Matching

<SectionFeedback page="T-Rex" section="핵심 방법론" />

T-Rex는 action generation을 flow matching으로 본다. 일반적인 flow matching은 noise에서 action으로 가는 denoising trajectory를 학습한다. T-Rex는 이 trajectory를 둘로 나눈다.

| 단계 | 담당 expert | 의미 |
| --- | --- | --- |
| 앞부분 denoising | action expert | visual-language context 기반의 coarse action 생성 |
| 뒷부분 denoising | tactile expert | 실제 tactile feedback을 반영한 빠른 action refinement |

이 구조를 “asynchronous cascaded”라고 볼 수 있다. 비싼 visual-language 계산은 action chunk 단위로 낮은 주파수에서 수행하고, tactile expert는 cached context를 재사용하면서 더 높은 주파수로 action을 보정한다.

### objective와 conditioning

두 expert는 같은 flow target velocity를 회귀하지만 condition이 다르다.

| expert | condition | 역할 |
| --- | --- | --- |
| action expert | head/wrist camera feature, language prompt, future-prediction token이 포함된 visual-language latent context | slow stream에서 noisy action을 intermediate action state까지 denoise |
| tactile expert | high-frequency tactile token, slow stream에서 detached된 intermediate state와 KV cache | raw visual observation을 다시 보지 않고 fast stream에서 남은 denoising 수행 |

전체 loss는 action expert loss, tactile expert loss, future-frame visual prediction loss를 함께 최적화한다. 논문은 `lambda_tac = 1.0`, `lambda_future = 0.5`를 사용한다. deployment와 training의 mismatch를 줄이기 위해 tactile token frame index를 visual context와 일부러 어긋나게 하는 delay augmentation도 넣는다. delay는 `{0, 4, 8, 12}` frame 중 하나로 sampling된다.

### inference loop

| stream | 주기/역할 | 계산 내용 |
| --- | --- | --- |
| slow stream | action chunk window마다 실행 | visual-language context 계산, Gaussian noise에서 `K_slow` step denoising, boundary state와 KV cache 저장 |
| fast stream | action chunk 내부 offset `{0, 4, 8, 12}`에서 실행 | real-time tactile stream을 읽고, cached KV와 boundary state를 clone한 뒤 `K_fast` Euler step으로 action chunk 갱신 |
| low-level control | 300Hz controller | policy가 비동기적으로 갱신한 target을 추종 |

thread safety를 위해 deployment runtime은 single-threaded request socket과 execution lock을 사용한다. fast tick은 in-flight slow tick이 cache와 boundary state를 shared memory에 commit하기 전에는 시작하지 않는다.

### 왜 이게 중요한가

Contact-rich manipulation에서는 몇 프레임 사이에도 상황이 바뀐다. 손가락이 살짝 미끄러지거나, 물체가 변형되거나, 삽입 축이 틀어질 수 있다. 이때 매번 큰 VLA를 다시 돌리면 느리고, tactile을 action chunk 시작점에만 넣으면 늦다. T-Rex의 구조는 이 문제를 architecture level에서 해결하려는 시도다.

## 학습 recipe

<SectionFeedback page="T-Rex" section="학습 recipe" />

T-Rex는 세 단계로 학습한다.

| 단계 | 데이터 | 목적 |
| --- | --- | --- |
| pre-training | 대규모 human egocentric video/action data | 넓은 visuomotor prior 학습 |
| mid-training | 100시간 T-Rex tactile robot dataset | robot embodiment와 tactile contact dynamics에 맞춤 |
| post-training | target task별 약 100개 demonstration | 특정 평가 task에 빠르게 적응 |

여기서 가장 중요한 단계는 mid-training이다. T-Rex는 tactile dataset을 단순 추가 데이터가 아니라, human-derived prior를 실제 robot contact dynamics에 연결하는 grounding 단계로 본다.

## 평가 task와 결과

<SectionFeedback page="T-Rex" section="평가 task와 결과" />

T-Rex는 12개 real-world tactile-reactive task에서 평가된다. 과제는 insertion, deformable object handling, force-sensitive transfer, bimanual coordination처럼 RGB만으로는 부족한 조작을 포함한다.

비교 대상은 다음 계열이다.

- vision-only 혹은 visuo-tactile imitation policy
- tactile-aware VLA
- EgoScale 계열
- pi_0 계열 flow-matching VLA
- RDP 같은 slow-fast tactile policy

평가는 task마다 16회 rollout으로 수행하고, object position과 rotation을 trial마다 randomize한다. metric은 task success rate이며, multi-stage task에는 progress-based 혹은 additive rubric을 적용해 부분 성공도 반영한다.

## 실패 사례와 한계

<SectionFeedback page="T-Rex" section="실패 사례와 한계" />

논문이 스스로 인정하는 한계는 꽤 현실적이다.

| 한계 | 해석 |
| --- | --- |

## 이 논문을 읽을 때 기억할 것

<SectionFeedback page="T-Rex" section="이 논문을 읽을 때 기억할 것" />

- tactile은 “추가 observation”이 아니라 **빠른 correction signal**로 다뤄야 한다.
- VLA의 느린 reasoning과 tactile의 빠른 feedback 사이에는 구조적 mismatch가 있다.
- T-Rex의 핵심은 이 mismatch를 variable-rate MoT와 cascaded flow matching으로 나눈 것이다.
- 100시간 tactile dataset은 task demo 모음이 아니라 reusable motor primitive와 contact behavior를 배우기 위한 mid-training corpus다.
- force history와 deformation map을 함께 쓰는 tactile representation이 중요하다.

### Appendix A. Model and Training Details

Appendix A는 T-Rex가 어느 정도 크기의 모델과 학습 자원 위에서 성립하는지 보여준다. 이 노트의 [모델 configuration](#모델-configuration)에 들어간 Table 4 정보가 이 부록에서 온다.

| 구성 | 핵심 설정 | 해석 |
| --- | --- | --- |
| latent expert | Qwen3VL-2B 기반, hidden dimension 2048, 28 layers, 1.41B params | vision-language context와 future visual prediction을 담당하는 무거운 slow expert다. |
| action expert | action dimension 62, chunk 16, inference step 6, 1.41B params | low-frequency coarse action을 생성한다. arm delta와 finger command를 한 action vector 안에 담는다. |
| tactile expert | action dimension 62, chunk 16, FFN intermediate 1536, inference step 4, 0.62B params | fast loop에서 돌기 위해 action expert보다 작게 만든다. 이 크기 차이가 variable-rate 구조의 실용성을 만든다. |
| training | AdamW, peak LR `1e-4`, cosine schedule, H100 24장, ZeRO-1, bf16 | 단순 finetuning 실험이라기보다 foundation-policy 규모의 mid/post-training recipe다. |

**해석:** T-Rex의 핵심은 tactile expert 하나를 추가한 것이 아니라, 느린 reasoning expert와 빠른 tactile expert의 계산량을 의도적으로 다르게 설계한 점이다. Appendix A를 보면 fast tactile expert가 0.62B로 작아져 있는데, 이것이 실제 robot loop에서 다시 실행 가능한 크기라는 주장을 뒷받침한다.

### Appendix B. Additional Details for Asynchronous Cascaded Denoising

Appendix B는 T-Rex의 가장 중요한 알고리즘 디테일을 보강한다. main text에서는 cascaded flow matching을 큰 그림으로 설명하지만, Appendix B는 objective, conditioning, KV cache, delay augmentation, runtime synchronization을 더 구체적으로 푼다.

| 요소 | 내용 | 왜 중요한가 |
| --- | --- | --- |
| action expert objective | visual-language latent context에 condition된 shared velocity target 회귀 | coarse action은 RGB/language 기반의 느린 reasoning에 맡긴다. |
| tactile expert objective | raw vision 없이 tactile token, detached intermediate state, KV cache에 condition | fast expert가 매번 vision backbone을 다시 돌리지 않아도 된다. |
| KV cache re-use | slow stream이 만든 visual-language cache와 partially-denoised boundary state를 fast stream이 복사해서 사용 | 계산량을 amortize하면서 tactile feedback을 action chunk 중간에 반영한다. |
| delay augmentation | visual context와 tactile frame index를 일부러 어긋나게 sampling | deployment에서 생기는 stale visual context와 real-time tactile mismatch를 학습 중에 미리 겪게 한다. |
| execution lock | slow tick이 cache를 commit하기 전 fast tick이 시작하지 않도록 lock 사용 | 실제 robot runtime에서 shared memory race condition을 막는다. |

**해석:** 이 부록은 “tactile을 input에 붙였다”와 “tactile-reactive policy를 만들었다”의 차이를 잘 보여준다. T-Rex는 tactile을 static observation으로 쓰지 않고, slow stream이 만든 action을 fast stream이 다시 denoise하는 control-frequency 문제로 재정의한다.

### Appendix C. Implementation Details for Spatial-Temporal Tactile Encoder

Appendix C는 tactile encoder가 왜 두 갈래인지 설명한다. T-Rex는 tactile을 6D force vector 하나로 끝내지 않고, temporal force dynamics와 spatial deformation geometry를 분리해서 표현한다.

| encoder | 입력 | 처리 방식 | 얻는 정보 |
| --- | --- | --- | --- |
| temporal force VQ-VAE | fingertip별 최근 force window | 1D temporal conv, strided downsampling, mean pooling, VQ codebook, EMA update | slip, grip force 변화, contact impulse 같은 시간 패턴 |
| instantaneous force projection | 현재 6D force | 직접 projection | 지금 당장의 contact magnitude |
| deformation encoder | single-channel deformation depth map | ResNet-18 변형, 앞 3개 residual stage, policy token projection | edge, shear, local contact geometry |

세부적으로 force VQ-VAE는 codebook collapse를 막기 위해 underutilized code를 current batch activation으로 reseeding하고, high-force contact frame에 더 큰 penalty를 주는 magnitude-weighted MSE를 사용한다. deformation encoder는 self-supervised convolutional autoencoder로 pretrain한 뒤 policy 학습 중 freeze한다.

**해석:** force vector는 compact하지만 contact geometry를 잃기 쉽고, deformation map은 geometry가 풍부하지만 계산량과 sensor-specific bias가 있다. T-Rex는 둘 중 하나를 고르지 않고, force는 temporal token으로 압축하고 deformation은 spatial embedding으로 보존한다. Table 2에서 `Deform only`, `MLP Force + VQ-VAE Force`, `MLP Force + Deform`이 모두 full model보다 낮은 이유를 이 부록이 설명해준다.

| pi_0.5 | OpenPI checkpoint에서 task별 finetune, bimanual joint-space setup | flow-matching VLA 계열의 대표 비교군이다. |
| pi_0.5 + tactile | state input에 single-step tactile force를 concatenate | tactile을 단순 state로 붙이면 오히려 성능이 떨어질 수 있음을 보여준다. |

**해석:** Appendix E를 보면 T-Rex의 장점은 baseline을 약하게 만든 데서만 나오지 않는다. 강한 human-pretrained baseline인 EgoScale도 포함하고, pi_0.5에 tactile을 붙인 naive variant도 비교한다. 다만 모든 baseline이 원 논문 그대로가 아니라 T-Rex hardware에 맞춰 adaptation된 결과라는 점은 기억해야 한다.

### Appendix F. Evaluation Tasks

Appendix F는 12개 evaluation task를 자세히 설명한다. 각 task는 force-reactive, tactile-deformation sensitive, bimanual coordination, long-horizon sequencing 중 하나 이상을 요구한다. 논문은 additive rubric과 progress-based rubric을 함께 사용한다.

| task 그룹 | task | tactile이 필요한 이유 |
| --- | --- | --- |
| 얇고 미끄러운 물체 | Flip Page, Extract Card, Deal Poker | 한 장만 분리하거나 friction을 일정하게 유지해야 한다. |
| fragile/deformable object | Transfer Egg, Apply Toothpaste | 너무 약하면 놓치고 너무 세면 깨지거나 과도하게 변형된다. |
| surface/contact state | Wipe Plate, Sort Mahjong | RGB만으로 압력, 표면 texture, 접촉 상태를 충분히 알기 어렵다. |
| insertion/torque | Open Lock, Screw Lightbulb | 정렬, 삽입, 회전 torque feedback이 중요하다. |
| small mechanism/liquid | Refill Tablet, Acid-Base Neutralization | 작은 버튼/뚜껑/액체 조작처럼 multi-stage precision이 필요하다. |
| nested/friction object | Split Cup | cup stack friction과 twist/rub force를 조절해야 한다. |

Appendix F의 key-stage 이미지들은 이 노트의 [task별 figure](#task별-figure)에 Figure 8부터 Figure 19까지 정리해두었다. 이 이미지들은 단순 illustration이 아니라 rubric을 이해하는 데 중요하다. 예를 들어 Extract Card는 “카드를 빼기”가 아니라 bottom card를 다시 밀어 넣고 top card만 빼야 하며, Screw Lightbulb는 잡기 성공이 아니라 thread engagement와 light-on까지 봐야 한다.

**해석:** Table 1의 평균 성공률을 읽을 때는 task별 rubric을 같이 봐야 한다. 여러 task가 partial credit을 포함하므로, 65%는 “완전 성공률만”의 의미라기보다 contact-rich multi-stage progress를 반영한 점수에 가깝다.

### Appendix G. T-Rex Dataset

Appendix G는 T-Rex Dataset이 어떻게 구성됐는지 자세히 설명한다.

![Appendix G dataset statistics](https://arxiv.org/html/2606.17055v1/x2.png)

그림 출처: [T-Rex Figure 2](https://arxiv.org/html/2606.17055v1)

| 항목 | 내용 | 해석 |
| --- | --- | --- |
| episode schema | RGB 3개, proprioception, wrist pose, tactile deformation/wrench, language instruction | policy가 vision, state, touch, language를 같은 timestamp 기준으로 배운다. |
| taxonomy | 207 objects x 22 motor primitives 후보에서 feasible pair만 유지 | dataset을 task list가 아니라 reusable primitive coverage로 설계한다. |
| 최종 규모 | 502 object-motor pair, 7,755 episodes, 100 hours | downstream task demo보다 넓은 contact behavior prior를 만든다. |
| scene diversity | 6개 tabletop backdrop, 210개 이상 distractor pool, 0-5개 distractor | language-conditioned target selection과 visual robustness를 압박한다. |
| cleaning | unstable tactile, corrupted stream, teleop failure, extreme joint velocity 제거 | tactile dataset은 sensor 품질 관리가 성능에 직접 연결된다. |
| language annotation | head camera 4-6 frame과 최소 label을 VLM에 넣고 사람이 검수 | task id가 아니라 episode motion을 설명하는 instruction을 만든다. |
| release/ethics | raw stream, derived tactile representation, annotation, loader/preprocess script 공개 계획 | 재현성을 dataset artifact 수준에서 제공하려는 방향이다. |

**해석:** Appendix G의 핵심은 100시간이라는 숫자보다 “무엇을 100시간 모았는가”다. T-Rex는 downstream task 12개만 반복 수집한 것이 아니라, object-primitive 조합을 통해 reusable contact behavior를 넓게 모았다. 그래서 Figure 6의 mid-training ablation에서 task-specific 100시간보다 T-Rex dataset mid-training이 generalization에 유리하다는 해석이 가능하다.

### Appendix H. Failure Case Analysis

Appendix H는 T-Rex가 어디서 실패하는지 보여준다.

![Appendix H failure case analysis](https://arxiv.org/html/2606.17055v1/x20.png)

그림 출처: [T-Rex Figure 20](https://arxiv.org/html/2606.17055v1)

| 실패 유형 | 예시 task | 원인 해석 | 다음 연구 질문 |
| --- | --- | --- | --- |
| Object collision | Screw Lightbulb | grasp 후 socket insertion alignment가 틀어져 base와 충돌 | visual alignment와 force-aware insertion controller를 어떻게 결합할까 |
| Slipping off | Open Lock | 작은 key를 안정적으로 유지하지 못하고 drop | grip maintenance와 in-hand stabilization을 어떻게 학습할까 |
| Imprecise position | Transfer Egg | egg를 깨지지 않게 잡지만 tray 안에 정확히 놓지 못함 | BC distribution shift를 줄이기 위해 online correction/RL이 필요한가 |
| Multi-finger friction | Sort Mahjong | thumb 위치가 낮아 다른 tile compartment까지 건드림 | finger-level collision/contact awareness를 어떻게 넣을까 |
| Excessive force | Apply Toothpaste | tube를 너무 세게 눌러 toothpaste가 과도하게 나옴 | deformable object의 force magnitude를 closed-loop로 제한할 수 있는가 |
| Sliding misalignment | Extract Card | card sleeve에서 균일한 sliding force를 못 만듦 | temporal tactile conditioning을 더 강하게 만들 방법은 무엇인가 |

**해석:** Appendix H는 T-Rex가 tactile-reactive 구조를 갖고도 여전히 offline BC 한계를 가진다는 점을 보여준다. 실패는 대부분 “touch를 못 봐서”만 생기지 않는다. alignment, fine positioning, multi-finger coordination, force magnitude regulation, recovery policy가 함께 필요하다. 그래서 후속 방향은 tactile encoder 개선만이 아니라, online adaptation, constraint-aware controller, task-level recovery policy까지 포함해야 한다.

## 저자/그룹 연구 흐름

<SectionFeedback page="T-Rex" section="저자/그룹 연구 흐름" />

T-Rex는 단독 tactile 논문이라기보다, **human egocentric data scaling**, **generalist VLA / robot foundation model**, **tactile-reactive dexterous manipulation** 세 흐름이 만나는 지점에 있다. Appendix E가 강한 baseline을 자세히 재현하는 이유도 여기에 있다. T-Rex의 주장은 “우리가 약한 baseline보다 좋다”가 아니라, 강한 VLA foundation model과 tactile baseline까지 같은 robot/action/evaluation 조건에 맞춘 뒤에도 **tactile을 architecture-level fast correction path로 넣어야 한다**는 것이다.

### 연구 흐름 요약

| 흐름 | 대표 연결점 | T-Rex에서 이어받은 문제의식 |
| --- | --- | --- |
| Human egocentric video scaling | [EgoScale](../egoscale-scaling-dexterous-manipulation/), GR00T 계열 | 사람 손/팔 동작과 egocentric video에서 dexterous prior를 먼저 얻는다. |
| Generalist VLA / robot foundation model | [pi_0](../pi0-vla-flow-model/), pi_0.5, GR00T N1 | visual-language reasoning과 continuous action generation을 foundation policy로 만든다. |
| Slow-fast tactile control | [Reactive Diffusion Policy](../reactive-diffusion-policy/), [ViTacFormer](../vitacformer-visuotactile-dexterous-manipulation/), [Tactile-VLA](../tactile-vla-physical-knowledge/) | 접촉 변화는 빠르므로 tactile feedback을 action chunk 실행 중에 반영해야 한다. |
| T-Rex의 합류점 | tactile-rich mid-training + variable-rate MoT + cascaded flow matching | human/VLA prior를 robot tactile contact dynamics에 접지하고, slow visual reasoning과 fast tactile correction을 분리한다. |

### Appendix E가 보여주는 출발점

Appendix E의 baseline 구성은 T-Rex 연구의 출발점을 읽는 데 중요하다.

| baseline 축 | Appendix E의 의미 | T-Rex의 해석 |
| --- | --- | --- |
| ViTacFormer, RDP, Tactile-VLA | tactile signal을 쓰는 task-specific 또는 tactile-aware policy 계열 | tactile을 쓰는 것만으로는 충분하지 않고, foundation-scale context와 fast correction path가 필요하다. |
| EgoScale | 대규모 human egocentric pretraining 기반의 강한 dexterous prior | T-Rex도 human video pretraining을 출발점으로 삼되, robot tactile data로 mid-training한다. |
| pi_0.5 | flow-matching VLA 계열의 general robot control baseline | continuous action flow 자체는 중요하지만, tactile feedback을 언제/어떻게 넣는지가 별도 문제다. |
| pi_0.5 + tactile | tactile force를 state input에 단순 concatenate한 variant | tactile은 input feature 하나가 아니라 action denoising 후반부를 맡는 expert로 들어가야 한다는 반례다. |

이 관점에서 T-Rex의 핵심 novelty는 tactile sensor 자체가 아니다. 이미 tactile baseline은 존재한다. 더 중요한 차이는 **tactile signal을 action expert의 output 이후 lower flow segment를 완성하는 high-frequency expert로 배치했다**는 점이다.

### 저자/그룹별 연구 맥락

| 그룹 / 저자 흐름 | 공개 연구 소개에서 보이는 방향 | T-Rex와의 연결 |

| Zhuoyang Liu 계열 | VLA models in robot manipulation, reasoning/action 분리, MoT, hybrid VLA 연구 흐름 | T-Rex의 variable-rate Mixture-of-Transformer-Experts와 slow/fast expert 분리와 맞닿아 있다. |

여기서 중요한 점은 T-Rex가 어느 한 그룹의 기존 연구를 단순 확장한 것이 아니라는 점이다. Berkeley 쪽의 VLM/VLA robotics, NVIDIA GEAR 쪽의 robot foundation model, human egocentric data scaling, tactile-reactive imitation learning이 같은 시스템 안에서 만난다.

### 한 문장으로 정리

T-Rex는 “tactile sensor를 추가한 VLA”라기보다, **human video foundation prior를 robot tactile contact dynamics에 접지시키고, visual-language reasoning과 tactile correction의 control-frequency mismatch를 variable-rate MoT와 cascaded flow matching으로 푼 연구**로 읽는 것이 가장 정확하다.
