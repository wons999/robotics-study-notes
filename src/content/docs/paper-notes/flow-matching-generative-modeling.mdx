## One-line Summary

Flow Matching trains a continuous normalizing flow by regressing vector fields along probability paths from noise to data, avoiding simulation during training.

## Problem Setting

- Data: high-dimensional generative modeling data.
- Objective: learn a vector field that transports samples from a simple prior distribution to a target data distribution.
- Model: continuous normalizing flow trained with fixed conditional probability paths.

## Core Idea

Instead of training by repeatedly simulating a flow during learning, Flow Matching directly regresses the vector field associated with a chosen probability path. Diffusion-like paths are one option; optimal-transport-inspired paths are another.

## Method Details

- Architecture: neural vector field over time and data/noise state.
- Training recipe: sample intermediate points along a path and regress the target velocity.
- Data: general generative modeling data.
- Inference / control loop: integrate the learned vector field to generate samples; in robotics, actions can be treated as generated samples.

## What I Should Remember

- pi_0 uses flow matching for continuous action generation.
- T-Rex splits a flow-matching trajectory into slow action-expert and fast tactile-expert segments.
- Flow matching is a clean way to generate continuous high-DoF action chunks without discretizing every action dimension.

## Limitations

- It is a generic generative modeling method, not a robot control solution by itself.
- Action-generation performance depends on conditioning, data, and real-time inference constraints.
- Numerical integration choices affect runtime and quality.
