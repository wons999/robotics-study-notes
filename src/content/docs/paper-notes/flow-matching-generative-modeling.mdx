# Flow Matching

## 메타데이터

| 항목 | 내용 |
| --- | --- |
| 논문 | Flow Matching for Generative Modeling |
| 저자 | Yaron Lipman, Ricky T. Q. Chen, Heli Ben-Hamu, Maximilian Nickel, Matt Le |
| 연도 | ICLR, 2023 |
| 링크 | https://arxiv.org/abs/2210.02747 |
| 태그 | flow matching, generative modeling, continuous normalizing flow, action generation |

## 한 줄 요약

Flow Matching은 noise distribution에서 data distribution으로 이동하는 vector field를 직접 regression하여, simulation-free 방식으로 continuous generative model을 학습하는 방법이다.

## 왜 T-Rex와 연결되는가

T-Rex의 action expert와 tactile expert는 flow matching 기반 denoising/action generation 관점 위에 있다. 따라서 Flow Matching은 T-Rex의 핵심 수학적 기반 중 하나다.

## 직관

데이터 분포를 바로 맞추기 어렵다면, 간단한 noise에서 시작해 점점 data로 이동하는 흐름을 학습할 수 있다. Flow Matching은 이 흐름의 각 지점에서 “어느 방향으로 움직여야 data에 가까워지는가”를 vector field로 학습한다.

robotics action generation으로 바꾸어 생각하면 다음과 같다.

| generative modeling | robot action generation |
| --- | --- |
| data sample | expert demonstration의 action chunk |
| noise | random action trajectory |
| vector field | action을 더 그럴듯한 trajectory로 바꾸는 update direction |
| sampling | learned vector field를 따라 action 생성 |

## 방법론

Flow Matching은 fixed conditional probability path를 정하고, 그 path 위의 intermediate point에서 target velocity를 regression한다. Diffusion path도 쓸 수 있고, optimal transport inspired path도 쓸 수 있다.

중요한 차이는 training 중에 expensive simulation을 반복하지 않고, vector field regression objective로 학습할 수 있다는 점이다.

## T-Rex에서의 역할

T-Rex는 flow matching denoising trajectory를 두 구간으로 나눈다.

| 구간 | 담당 | 의미 |
| --- | --- | --- |
| 앞부분 | action expert | vision-language context로 coarse action 생성 |
| 뒷부분 | tactile expert | tactile feedback으로 action refinement |

이 구조 때문에 T-Rex의 방법론을 읽을 때 Flow Matching은 단순 배경지식이 아니라, action expert와 tactile expert가 어떻게 같은 action distribution을 나누어 다루는지 이해하는 핵심이다.

## 읽을 때 주의할 점

- Flow Matching 자체는 robot control 방법이 아니다.
- robot에서 성능은 conditioning, data quality, inference latency, controller interface에 달려 있다.
- T-Rex의 새로움은 flow matching을 쓴 것 자체보다, tactile expert가 denoising 후반부를 빠르게 보정하도록 설계한 점이다.
