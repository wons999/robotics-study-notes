# Open X-Embodiment / RT-X

## 메타데이터

| 항목 | 내용 |
| --- | --- |
| 논문 | Open X-Embodiment: Robotic Learning Datasets and RT-X Models |
| 저자 | Open X-Embodiment Collaboration |
| 연도 | arXiv, 2023 |
| 링크 | https://arxiv.org/abs/2310.08864 |
| 태그 | robot dataset, cross-embodiment learning, RT-X, VLA, robot data scaling |

## 한 줄 요약

Open X-Embodiment는 21개 기관, 22개 robot embodiment의 manipulation data를 표준화해 cross-robot policy training과 RT-X model을 가능하게 한 대규모 robot dataset 흐름이다.

## 이 논문의 위치

Open X-Embodiment / RT-X는 robot learning에서 cross-embodiment data scaling이 실제 generalization에 어떤 영향을 주는지 보여주는 핵심 논문이다. 이후 robot foundation model 논문을 읽을 때 “데이터 규모와 embodiment 다양성”의 기준점으로 쓸 수 있다.

| 관점 | Open X-Embodiment | 읽을 때 확인할 질문 |
| --- | --- | --- |
| 목표 | 여러 robot embodiment를 아우르는 broad robot data scaling | embodiment diversity가 실제 transfer에 얼마나 도움이 되는가 |
| modality | 주로 vision, action, robot state 중심 | force/tactile이 없는 dataset은 어떤 조작을 놓치는가 |
| 장점 | cross-embodiment generality | 데이터 규모와 modality richness의 trade-off는 무엇인가 |
| 약점 | dense tactile/contact modality가 부족할 수 있음 | high-DoF hand data는 어떻게 표준화할 수 있는가 |

## 데이터셋

Open X-Embodiment는 다양한 로봇과 과제를 하나의 표준 format으로 묶는다. 논문은 22개 robot, 527 skills, 160,266 tasks 규모를 보고한다. 중요한 점은 “각 로봇마다 따로 policy를 학습한다”는 관행에서 벗어나, 여러 robot data를 합쳐 positive transfer를 노린다는 것이다.

## 모델 구조와 방법론

RT-X는 이런 multi-robot dataset 위에서 학습되는 generalist robot policy family다.

| 구성 | 의미 |
| --- | --- |
| standardized data format | 서로 다른 robot data를 하나의 training pipeline에 넣기 위한 기반 |
| multi-robot training | 다른 robot의 경험을 활용해 target robot 성능을 높임 |
| high-capacity policy | 다양한 embodiment/task를 하나의 model에 압축 |
| downstream adaptation | 새 robot/task에 더 효율적으로 적응 |

## 기억할 점

- Open X-Embodiment는 broad data scaling의 대표 사례다.
- 앞으로 robot foundation model에는 broad embodiment data와 rich physical feedback data가 모두 필요할 가능성이 크다.
- robot dataset을 볼 때는 robot 수, task 수, modality, control interface를 함께 봐야 한다.

## 한계

- cross-embodiment data를 모아도 tactile/force/contact modality가 충분하지 않으면 contact-rich task는 약할 수 있다.
- robot morphology가 다르면 action representation과 controller interface가 크게 달라진다.
- high-DoF hand data는 일반 manipulator data보다 훨씬 표준화가 어렵다.
